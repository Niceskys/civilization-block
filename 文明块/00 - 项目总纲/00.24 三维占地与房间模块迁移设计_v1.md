# 00.24 三维占地与房间模块迁移设计 v1

> 本文件裁决从“每Plot每Layer单建筑”迁移到“统一三维网格与可变占地”的实现边界。它是迁移任务书，不直接改写01至10目录的现行玩法数值；每个阶段验收后才回写对应A级规则。

## 一、为什么必须迁移

当前Runtime用`PlotId + Layer`定位建筑，同一地皮同一层只能有一个建筑。该模型无法可靠支持：

- 3x3完整房间模块；
- 不同宽度、深度和高度的建筑；
- 90度旋转后的占地变化；
- 多栋建筑共同支撑上层模块；
- 连廊、墙、管线和陷阱等连接或附着设施；
- 建筑剖面中可读的房间、库存与物流位置。

禁止直接把现有`1x1`数值替换为`3x3`。这样会让碰撞、承重、存档和联机状态同时失真。

## 二、空间模型裁决

### 2.1 基础网格

- 保留2m水平基础格，所有权威坐标使用整数，不使用浮点坐标参与模拟、存档或联机校验。
- `Plot`改为一片拥有边界的土地容器，不再等同于一个建筑槽位。
- 建筑位置使用`AnchorX + AnchorY + BaseLayer`；锚点是旋转前占地矩形的统一基准角。
- 建筑定义增加`FootprintWidth + FootprintDepth + FootprintHeight`，均为正整数。
- 旋转只允许`0/90/180/270`度；90或270度交换宽度和深度。
- 首个兼容版本中，未声明占地的旧定义默认`1x1x1`。

### 2.2 房间模块

- 主要建筑以3x3水平格的完整房间模块作为首选视觉与空间方向，即约6m x 6m。
- 并非所有对象强制3x3；道路、楼梯塔、仓储、生产和大型公共建筑可使用不同整数占地。
- 墙、管线、连廊、陷阱等不再用`0.5格建筑`模拟，改为`Attachment/Connector`类型，占用格面、格边或连接槽。
- 附着设施默认不承重；需要承重时必须在定义中明确声明，不能继承普通建筑规则。

### 2.3 占用快照

- 建筑定义提供默认占地，但建筑实例和施工任务必须保存放置时的占地快照与旋转。
- 模组更新或定义改动不得让旧建筑在读档后突然扩张、重叠或改变承重接触面。
- 施工任务与完工建筑使用同一占用计算，不允许施工预留与完工碰撞规则不同。

## 三、承重模型裁决

- 承重从“同一Plot下层单建筑”改为基于三维占用格的有向支撑图。
- 上层建筑的底面占用格与下一层建筑顶面相接时形成支撑接触；同一支持者的多个接触格合并记录。
- 建筑自重与收到的上层载荷按接触格数量确定性分配给直接支持者。
- 计算使用整数定点数，排序使用稳定ID，禁止浮点误差造成单机、服务器和回放结果不同。
- 支撑图只从高层指向低层，不允许环；验证顺序从最高层向地基层推进。
- 部分悬空、最低支撑比例、拆除后的30分钟修复期仍保留为玩法目标，但具体比例必须在承重专项阶段通过测试后回写1.2。
- 地面、自然岩层和特殊地基使用显式支持节点，不能用不存在的隐藏建筑代替。

## 四、建议数据契约

```text
BuildingDefinition
  FootprintWidth
  FootprintDepth
  FootprintHeight
  OccupancyKind = Solid | Attachment | Connector

BuildingInstanceState / ConstructionTaskState
  PlotId
  AnchorX
  AnchorY
  BaseLayer
  RotationQuarterTurns
  PlacedWidth
  PlacedDepth
  PlacedHeight
  PlacementSchemaVersion

BuildCommandPayload
  DefinitionId
  PlotId
  AnchorX
  AnchorY
  BaseLayer
  RotationQuarterTurns
```

`PlacedWidth/Depth/Height`是存档快照，不从新版本定义反推。未来若引入非矩形占地，应增加版本化占用掩码，不复用字符串拼接坐标。

## 五、分阶段实施

### Phase A：兼容数据层

状态：已完成（2026-06-19）。

- 增加占地、锚点、旋转和schema字段；
- 所有默认值保持旧行为`1x1x1`；
- SaveSystem把旧`Layer`迁移为`BaseLayer`，锚点来自旧Plot坐标；
- 新旧存档往返、StateHash和远程快照测试必须通过。

实现结果：存档版本升级为1.1；旧`PlotId + Layer`迁移为锚点与放置快照；定义、运行时验证和读档均拒绝尚未启用的多格、旋转、附着与连接占用。

### Phase B：纯占用服务

状态：已完成（2026-06-19）。

- 建立独立`SpatialOccupancy`组件；
- 输入定义、实例坐标和旋转，输出确定性的占用格集合；
- 测试边界、旋转、重叠、施工预留、跨Plot和越界；
- 此阶段不修改承重和建筑数值。

实现结果：新增纯`SpatialOccupancy`组件，以整数坐标和实例占地快照生成确定性的`Layer -> Y -> X`占用格序列；支持矩形占地四向旋转、边界与重叠校验、施工预占冲突详情，并为单个对象设置256格单轴、65,536格总占用上限。非法尺寸、旋转、坐标溢出、越界和损坏的既有占用状态均明确失败；该组件随后已由Phase C接入权威建造入口。

### Phase C：建造入口切换

状态：已完成（2026-06-19）。

- `BuildingSystem`只通过占用服务验证放置；
- 命令、事件、诊断和错误码同步使用新坐标；
- 禁止UI或模组自行复制碰撞算法。

实现结果：`PlotState`增加默认`1x1`的整数宽深边界；`BuildCommandPayload`可选提交成对锚点、基础层与四分之一旋转，旧命令仍回退到Plot原点和旧`Layer`。`BuildingSystem`对完工建筑与施工预占统一调用`SpatialOccupancy`，放置和完工事件携带完整快照；多格`Solid`定义已开放，`Attachment/Connector`继续关闭。存档升级至1.2并在读档时拒绝越界或重叠状态，诊断使用同一空间规则。命令拒绝增加稳定错误码并在远端传输中保留；因协议载荷变化，会话协议升级至2。现行18类正式建筑数值尚未批量修改，承重仍等待Phase D替换。

### Phase D：支撑图与承重

状态：D1纯支撑图、D2a权威建造/诊断接入与D2b拆除宽限状态机均已完成（2026-06-19）。

- 建立确定性支撑图与载荷传播；
- 覆盖共同支撑、部分悬空、多层传递、拆除和施工取消；
- 单机、服务器、存档重载和不同注册顺序必须得到同一结果。

D1实现结果：新增纯`StructuralSupport`组件，使用1000倍整数定点载荷与稳定ID顺序，从高到低建立有向支撑边；同一支持者的接触格合并，共同支撑按接触格比例分配并精确守恒。正常建造最低接触比例裁决为50%且向上取整；地面为显式基础支撑。施工任务传递重量但完工前不提供承重。组件限制最多10,000节点和1,000,000占用格，并明确拒绝重复ID、空间重叠、无支撑、接触不足、容量超限及算术溢出。

D2a实现结果：`BuildingSystem`已删除旧`PlotId + Layer`承重汇总代码，每次放置在扣资源前验证同地皮完整支撑图；完工实例可按定义承重，施工任务只传递重量。`StateDiagnostics`复用同一组件报告无支撑、接触不足、容量超限、规模与算术问题；结构错误码经服务器传输保持不变。因新增协议错误分类，会话协议升级至3。

D2b时间与难度契约已完成：`1 Tick = 1游戏秒`，1倍速每现实秒推进72 Tick，2倍速推进144 Tick，暂停为0；退出期间不补算。简单/普通/困难/极难的1倍速结构宽限分别为60/30/15/5现实分钟，连续崩塌间隔为10/5/3/1现实秒；自定义允许0至120分钟、1至10秒，并可选择“停用但不自动摧毁”。配置写入世界存档，存档版本升级至1.3，联机协议升级至4。

D2b状态机实现结果：新增权威拆除命令与`normal/grace/disabled`持久状态；拆除后统一重算受影响结构并写入截止Tick。失稳建筑保留空间占用但不提供承重，允许玩家在空出的下层位置补建支撑；支撑完工后自动解除失稳状态。自动崩塌按最高占用层降序、同层稳定ID升序及难度间隔推进；“仅停用”模式保留建筑实例。正常拆除按实际支付成本返还75%，记录容量溢出，坍塌不退款；统一运行状态入口供后续工人、生产和物流系统使用。存档版本升级至1.5，联机协议升级至6。

D2b后续接入结果：最小NPC岗位权威状态、生产批次与建筑局部库存均已完成。确定性物流任务真实锁定源资源、预约目标容量并在到期时原子改变库存；正式管道连接设施已具备施工、取消、完工、配置、拆除及端点摧毁生命周期，完工路线固定为下层到上层与单一资源，每根管道最多调度一个自动批量任务。存档版本升级至1.9，联机协议升级至10。管道暂用专用连接状态，不提前开放半格通用`Connector`占用；后续Phase E再确定其表现锚点和连接空间schema。

### Phase E：玩法与文档迁移

状态：基础入口已完成，普通建筑正在逐栋迁移（2026-06-20）。

- 已建立`RuntimeComposition`统一装配、`IDefinitionModule`内容边界及定义快照/封存/重名拒绝；
- 已注册4.1全部16种正式资源，并在封存时校验建筑成本、管道成本、配方资源与所属建筑引用；
- 已将完成生命周期迁移的管道作为首个正式核心内容注册；
- 房屋、农田、水井、树场已逐栋完成首批迁移：均使用标准`1x1x1`占地与20单位本地缓存；派生重量依次为1、1、2、1；房屋0工位，其余各2工位；
- 树场应急炭化已作为正式批次配方注册；农田、水井、树场的基础持续产出已由独立权威系统实现，不伪装为手动批次；
- 采掘场与熔炉房已完成第二批迁移：均使用标准`1x1x1`占地、2工位与30单位本地缓存，派生重量分别为3和4；采掘副产物与熔炼燃料输入已进入权威模拟；
- 仓库已完成第三批迁移：使用标准`1x1x1`占地、派生重量4且无本地缓存，向全局共享容量提供200单位增量；
- 垃圾处理场已完成第四批迁移：使用标准`1x1x1`占地、派生重量4、1工位与30单位本地缓存，四模式共用单生产槽；
- 其余9类普通建筑不得在占地、重量、库存与配方未核对时批量注册。

- 逐栋确认18种建筑的占地，不批量套用3x3；
- 回写1.1、1.2、1.5、2.1、2.2、2.3、5.2、5.3、6.1及建筑详情；
- 将墙、管线、连廊和陷阱迁移为附着或连接设施；
- 建立旧存档视觉兼容策略后才能修改正式数值。

### Phase F：表现层

- 默认城市视角只渲染外部与低成本状态信号；
- 选中建筑后按需启用室内、NPC工作、库存摆放和物流路径；
- 表现层读取占用和模拟结果，不参与权威碰撞或产出计算。

## 七、验收底线

- 旧存档加载后位置、占用和承重行为不变；
- 所有坐标、旋转、占用和载荷计算确定且可回放；
- 同一施工任务不能与完工建筑产生不同占地；
- 定义更新不能改变已放置实例的占地快照；
- 客户端不能提交越界、重叠或非法旋转并污染服务器状态；
- 模组必须声明空间schema版本，缺失时使用明确兼容规则；
- Phase A至D全部通过前，不将正式建筑批量改为3x3。
