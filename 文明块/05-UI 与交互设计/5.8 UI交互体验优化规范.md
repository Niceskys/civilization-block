# UI交互体验优化规范
> 基于HCI研究 + 城市建造类游戏UI设计最佳实践
> 目标：每一个操作≤2次点击，每一个信息≤1.5秒找到

---

## 一、HCI准则审查：当前UI存在的10个问题

基于Fitts's Law、Hick's Law、Jakob's Law、Miller's Law等人机交互研究，对当前UI设计进行全面审查：

| # | 问题 | 违反原则 | 后果 | 位置 |
|---|------|---------|------|------|
| 1 | 底栏8个按钮（B/S/Q/N/T/R/L/D） | ❌ Miller's Law（7±2临界）+ Hick's Law（选项过多） | 玩家找按钮需要扫读+思考，每次多花0.5~1秒 | 5.1底栏 |
| 2 | 面板缺少层级管理 | ❌ 缺少Doherty Threshold（反馈不够快） | 面板叠面板，玩家找不到"刚才打开的NPC详情" | 多个面板 |
| 3 | 资源栏默认只显示5种，其余要展开 | ❌ 违反了"信息可见性"原则 | 玩家不知道"有哪些资源" → 漏看关键数据 | 5.1顶栏 |
| 4 | 建筑面板Tab过多（全部/生产/居住/防御/功能/科技/装饰/规划） | ❌ Hick's Law（8个分类选项） | 切换标签找建筑耗时增加 | 5.2建造面板 |
| 5 | 建筑详情面板信息堆叠 | ❌ 缺少Progressive Disclosure（渐进式呈现） | 玩家刚点击建筑就被大量信息淹没 | 5.3 |
| 6 | 快捷键需要记忆（无提示） | ❌ 违反Jakob's Law（应使用通用模式） | 新手不知道按N打开NPC列表 | 5.6 |
| 7 | NPC管理面板缺少快速操作 | ❌ 违反Fitts's Law（操作目标太小） | 频繁操作需要精确点击，疲劳累积 | 5.4 |
| 8 | 没有"全局搜索" | ❌ 缺少Tesler's Law的补偿设计 | 建筑多了找不到某个特定建筑 | 全局 |
| 9 | 面板关闭只能点击"X" | ❌ 违反"容错原则" | 误关面板后需重新打开 | 所有面板 |
| 10 | 缺少视觉反馈层级 | ❌ 违反"状态可见性原则" | 点击按钮后不确定是否生效 | 全局 |

## 二、核心优化方案（7大改进）

### 优化1：底栏重构 — 从8个按钮→5个核心按钮（符合Miller's Law）

#### 旧底栏（8按钮 + 无分类）
```
[B建造] [S商店] [Q任务] [N人物] [T科技] [R贸易] [L日志] [D数据]
```

#### 新底栏（5按钮 + 双状态折叠 + 快捷键可见）
```
┌─────────────────────────────────────────────────────────────┐
│ [🏗️建造 B] [🏛️城市 C] [📋任务 Q] [🗺️地图 M] [⚙️设置 Esc]   │
│                                       [⏸️▶▶▶] [☀️ D:45]    │
└─────────────────────────────────────────────────────────────┘
```

| 按钮 | 快捷键 | 展开后包含 |
|------|--------|----------|
| 🏗️ **建造** B | B | 建筑列表（分类收起在子菜单中） |
| 🏛️ **城市** C | C | 商店+NPC列表+科技树+贸易+数据仪表盘+挑战（5合1） |
| 📋 **任务** Q | Q | 任务列表+事件日志+成就 |
| 🗺️ **地图** M | M | 世界全景+生物群系+特殊地皮查看 |
| ⚙️ **设置** Esc | Esc | 设置+存档+退出+名人堂+蓝图管理 |

**设计意图**：
- 5个按钮 < 7±2，玩家一目了然
- 每个按钮打开一个**聚合面板**（而非一个独立面板），减少面板数量
- 快捷键大写字母显示在按钮上，新手无需记忆

---

### 优化2：面板聚合管理 — "城市"按钮 = 5合1枢纽面板

#### 聚合面板结构
```
┌────────────────────────────────────────────────────┐
│ 🏛️ 城市                              [×] [📌] [—] │
├────────────────────────────────────────────────────┤
│ [商店] [人物] [科技] [贸易] [数据] [挑战]           │ ← 子Tab一行
├────────────────────────────────────────────────────┤
│ 【当前Tab内容区域】                                 │
│                                                    │
│  统一滚动条/统一风格/统一关闭机制                     │
│                                                    │
└────────────────────────────────────────────────────┘
```

**设计意图**：
- 玩家打开"城市"后，所有城市管理功能集中在一个大面板里
- 子Tab切换 = 0.1秒，不需要关闭再打开另一个面板
- 📌固定按钮 = 面板保持打开（即使点击空白处也不关闭）
- — 最小化按钮 = 面板收起到底栏（不关闭）

### 优化3：信息层级 — 渐进式呈现（Progressive Disclosure）

#### 建筑详情面板：3级深度

| 级别 | 触发 | 内容 | 交互目标大小 |
|------|------|------|------------|
| **Lv1-悬浮** | 鼠标悬停建筑1秒 | 建筑名+等级+当前状态图标+进度条 | 不占用点击 |
| **Lv2-点击** | 左键单击建筑 | 简洁面板：名称/耐久条/产出/工人/3个操作按钮 | **按钮≥44px**（Fitts's Law最低标准） |
| **Lv3-详情** | 点击Lv2的"▶详情" | 全部信息：属性/历史/社交/升级 | 滚动条查看 |

#### NPC详情面板：2级深度

| 级别 | 触发 | 内容 |
|------|------|------|
| **Lv1-悬浮** | 鼠标悬停NPC 0.5秒 | 名字+职业+效率数值+头顶进度环颜色 |
| **Lv2-点击** | 左键单击NPC | 详情面板（属性+社交+操作按钮） |

- [x] 快捷键使用**单大写字母**（避免Shift+、Ctrl+等组合键）
- [x] 快捷键冲突时，优先度高者保留单键，次者使用Alt+键

| Space | 暂停/继续 | ⭐核心 |
| 1 | 时间1倍速 | ⭐核心 |

- [x] 玩家新手期（前10游戏日），底栏上方显示浮动快捷键提示条：

### 优化6：面板操作三件套 — 所有面板统一的交互模式

#### 统一面板控件
| 控件 | 位置 | 功能 | 操作面积 |
|------|------|------|---------|
| 📌 **固定** | 面板右上角第1个 | 点击后面板不随空白处点击关闭 | ≥40×40px |
| — **最小化** | 面板右上角第2个 | 面板收起为底栏标签 | ≥40×40px |
| × **关闭** | 面板右上角第3个 | 关闭面板 | ≥40×40px |
| 面板标题栏 | 面板顶部 | 鼠标拖拽移动面板 | 全宽 |

#### 面板行为规则
- [x] 所有面板必须支持：拖拽移动、最小化、固定、关闭
- [x] 面板不互相遮挡标题栏（新面板在旧面板下方偏移20px打开）
- [x] 固定后的面板**半透明化**（不遮挡游戏画面）
- [x] 最小化后的面板缩小为底栏标签（点击恢复）
- [x] 打开面板数量上限：5个（超过时最早打开的面板自动最小化）
- [x] **面板背景**：采用高斯模糊效果（非纯色背景），让玩家仍能看到背后游戏画面
  - 模糊强度：15px（足够辨别背后内容，但不干扰阅读文字）

### 优化7：视觉反馈 — 每一次操作都有"被感知"的反馈

| 操作 | 视觉反馈 | 听觉反馈 | 触觉反馈（手柄） |
|------|---------|---------|----------------|
| 点击按钮 | 0.95倍缩放（100ms） | 轻微"滴"声 | 微震动 |
| 悬停按钮 | 1.05倍放大+颜色微变 | — | — |
| 拖拽面板 | 半透明跟随+阴影 | — | — |
| 无效操作（资源不足） | 红色闪烁+按钮抖动 | 错误提示音 | 双震 |
| 成功操作（建造确认） | 绿色闪光 | 确认音效 | 单震 |
| 信息更新（产出变化） | 数值飘动动画（+数字上浮/下浮） | — | — |
| 警告（资源不足） | 顶栏资源图标变为橙色/红色+脉动 | 柔和警告音 | — |
| BOSS出现 | 全屏红色边框闪烁3次 | 警报+低音轰鸣 | 持续震动 |
| NPC死亡 | 屏幕左上角弹出小头像+灰色淡出 | 悲伤旋律短音 | — |

#### 数值变化可视化
- [x] **产出数字实时飘动**：当建筑产出资源时，该资源图标旁边显示 **"+2↑"**（绿色上浮）或 **"-1↓"**（红色下浮）
- [x] **对比箭头**：每日结算面板中所有数据旁显示与昨日的对比（↑/↓/—）
- [x] **进度环**：建筑建造/升级/维修时，建筑周围显示半透明进度环

## 三、UI易用性检查清单

### 视觉舒适性

| 标准 | 当前状态 | 优化方案 |
|------|---------|---------|
| 文字大小 | 12px~14px | **最小16px**（1080p基准，低于16px阅读疲劳） |
| 按钮大小 | 32×32px | **最小44×44px**（Fitts's Law推荐至少44px触摸目标） |
| 颜色对比度 | 暗色面板+灰色文字 | **至少4.5:1对比度**（WCAG AA标准） |
| 行高 | 1.2em | **1.5em**（增加可读性） |
| 面板圆角 | 无 | **8px圆角**（减少尖锐感） |
| 面板阴影 | 无 | **柔和阴影**（层级感） |

### 点击舒适性

| 场景 | 旧设计 | 新设计 |
|------|--------|--------|
| NPC列表中点击NPC | 整行可点击 | 整行可点击+高亮反馈 |
| 资源购买数量调整 | 加减按钮（小） | 滑块+预设按钮（×5/×10/全部） |
| 建筑建造确认 | 面板底部按钮 | 鼠标左键直接放置（虚影模式下） |
| 面板关闭 | 仅×按钮 | ×/Esc/点击面板外/右键 四种方式 |

### 展开舒适性

| 场景 | 优化 |
|------|------|
| 面板打开速度 | 所有面板打开动画 ≤100ms（Doherty Threshold上限400ms） |
| 面板切换 | 子Tab切换无延迟（预加载下一个Tab内容） |
| 滚动条 | 所有可滚动区域使用**圆角滚动条**+**触底加载**（无分页器） |
| 列表加载 | 超过50条数据时使用**虚拟列表**（只渲染可见部分） |

### 贴合游戏风格

- [x] UI整体风格：**嵌入游戏的半透明悬浮式**，不是独立窗口
- [x] 面板背景：使用**高斯模糊**效果（不是纯色），让玩家始终看到背后的城市
- [x] 字体：使用**像素风格字体**（如Press Start 2P, 04b03）但不小于16px
- [x] 图标：所有资源/建筑使用**方块风格的像素图标**，与3D体素风格一致
- [x] 分隔线：使用**发光晶脉线条**（淡蓝色细线），呼应世界观
- [x] 色彩主调：深色背景+金色强调+蓝色信息，与"文明块/晶脉"世界观一致

## 四、UI设计原则总结

```
┌─────────────────────────────────────────────────────┐
│                  UI设计总原则                          │
├─────────────────────────────────────────────────────┤
│                                                     │
│  🎯 1. 每一个操作 ≤ 2次点击                           │
│      例外：删除/转职等不可逆操作 → 必须二次确认        │
│                                                     │
│  🎯 2. 每一个信息 ≤ 1.5秒找到                        │
│      超过1.5秒 → 需要全局搜索或快捷键                  │
│                                                     │
│  🎯 3. 所有按钮 ≥ 44×44px（Fitts's Law）             │
│      越小 → 点击越疲劳 → 玩家越烦躁                    │
│                                                     │
│  🎯 4. 面板数量 ≤ 5（同时打开上限）                   │
│      超过5个 → 最早打开的面板自动最小化                 │
│                                                     │
│  🎯 5. 快捷键 ≤ 5个核心（B/C/Q/M/Esc）               │
│      超过5个 → 使用Alt+组合键（二级）                  │
│                                                     │
│  🎯 6. 反馈延迟 ≤ 100ms（Doherty Threshold）         │
│      超过100ms → 显示加载动画/进度条                   │
│                                                     │
│  🎯 7. 文字大小 ≥ 16px（1080p基准）                   │
│      小于16px → 玩家前倾阅读 → 颈椎疲劳 → 放弃游戏    │
│                                                     │
│  🎯 8. 面板永远半透明（高斯模糊背景）                   │
│      不遮挡游戏画面 → 玩家感觉"UI在游戏里，不是上面"    │
│                                                     │
└─────────────────────────────────────────────────────┘
```

## 六、教程提示状态机

### 教程节点定义
- 每一条独立教程提示称为"教程节点"。
- 每个教程节点必须拥有稳定且唯一的教程标识。
- 教程标识一旦进入正式存档，不得因修改显示文案、调整提示位置、调整图标、调整目标控件名称或调整文件章节位置而随意更换。
- 每个教程节点至少定义：教程标识、所属教程阶段、前置节点、触发条件、完成条件、展示方式、目标UI或目标对象、队列优先级、暂时关闭后的重新触发条件、是否允许跳过。
- 具体程序字段名由实现阶段决定，本轮不固定英文变量名。
- 教程节点与任务系统是两个独立系统：教程节点负责解释操作；主线或支线任务负责目标、进度和奖励。任务事件可以作为教程触发条件；教程完成不得自动完成任务；教程跳过不得自动完成任务；任务完成可以使对应教程静默完成；教程提示不得自行发放任务奖励。

### 状态列表
每个存档中的每个教程节点使用以下八种状态：
- **未解锁**：前置条件尚未满足，不得展示也不得占用教程队列。
- **可触发**：前置条件已满足，但实际触发事件尚未发生，保持等待不主动弹出。
- **已排队**：触发条件已发生，完成条件尚未满足，当前可能有其他更高优先级提示正在展示，或目标UI暂时不可见。
- **展示中**：教程提示已经显示。全局同一时刻最多只能有一条教程提示处于展示中。展示期间游戏时间继续运行，不强制暂停游戏，不锁定与教程无关的操作，不强制玩家按照唯一顺序点击，不使用全屏阻断弹窗。可以高亮目标控件，可以显示指向箭头、边框或锚点说明。只有其他A级规则明确要求阻断式确认时，才允许使用阻断窗口。
- **暂时关闭**：玩家点击普通关闭按钮后进入的状态。不等于已完成或已跳过。暂时关闭后当前提示立即隐藏，同一连续触发条件不得立刻再次弹出，必须等触发条件先失效、再次从无效变为有效才允许重新排队，或等待玩家重新进入该功能界面后由对应节点的重新触发条件决定。不得在每帧、每秒或每次状态刷新时重复弹出。
- **已完成**：终态。只有实际完成条件成立时进入。完成条件必须依据正式游戏状态或业务事件判断，不得仅因玩家看过提示、点击"知道了"、提示展示时间结束或关闭提示就自动标记为已完成。节点进入已完成后立即从队列移除，若正在展示则立即结束展示，永不自动再次弹出，存档读档后保持已完成。
- **已跳过**：终态。只有玩家明确点击"不再提示"或明确执行单节点跳过操作后进入。表示玩家选择不再自动接收该节点，从队列移除，不自动再次展示，不视为玩家完成了对应业务操作，不完成主线或支线任务，不发放任何奖励，不改变游戏机制状态。普通关闭按钮不得把节点标记为已跳过。

### 状态转换
正式状态转换如下：
- 未解锁 → 前置条件满足 → 可触发
- 可触发 → 触发事件发生 → 已排队
- 已排队 → 获得展示资格且目标可用 → 展示中
- 展示中 → 玩家普通关闭 → 暂时关闭
- 暂时关闭 → 重新触发条件成立 → 已排队
- 任意非终态 → 完成条件成立 → 已完成
- 展示中或已排队 → 玩家明确选择不再提示 → 已跳过

不得出现以下自动转换：展示中超时→已完成；展示中普通关闭→已完成；展示中普通关闭→已跳过；暂时关闭每帧检测→已排队；已跳过自动重新展示；已完成自动重新展示。

### 触发和完成顺序
- 教程系统收到业务事件后，必须先完成业务状态写入，再检查节点完成条件。已满足完成条件的节点直接标记已完成。然后再检查教程触发条件。仍未完成且满足触发条件的节点进入队列。最后尝试展示队首节点。
- 目的是避免玩家已经完成操作后教程仍然延迟弹出。

### 重复触发去重
- 同一个教程节点重复收到触发事件时：未入队时可以进入队列一次；已排队时不得创建第二个队列项；展示中时不得再创建队列项；暂时关闭且重新触发条件未满足时不得入队；已完成或已跳过时忽略触发。
- 同一逻辑事件被多个系统重复上报时，按教程标识和逻辑事件实例去重。不得依赖显示文案进行去重。

### 全局教程队列
- 全局维护一个教程展示队列，队列排序依次使用：教程优先级从高到低、教程阶段顺序从早到晚、首次有效触发时间从早到晚、稳定教程标识从小到大。
- 不得使用当前UI列表排序、随机顺序、每次读档重新洗牌、文件读取顺序或文案字母汉字排序。
- 同一时刻多个教程触发时，只有队首节点展示，其他节点保持已排队。当前节点结束后重新评估队列。已经提前完成的节点静默移除。目标已不可用的节点留在队列或退回等待，不得阻塞后续全部教程。

### 展示资格与展示形式
- 教程节点只有在以下条件全部满足时才能展示：状态为已排队、全局教程提示已启用、当前没有其他教程正在展示、前置条件仍然有效、完成条件仍未满足、目标UI或对象当前可定位、当前没有更高优先级的阻断式系统警告、当前没有存档加载场景切换或界面重建过程。
- 目标UI暂时不可见时，节点保持排队，不显示悬空箭头，不自动打开目标面板，不强制切换玩家界面，不阻塞后续不依赖该目标的教程。目标恢复可用时重新评估。
- 教程提示默认采用非模态形式：目标控件高亮、小型锚点说明、屏幕边缘提示卡、非阻断式箭头、简短操作说明。每条提示至少提供"关闭"和"不再提示"。允许根据界面空间增加"下一步""上一步"，但下一步只切换同一教程内部步骤，不代表业务完成。
- 教程提示不得：自动替玩家执行操作、自动扣除资源、自动确认建造、自动分配NPC、自动完成任务、自动切换游戏倍速、自动暂停游戏、自动关闭其他关键面板、遮挡目标控件、在玩家拖拽或确认关键操作时突然抢占输入焦点。

### 普通关闭与不再提示
- 普通关闭进入"暂时关闭"状态，不代表已完成或已跳过。
- "不再提示"进入"已跳过"终态，不再自动展示。
- 普通关闭按钮不得把节点标记为已跳过。

### 多步骤教程
- 一个教程节点可以包含多个展示步骤，所有步骤共用一个节点状态。
- 步骤索引可以在展示期间记录。关闭后再次展示时可以从尚未完成的步骤继续。
- 业务完成条件成立时整个节点立即完成。不得为每个纯文案步骤创建独立冲突编号。不得因为点击"下一步"就标记整个节点完成。
- 存档时保存当前步骤。读档后不得重复从第1步开始，除非当前步骤目标已失效且需要安全回退。
- 若步骤目标永久不存在，跳过该展示步骤，重新检查节点整体完成条件，不得让教程永久卡死。

### 教程全局开关
- 设置中允许存在"教程提示：开启/关闭"，属于玩家全局偏好。
- 关闭全局教程提示时：当前展示立即隐藏，不把节点标记为已跳过，不把节点标记为已完成，不再自动展示新教程，游戏业务正常运行，完成条件仍可在后台判定，已经实际完成的教程可以静默标记已完成。
- 重新开启后：不集中补弹所有历史触发，已完成和已跳过节点保持终态，未完成节点只有在当前触发条件有效或未来再次发生有效触发时才进入队列，不得出现一次性连续弹出大量旧教程。
- 本轮只定义开关行为，不要求修改设置界面文件。

### 存档范围
- 教程节点进度属于每个存档独立保存。每个存档至少保存：教程节点状态、当前多步骤索引、是否已经发生首次有效触发、暂时关闭后的重新触发锁定状态、当前已排队节点、当前展示节点、稳定触发顺序信息。
- 全局保存教程提示总开关。
- 不同存档之间不得共享节点完成状态。玩家可通过全局关闭教程提示避免重复学习。

### 保存与读档
- 展示中保存：保存节点状态和当前步骤。读档后先完成场景和UI恢复，再将该节点恢复为已排队。目标UI可用后重新展示，不创建第二个队列项，不重新记录首次触发时间。不得在加载画面上展示教程。
- 已排队保存：读档后恢复队列，重新验证前置条件，重新验证完成条件，已经完成的节点静默移除，目标暂不可用的节点继续等待。
- 暂时关闭保存：读档后仍保持暂时关闭。不得因为读档而立即重新展示。只有对应重新触发条件真实成立后才能再次排队。
- 已完成或已跳过保存：读档后保持终态，不得重置。

### 新旧存档初始化
- 新存档中的教程节点初始为"未解锁"。满足无前置条件的节点可以立即转为可触发，但不得在加载过程弹出。
- 旧存档没有教程状态数据时，不得把全部教程视为未完成并集中弹出。必须逐节点执行：检查正式业务完成条件，已经完成对应操作的节点标记为已完成，已经超过适用阶段且无法合理再次教学的节点标记为已完成，尚未完成且当前触发条件有效的节点进入可触发或排队，其他节点保持未解锁或可触发，不生成教程弹窗风暴。迁移完成后保存正式教程状态。

### 与任务和通知系统的边界
- 教程提示不得阻挡：存档错误、资源溢出处理、建筑即将摧毁、NPC死亡、严重规则冲突、其他A级规则定义的关键警告。
- 优先级关系：阻断式系统错误 > 必须确认的安全警告 > 当前教程提示 > 普通信息通知。
- 发展篇章更新与教程节点可以由同一业务事件触发，但必须分别去重。教程节点不拥有篇章奖励。
- 任务弹窗的具体展示频率和形式由后续C023处理，本轮不裁决。
