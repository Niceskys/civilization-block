# UI交互体验优化规范
> 基于HCI研究 + 城市建造类游戏UI设计最佳实践
> 目标：每一个操作≤2次点击，每一个信息≤1.5秒找到

---

## 一、HCI准则审查：当前UI存在的10个问题

基于Fitts's Law、Hick's Law、Jakob's Law、Miller's Law等人机交互研究，对当前UI设计进行全面审查：

| # | 问题 | 违反原则 | 后果 | 位置 |
|---|------|---------|------|------|
| 1 | 底栏8个按钮（B/S/Q/N/T/R/L/D） | ❌ Miller's Law（7±2临界）+ Hick's Law（选项过多） | 玩家找按钮需要扫读+思考，每次多花0.5~1秒 | 5.1底栏 |
| 2 | 面板缺少层级管理 | ❌ 缺少Doherty Threshold（反馈不够快） | 面板叠面板，玩家找不到"刚才打开的NPC详情" | 多个面板 |
| 3 | 资源栏默认只显示5种，其余要展开 | ❌ 违反了"信息可见性"原则 | 玩家不知道"有哪些资源" → 漏看关键数据 | 5.1顶栏 |
| 4 | 建筑面板Tab过多（全部/生产/居住/防御/功能/科技/装饰/规划） | ❌ Hick's Law（8个分类选项） | 切换标签找建筑耗时增加 | 5.2建造面板 |
| 5 | 建筑详情面板信息堆叠 | ❌ 缺少Progressive Disclosure（渐进式呈现） | 玩家刚点击建筑就被大量信息淹没 | 5.3 |
| 6 | 快捷键需要记忆（无提示） | ❌ 违反Jakob's Law（应使用通用模式） | 新手不知道按N打开NPC列表 | 5.6 |
| 7 | NPC管理面板缺少快速操作 | ❌ 违反Fitts's Law（操作目标太小） | 频繁操作需要精确点击，疲劳累积 | 5.4 |
| 8 | 没有"全局搜索" | ❌ 缺少Tesler's Law的补偿设计 | 建筑多了找不到某个特定建筑 | 全局 |
| 9 | 面板关闭只能点击"X" | ❌ 违反"容错原则" | 误关面板后需重新打开 | 所有面板 |
| 10 | 缺少视觉反馈层级 | ❌ 违反"状态可见性原则" | 点击按钮后不确定是否生效 | 全局 |

## 二、核心优化方案（7大改进）

### 优化1：底栏重构 — 从8个按钮→5个核心按钮（符合Miller's Law）

#### 旧底栏（8按钮 + 无分类）
```
[B建造] [S商店] [Q任务] [N人物] [T科技] [R贸易] [L日志] [D数据]
```

#### 新底栏（5按钮 + 双状态折叠 + 快捷键可见）
```
┌─────────────────────────────────────────────────────────────┐
│ [🏗️建造 B] [🏛️城市 C] [📋任务 Q] [🗺️地图 M] [⚙️设置 Esc]   │
│                                       [⏸️▶▶▶] [☀️ D:45]    │
└─────────────────────────────────────────────────────────────┘
```

| 按钮 | 快捷键 | 展开后包含 |
|------|--------|----------|
| 🏗️ **建造** B | B | 建筑列表（分类收起在子菜单中） |
| 🏛️ **城市** C | C | 商店+NPC列表+科技树+贸易+数据仪表盘+挑战（5合1） |
| 📋 **任务** Q | Q | 任务列表+事件日志+成就 |
| 🗺️ **地图** M | M | 世界全景+生物群系+特殊地皮查看 |
| ⚙️ **设置** Esc | Esc | 设置+存档+退出+名人堂+蓝图管理 |

**设计意图**：
- 5个按钮 < 7±2，玩家一目了然
- 每个按钮打开一个**聚合面板**（而非一个独立面板），减少面板数量
- 快捷键大写字母显示在按钮上，新手无需记忆

---

### 优化2：面板聚合管理 — "城市"按钮 = 5合1枢纽面板

#### 聚合面板结构
```
┌────────────────────────────────────────────────────┐
│ 🏛️ 城市                              [×] [📌] [—] │
├────────────────────────────────────────────────────┤
│ [商店] [人物] [科技] [贸易] [数据] [挑战]           │ ← 子Tab一行
├────────────────────────────────────────────────────┤
│ 【当前Tab内容区域】                                 │
│                                                    │
│  统一滚动条/统一风格/统一关闭机制                     │
│                                                    │
└────────────────────────────────────────────────────┘
```

**设计意图**：
- 玩家打开"城市"后，所有城市管理功能集中在一个大面板里
- 子Tab切换 = 0.1秒，不需要关闭再打开另一个面板
- 📌固定按钮 = 面板保持打开（即使点击空白处也不关闭）
- — 最小化按钮 = 面板收起到底栏（不关闭）

### 优化3：信息层级 — 渐进式呈现（Progressive Disclosure）

#### 建筑详情面板：3级深度

| 级别 | 触发 | 内容 | 交互目标大小 |
|------|------|------|------------|
| **Lv1-悬浮** | 鼠标悬停建筑1秒 | 建筑名+等级+当前状态图标+进度条 | 不占用点击 |
| **Lv2-点击** | 左键单击建筑 | 简洁面板：名称/耐久条/产出/工人/3个操作按钮 | **按钮≥44px**（Fitts's Law最低标准） |
| **Lv3-详情** | 点击Lv2的"▶详情" | 全部信息：属性/历史/社交/升级 | 滚动条查看 |

#### NPC详情面板：2级深度

| 级别 | 触发 | 内容 |
|------|------|------|
| **Lv1-悬浮** | 鼠标悬停NPC 0.5秒 | 名字+职业+效率数值+头顶进度环颜色 |
| **Lv2-点击** | 左键单击NPC | 详情面板（属性+社交+操作按钮） |

### 优化4：全局快捷键+动态提示

#### 快捷键显示规则
- [x] 所有可交互按钮**必须显示快捷键字母**（用方括号包裹，例：[B]建造）
- [x] 快捷键使用**单大写字母**（避免Shift+、Ctrl+等组合键）
- [x] 快捷键冲突时，优先度高者保留单键，次者使用Alt+键

#### 完整快捷键表
| 快捷键 | 功能 | 优先级 |
|--------|------|--------|
| B | 建造面板 | ⭐核心 |
| C | 城市面板（5合1） | ⭐核心 |
| Q | 任务面板 | ⭐核心 |
| M | 地图全景 | ⭐核心 |
| Esc | 设置/关闭面板 | ⭐核心 |
| Space | 暂停/继续 | ⭐核心 |
| 1 | 时间1倍速 | ⭐核心 |
| 2 | 时间2倍速 | ⭐核心 |
| F12 | 截图模式 | 次要 |
| WASD | 视角平移 | 核心 |
| 滚轮 | 缩放 | 核心 |
| 鼠标右键拖拽 | 视角旋转 | 核心 |
| 鼠标左键 | 选中/确认 | 核心 |
| 鼠标右键 | 取消/返回 | 核心 |
| Tab | 切换当前面板子标签 | 次要 |
| Delete | 拆除选中建筑 | 危险操作（需确认） |

#### 快捷键提示条
- [x] 玩家新手期（前10游戏日），底栏上方显示浮动快捷键提示条：
  ```
  [B]建造  [C]城市  [Q]任务  [M]地图  滚轮缩放  右键旋转
  ```
- [x] 10日后自动淡出
- [x] 玩家随时可按 `F1` 重新打开快捷键参考表

### 优化5：全局搜索 — 当城市变大时快速定位

#### 触发
- [x] 快捷键 `Ctrl+F` 或 双击鼠标中键
- [x] 在任意界面下可用（不关闭当前面板）

#### 搜索框
```

│ 🔍 搜索建筑/NPC...          结果:5条│
├─────────────────────────────────────┤
│ 🏠 房屋 Lv2 → 地皮B2·第3层        │
```

- [x] 搜索范围：建筑名、NPC姓名、职业名称、地皮编号

### 优化6：面板操作三件套 — 所有面板统一的交互模式

#### 统一面板控件
| 控件 | 位置 | 功能 | 操作面积 |
|------|------|------|---------|
| 📌 **固定** | 面板右上角第1个 | 点击后面板不随空白处点击关闭 | ≥40×40px |
| — **最小化** | 面板右上角第2个 | 面板收起为底栏标签 | ≥40×40px |
| × **关闭** | 面板右上角第3个 | 关闭面板 | ≥40×40px |
| 面板标题栏 | 面板顶部 | 鼠标拖拽移动面板 | 全宽 |

#### 面板行为规则
- [x] 所有面板必须支持：拖拽移动、最小化、固定、关闭
- [x] 面板不互相遮挡标题栏（新面板在旧面板下方偏移20px打开）
- [x] 固定后的面板**半透明化**（不遮挡游戏画面）
- [x] 最小化后的面板缩小为底栏标签（点击恢复）
- [x] 打开面板数量上限：5个（超过时最早打开的面板自动最小化）
- [x] **面板背景**：采用高斯模糊效果（非纯色背景），让玩家仍能看到背后游戏画面
  - 模糊强度：15px（足够辨别背后内容，但不干扰阅读文字）

### 优化7：视觉反馈 — 每一次操作都有"被感知"的反馈

| 操作 | 视觉反馈 | 听觉反馈 | 触觉反馈（手柄） |
|------|---------|---------|----------------|
| 点击按钮 | 0.95倍缩放（100ms） | 轻微"滴"声 | 微震动 |
| 悬停按钮 | 1.05倍放大+颜色微变 | — | — |
| 拖拽面板 | 半透明跟随+阴影 | — | — |
| 无效操作（资源不足） | 红色闪烁+按钮抖动 | 错误提示音 | 双震 |
| 成功操作（建造确认） | 绿色闪光 | 确认音效 | 单震 |
| 信息更新（产出变化） | 数值飘动动画（+数字上浮/下浮） | — | — |
| 警告（资源不足） | 顶栏资源图标变为橙色/红色+脉动 | 柔和警告音 | — |
| BOSS出现 | 全屏红色边框闪烁3次 | 警报+低音轰鸣 | 持续震动 |
| NPC死亡 | 屏幕左上角弹出小头像+灰色淡出 | 悲伤旋律短音 | — |

#### 数值变化可视化
- [x] **产出数字实时飘动**：当建筑产出资源时，该资源图标旁边显示 **"+2↑"**（绿色上浮）或 **"-1↓"**（红色下浮）
- [x] **对比箭头**：每日结算面板中所有数据旁显示与昨日的对比（↑/↓/—）
- [x] **进度环**：建筑建造/升级/维修时，建筑周围显示半透明进度环

## 三、UI易用性检查清单

### 视觉舒适性

| 标准 | 当前状态 | 优化方案 |
|------|---------|---------|
| 文字大小 | 12px~14px | **最小16px**（1080p基准，低于16px阅读疲劳） |
| 按钮大小 | 32×32px | **最小44×44px**（Fitts's Law推荐至少44px触摸目标） |
| 颜色对比度 | 暗色面板+灰色文字 | **至少4.5:1对比度**（WCAG AA标准） |
| 行高 | 1.2em | **1.5em**（增加可读性） |
| 面板圆角 | 无 | **8px圆角**（减少尖锐感） |
| 面板阴影 | 无 | **柔和阴影**（层级感） |

### 点击舒适性

| 场景 | 旧设计 | 新设计 |
|------|--------|--------|
| NPC列表中点击NPC | 整行可点击 | 整行可点击+高亮反馈 |
| 资源购买数量调整 | 加减按钮（小） | 滑块+预设按钮（×5/×10/全部） |
| 建筑建造确认 | 面板底部按钮 | 鼠标左键直接放置（虚影模式下） |
| 面板关闭 | 仅×按钮 | ×/Esc/点击面板外/右键 四种方式 |

### 展开舒适性

| 场景 | 优化 |
|------|------|
| 面板打开速度 | 所有面板打开动画 ≤100ms（Doherty Threshold上限400ms） |
| 面板切换 | 子Tab切换无延迟（预加载下一个Tab内容） |
| 滚动条 | 所有可滚动区域使用**圆角滚动条**+**触底加载**（无分页器） |
| 列表加载 | 超过50条数据时使用**虚拟列表**（只渲染可见部分） |

### 贴合游戏风格

- [x] UI整体风格：**嵌入游戏的半透明悬浮式**，不是独立窗口
- [x] 面板背景：使用**高斯模糊**效果（不是纯色），让玩家始终看到背后的城市
- [x] 字体：使用**像素风格字体**（如Press Start 2P, 04b03）但不小于16px
- [x] 图标：所有资源/建筑使用**方块风格的像素图标**，与3D体素风格一致
- [x] 分隔线：使用**发光晶脉线条**（淡蓝色细线），呼应世界观
- [x] 色彩主调：深色背景+金色强调+蓝色信息，与"文明块/晶脉"世界观一致

## 四、UI设计原则总结

```
┌─────────────────────────────────────────────────────┐
│                  UI设计总原则                          │
├─────────────────────────────────────────────────────┤
│                                                     │
│  🎯 1. 每一个操作 ≤ 2次点击                           │
│      例外：删除/转职等不可逆操作 → 必须二次确认        │
│                                                     │
│  🎯 2. 每一个信息 ≤ 1.5秒找到                        │
│      超过1.5秒 → 需要全局搜索或快捷键                  │
│                                                     │
│  🎯 3. 所有按钮 ≥ 44×44px（Fitts's Law）             │
│      越小 → 点击越疲劳 → 玩家越烦躁                    │
│                                                     │
│  🎯 4. 面板数量 ≤ 5（同时打开上限）                   │
│      超过5个 → 最早打开的面板自动最小化                 │
│                                                     │
│  🎯 5. 快捷键 ≤ 5个核心（B/C/Q/M/Esc）               │
│      超过5个 → 使用Alt+组合键（二级）                  │
│                                                     │
│  🎯 6. 反馈延迟 ≤ 100ms（Doherty Threshold）         │
│      超过100ms → 显示加载动画/进度条                   │
│                                                     │
│  🎯 7. 文字大小 ≥ 16px（1080p基准）                   │
│      小于16px → 玩家前倾阅读 → 颈椎疲劳 → 放弃游戏    │
│                                                     │
│  🎯 8. 面板永远半透明（高斯模糊背景）                   │
│      不遮挡游戏画面 → 玩家感觉"UI在游戏里，不是上面"    │
│                                                     │
└─────────────────────────────────────────────────────┘
```
