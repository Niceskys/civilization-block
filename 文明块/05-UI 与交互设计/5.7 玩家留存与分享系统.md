## 三、截图模式（F12）

### 入口
- [x] 快捷键 F12 / 底栏 📷 按钮
- [x] 在任何游戏状态下可进入（包括建造/战斗/暂停时）

### 模式功能
| 功能 | 操作 | 说明 |
|------|------|------|
| 隐藏/显示UI | 按 F12 进入/退出 | 进入时所有UI淡出消失 |
| 自由视角 | WASD平移 | 无边界限制，可移动到任何位置 |
| 缩放 | 滚轮 | 与游戏内缩放一致（2~100格） |
| 旋转 | 鼠标拖拽 | 360°自由旋转 |
| 滤镜 | 按 F1~F4 切换 | F1=标准 / F2=暖色 / F3=冷色 / F4=复古棕 |
| 光照 | 按 空格 切换 | 白天→黄昏→夜晚→白天循环 |
| 网格线 | 按 C 切换 | 显示/隐藏地面网格 |
| 截图保存 | 按 Enter 截图 | 保存至 `文明块/Screenshots/` |
| 退出 | 按 ESC 或 再次F12 | 返回游戏 |

### 截图保存规范
- [x] 文件名格式：`文明块_种子码_游戏日_时间戳.png`
  - 例：`文明块_ABC123_D45_20260614_230000.png`
- [x] 分辨率：当前游戏分辨率（最大3840×2160支持）
- [x] 截图包含：城市全景+当前光照+当前滤镜
- [x] 截图不包含：鼠标指针、UI元素、调试信息
- [x] 截图保存成功后弹出提示"📷 截图已保存！"

## 四、建筑蓝图分享系统

### 导出蓝图
- [x] **选择区域**：按住Ctrl+鼠标拖拽框选需要导出的建筑区域
- [x] **导出操作**：框选后弹出菜单 → "导出为蓝图" → 命名 → 确认
- [x] **蓝图文件格式**：`.civblueprint`（纯文本JSON格式）
- [x] **蓝图内容**（仅保存以下信息）：
  - 建筑类型+位置坐标（相对位置，而非绝对坐标）
  - 建筑等级+配色方案
  - 装饰品类型+位置+配色
  - **不包含**：NPC分配、进度、耐久、库存等运行时数据
- [x] **保存位置**：`文明块/Blueprints/` 文件夹
- [x] **文件大小**：约5~50KB（取决于建筑数量）

### 导入蓝图
- [x] **导入操作**：建造面板 → "📐 规划"Tab → "导入蓝图"按钮 → 选择`.civblueprint`文件
- [x] **导入适配**：
  - 如果目标地皮大小≥蓝图所需 → 正常导入
  - 如果目标地皮大小<蓝图所需 → 提示"该地皮空间不足，需要至少X×Y格"
  - 蓝图不包含地表生物群系信息（导入后适应当前生物群系视觉风格）
- [x] **确认导入**：显示蓝图预览（半透明覆盖在当前地皮上）→ 确认后消耗对应建造资源 → 自动开始施工

### 蓝图分享激励
- [x] 每导出一个蓝图（首次）→ 获得 **50星辉** 奖励
- [x] 每导入一个蓝图（首次）→ 获得 **30星辉** 奖励
- [x] 蓝图文件可以分享给其他玩家（复制`.civblueprint`文件发送）
- [x] 未来可扩展：创意工坊支持（EA阶段后）

## 五、存档纪念册

### 自动记录内容
- [x] 每个存档自动记录以下事件，按游戏日排序：
  - **NPC里程碑**：出生、成年、学习职业、转职、恋爱、死亡（含死因）
  - **建筑里程碑**：首次建造每种建筑、首次堆叠至新层数、首次升级Lv3/Lv4
  - **战斗里程碑**：首次遭遇怪物、首次击败每种BOSS、首次建筑被毁
  - **探索里程碑**：首次地下挖掘、首次到达第3/5/7/10层、首次发现特殊地皮
  - **经济里程碑**：首次星辉达到100/500/1000/5000、首次资源某项满仓
  - **社交里程碑**：首次NPC恋爱、首次NPC生日、首次聚餐事件

### 纪念册查看
- [x] 入口：主菜单 → "加载存档" → 选中存档 → "📖 城市纪事"按钮
- [x] 进入后显示时间线（类似历史时间线视图）
- [x] 每个事件显示：游戏日+图标+简短描述
- [x] 可筛选事件类型（全部/NPC/建筑/战斗/探索/经济/社交）
- [x] 可跳转到指定游戏日（跳转到该日存档状态，如果有自动存档）

### 存档删除保护
- [x] 删除存档时，弹出二次确认面板：
  ```
  ⚠️ 确定要删除这个城市吗？
  
  🏙️ 存在时间: 87 游戏日
  👥 诞生过: 23 名NPC
  ⚔️ 击败过: 7 次BOSS
  🏆 最高堆叠: 12 层
  💎 地下最深: 10 层
  
  [取消]  [查看城市纪事]  [确认删除]
  ```
- [x] "查看城市纪事"按钮让玩家在删除前可以回顾→很可能取消删除

## 六、最强NPC名人堂

### 评选规则
- [x] 每个存档结束时（弃档/通关/全灭），自动评选该存档中**一生总产出最高**的NPC
- [x] 评选标准：总产出 = 该NPC一生产出的所有资源数量之和（按统一价值折算为星辉）
- [x] 该NPC自动进入**跨存档名人堂**
- [x] 名人堂数据存储在 `文明块/HallOfFame.dat`

### 名人堂条目内容
| 字段 | 示例 |
|------|------|
| NPC全名 | 张三 |
| 职业 | 农民 🧑‍🌾 |
| 性格 | 勤奋 🟢 |
| 传奇特质（如有） | 🌟天选之人（全属性+20） |
| 总产出 | 1,247 星辉等价物 |
| 存活天数 | 28 游戏日 |
| 所属存档名 | 我的第一座城 |
| 种子码 | ABC123 |
| 评价 | "一位勤劳的农民，一生种出了1120份食物" |

### 名人堂排行
- [x] 名人堂显示Top20（按总产出排序）
- [x] 第1名显示金色🏆图标，第2-3名银色🥈，第4-10名铜色🥉
- [x] 玩家可随时在主菜单点击"🏛️ 名人堂"按钮查看
- [x] 当新NPC进入且排名前20时，弹出通知"🏛️ XXX进入了名人堂！当前排名第X！"

### 设计意图
- 玩家会为了"让这个NPC进名人堂"而尽量让NPC存活更久+产出更多
- 而不是"NPC死了就死了，无所谓"
- 跨存档的排名竞争 → 玩家开新档的动力："这次我要培养一个更强的NPC"
- 看到自己以前的存档名和种子码 → 怀旧感 → "那个档我玩了很久呢"
