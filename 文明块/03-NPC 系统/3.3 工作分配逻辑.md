# 工作与分配逻辑

## 权威边界

- 本文件是岗位有效性、分配、重新分配、解除岗位及状态变化顺序的唯一权威源。
- `3.5 职业系统通用规则`维护职业学习、职业数量和职业倍率，不定义分配流程。
- `3.6 职业详情清单`维护职业与建筑、属性偏好的映射，不定义分配流程。
- `5.4 NPC管理UI`维护候选列表、排序、展示和确认界面，不得自行改变岗位状态。
- 建筑详情只引用本文件中的工作循环，不重复定义通用分配规则。

### Runtime最小岗位契约

- `WorkerAssignmentSystem`是当前岗位分配、解除、原子换岗和失效清理的权威实现；UI只能提交命令并消费事件。
- 建筑定义声明非负工位数量，岗位保存NPC稳定ID、建筑稳定ID、从0开始的槽位编号和分配Tick；空槽按编号升序选择。
- NPC死亡、永久离场、昏迷，或建筑拆除、摧毁、结构失稳和结构停用时立即解除岗位并记录原因。
- 不可取消任务锁阻止新分配和换岗，但不删除已有岗位；睡眠、距离和移动仍不影响岗位持久状态。
- 当前切片尚未实现职业硬条件、属性适配、班次、到岗寻路和生产贡献；这些规则不得由UI临时模拟或写入岗位系统的占位字段。

### 工作分配方式
- [x] 工作分配方式：点击建筑→选择NPC分配
  - 玩家点击建筑打开工作槽位面板
  - 从满足硬性条件的候选NPC中选择；候选可为空闲NPC，也可为允许重新分配的在岗NPC
  - 点击候选只进入待确认状态，玩家确认后才提交分配
  - 类似RimWorld风格的工作面板交互

### 硬性分配条件

NPC、目标建筑和目标岗位必须在确认时同时满足：

- NPC存在、存活、已成年且当前可工作；
- 玩家拥有该NPC的工作分配权限；
- NPC不处于死亡、永久离场、重伤昏迷或不可取消任务锁定；
- 目标建筑存在且可用，目标岗位已解锁、未满员且工作槽位仍有效；
- NPC满足目标岗位明确规定的硬性职业或能力要求；
- 分配命令标识尚未处理，避免重复提交。

睡眠、距离较远或正在移动属于暂时无法到岗，不会永久取消分配资格；确认后的到岗时间由行为系统处理。

### 分配与重新分配事务

1. 确认时重新验证NPC、原岗位和目标岗位，不信任界面打开时的旧候选结果。
2. 空闲NPC分配成功后写入目标岗位与NPC当前岗位，并产生一次分配事件。
3. 在岗NPC重新分配时，解除原岗位与写入新岗位必须属于同一原子事务。
4. 任一步骤失败时不修改NPC原岗位，不占用目标槽位，不自动选择其他NPC。
5. 相同命令重复到达时只接受一次，不得重复占槽或重复解除岗位。
6. NPC死亡、永久离场或建筑失效时，由对应系统解除无效岗位并产生可诊断事件。

### 自动分配边界

- 系统不得自动替玩家确认首次分配、自动换岗或自动解除原岗位。
- 推荐候选、适配度排序和教程提示只提供信息，不产生岗位状态变化。
- 空闲NPC的低效杂活不是正式岗位，不占用工作槽位，也不等于自动分配。

### 工人与岗位
- [x] 工人顺序的具体作用：第1位工人100%效率，第2位及之后每人效率递减5%（如2人时：100%+95%）
- [x] NPC是否会自动更换工作：否（需玩家确认重新分配）
- [x] 空闲NPC默认行为：自动做低效杂活
  - 空闲NPC以 25% 效率随机帮助附近有人的建筑
  - 不会主动抢占已满工人槽位
  - 一旦被玩家手动分配工作，立即进入正常效率状态

### 夜班机制
- [x] 玩家可在NPC详情面板将NPC切换为"夜班"模式
  - 夜班NPC：夜晚工作，白天睡觉
  - 白班NPC（默认）：白天工作，夜晚睡觉
  - 夜班NPC白天不响应工作分配
  - 建议维持至少50%人口为白班（白天防御建筑需工人）

### 属性与工作适配
- [x] 属性与工作适配使用6.2定义的属性适配倍率公式和数值边界
  - 3.6只提供每个职业的3项偏好属性及顺序
  - 分配确认时根据当前NPC属性和目标职业计算倍率
  - 本文件不维护公式副本；Runtime实现必须读取与6.2一致的配置或常量

### NPC死亡规则
- [x] 死亡原因：
  - 自然死亡（寿命耗尽）
  - 玩家击杀
  - 怪物击杀
  - 随机事件致死（灾害）
- [x] 死亡后是否产生尸体/垃圾：是（产生 1 单位垃圾）
- [x] 死亡对其他NPC的影响：全体恐惧值 +5（任何非自然原因死亡）

### 动画播放速度与效率挂钩
- [x] **基础播放速度** = 1倍速（普通NPC，100%效率）
- [x] **高效率NPC**（>150%效率）：动画播放速度 ×1.3
- [x] **高属性适配NPC**（属性适配倍率>1.5）：动画播放速度 ×1.5
- [x] **悠闲性格**：阶段之间可播放短暂观察或伸懒腰动作，但动画不得额外降低模拟效率
- [x] **勤奋性格**：阶段切换更利落；实际个人贡献使用6.2的情境倍率，不从动画时长重复获得加成
- [x] **夜班惩罚**：夜晚工作动画速度 ×0.85（除非有太阳灯/哨塔探照灯覆盖）

### 职业效率加成规则
- [x] 职业匹配、不匹配、无职业及多职业倍率全部以6.2 NPC数值表为准
- [x] 职业加成与属性加成的计算方式：相乘
  - 岗位基础效率 = 职业匹配倍率 × 属性适配倍率
  - 性格只在6.2明确规定的岗位或情境中追加一次，不是所有工作的通用乘数

### NPC个体交互操作
- [x] **点击NPC**：显示小头像气泡 + 当前状态文字 + 当前工作阶段名
- [x] **拖拽NPC**：可直接拖拽空闲NPC到建筑上分配工作（快捷操作）
- [x] **双击NPC**：打开NPC详情面板
- [x] **右键NPC**：弹出快捷菜单（分配工作/转职/促育/切换班次/强制休息）
- [x] **工作阶段指示器**：NPC头顶进度环 (0%→100%)，当前阶段颜色与动画同步
