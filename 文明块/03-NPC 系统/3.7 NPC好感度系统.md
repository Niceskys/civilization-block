# NPC好感度系统
> 方案B：简单好感度关系 — 一起工作+好感，拥挤住房-好感，影响效率±15%

---

## 一、好感度基础

### 数值范围
- [x] 每对NPC之间存在独立的双向好感度：0 ~ 100
- [x] 默认初始好感度：30（中立友好）
- [x] 好感度显示在NPC详情面板中（头像旁 ❤️数值）

### 好感度档位效果
| 档位 | 范围 | 效果 |
|------|------|------|
| 🤝 友好 | 60-100 | 同建筑工作效率 +15%（两人在同一建筑工作时触发） |
| 😐 中立 | 30-59 | 无加成也无惩罚 |
| 😞 冷淡 | 10-29 | 同建筑工作效率 -10% |
| 😡 敌对 | 0-9 | 同建筑工作效率 -15%，有概率一方辞职 |

---

## 二、好感度增减规则

### 增加途径
| 事件 | 增加值 | 备注 |
|------|--------|------|
| 同建筑工作满1游戏日 | +2 | 每对同建筑NPC每日触发1次 |
| 同房屋居住满1游戏日 | +3 | 居住同房屋的NPC每日+3 |
| 一方被怪物攻击后被另一守卫救下 | +15 | 一次性 |
| 商店道具「社交茶会」使用 | +20（全体互加） | 消耗：3食物+3水 |

### 减少途径
| 事件 | 减少值 | 备注 |
|------|--------|------|
| 拥挤住房（房屋无空位） | -3/日 全体 | 床位不足时，每NPC每日减3（对屋内所有人） |
| 一方死亡（怪物/寿命/事故） | -20 | 对该NPC所有熟人的好感一次性暴跌 |
| 失业超过3游戏日 | -5/日 | 对未分配工作的NPC，与其熟人的好感日降5 |

---

## 三、好感度对建筑分配的影响
- [x] 玩家分配工人时，UI面板显示该NPC与目标建筑中已有工人的好感度平均值
  - 显示为颜色标记：绿色(友好) / 黄色(中立) / 红色(敌对)
- [x] 好感度高(≥60)的NPC优先自动配对至同建筑（自动工作分配逻辑）
- [x] 敌对关系(≤9)的NPC不会被自动分配到同建筑

---

## 四、社交UI
- [x] NPC详情面板底部新增"社交圈"区域：
  - 显示该NPC的Top5好友（按好感度排序）
  - 每个好友显示：头像 + 姓名 + 好感度数值 + 颜色标记
  - 点击好友头像→跳转该好友的NPC详情面板
- [ ] NPC关系图谱全视图：👉 EA期间补充（显示所有NPC关系网络）

---

## 五、与恐惧值联动
- [x] NPC好感度最高的朋友死亡时：额外触发全体恐惧值 +5
- [x] NPC好感度最高的朋友被玩家击杀：该NPC好感度清零 + 全体恐惧值额外 +10

---

## 六、边界规则
- [x] 城镇只有1个NPC时：好感度系统不生效（无配对对象）
- [x] NPC上限50人时：好感度存储矩阵 50×50 = 2500条，性能可接受
- [x] NPC死亡后：其所有好感度数据清空（不保留历史）
