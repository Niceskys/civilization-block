# NPC属性与生命周期

> 数值边界：基础寿命、每日消耗、属性范围和概率属于6.2维护的受控数值。本文件负责生命周期行为、状态和延长方式；出现数值冲突时以6.2为准。

### 基础属性列表（共7项）
- [x] **力量**：影响体力劳动产出（矿场、伐木等），影响承重相关工作
- [x] **敏捷**：影响工作速度和移动效率
- [x] **智力**：影响技术型工作产出（熔炉、研究等），影响职业学习速度
- [x] **体力**：影响工作耐力，高体力NPC可持续工作更久才需要休息
- [x] **创造力**：影响特殊产出概率（稀有资源、高品质产品暴击）
- [x] **社交**：影响生育成功率、恐惧值恢复速度
- [x] **幸运**：轻微影响各方面产出，触发随机暴击几率

### 属性数值规则
- [x] 属性范围：0 ~ 100
- [x] 属性平均值：50
- [x] 正常波动范围：50 ± 20%（即 40~60 占主体约68%）
- [x] 极强属性(>85)出现概率：5%，极值范围：86~100
- [x] 极弱属性(<15)出现概率：5%，极值范围：0~14
- [x] 属性各档位对工作效率的影响（叠加计算）：
  - 属性 71~85：该属性相关产出 +10%
  - 属性 86~95：该属性相关产出 +20%
  - 属性 96~100：该属性相关产出 +30%
  - 属性 15~29：该属性相关产出 -10%
  - 属性 0~14：该属性相关产出 -20%

### 属性获取与遗传
- [x] 属性获取方式：遗传 + 随机变异
  - 子代属性 =（父母属性平均值）+ 随机变异波动
  - 变异概率：60%（每项属性独立判定）
  - 变异幅度：±5~15（均匀分布），有10%概率出现大幅变异±20~30
  - 极端情况下可能出现远超/远低于父母的天赋异禀/缺陷个体

### 婴儿阶段规则
- [x] 是否占用房屋床位：否（婴儿跟随父母同住，不额外占床位）
- [x] 是否可以工作：否
- [x] 是否消耗资源：是
  - 每日食物消耗：0.25份/游戏日（食物消耗相对通用NPC基准减半）
  - 每日水消耗：0.5单位/游戏日（水消耗仍采用通用NPC每日0.5单位基准）

### 成长规则
- [x] 婴儿长大时长：2.5 游戏日
- [x] 成长计时起点：出生时刻
- [x] 成年后是否自动加入劳动力池：是
- [x] 成年时是否自动分配/选择职业：否（需玩家手动分配职业）

- [x] 性格是情境专长和行为表达，不是NPC品质评级；不得把性格作为全局效率优劣排序
- [x] 性格稳定标识依次为`core:personality:diligent`、`relaxed`、`optimistic`、`vigilant`、`brave`、`cautious`，显示名称可本地化，存档不得只保存显示文字

| 性格 | 主要倾向 | 行为表达 |

### 居住分配与自动入住

#### 床位占用原则
- 每名成年NPC最多占用1张床位。
- 每张床位最多分配1名成年NPC。
- 婴儿不占用床位，跟随父母同住。
- 聚落中已存在的NPC即使没有床位也可以暂时存在，但处于"无家可归"或"等待住房"状态；自动入住系统只在有效床位出现时建立正式居住分配。其他NPC加入聚落的前置条件仍以各自获取规则为准。
- 开局时初始NPC无床位，第一座房屋建成后自动填充。

#### 自动入住候选条件
同时满足以下条件的NPC进入自动入住候选列表：
- NPC已存在于当前聚落；
- NPC存活；
- NPC已成年；
- NPC当前没有有效床位；
- NPC没有被分配到其他有效房屋；
- NPC不处于婴儿阶段。

#### NPC固定排序
自动入住使用不可变化的"NPC创建顺序"确定优先级：
- NPC创建序号从小到大。
- 该顺序在NPC创建时确定，存档后保持不变，读取存档后保持不变。
- NPC改名、年龄变化、职业变化、属性变化均不影响该顺序。
- NPC管理界面的UI排序方式不影响此顺序。
- 严禁使用剩余寿命、年龄、性别、属性高低、职业、工作效率、好感度或物理距离作为入住优先级依据。

#### 空床位固定排序
当同时存在多个空床位时，按以下顺序确定床位：
1. 房屋创建序号从小到大。
2. 同一房屋内床位槽位序号从小到大。
3. 房屋升级新增的床位使用该房屋内后续床位槽位序号。
- 不得随机选择房屋或床位。

#### 自动入住触发条件
以下事件发生后执行一次自动入住结算：
- 房屋完成施工并正式提供床位；
- 房屋升级完成并增加床位；
- 已占用床位因NPC死亡而释放；
- 已占用床位因NPC被玩家迁出而释放；
- 某个已存在聚落中的成年NPC失去原住房后，其他房屋仍有有效空床；
- 读取存档后检测到"空床位"和"无床位成年NPC"同时存在；
- 新的成年NPC进入聚落，且聚落已有有效空床。

#### 自动匹配过程
每次自动入住结算时：
1. 收集全部有效空床位；
2. 收集全部合资格无床位成年NPC；
3. 分别按床位顺序和NPC创建顺序排序；
4. 第一名NPC分配至第一张空床，第二名NPC分配至第二张空床；
5. 依次匹配，直到空床位耗尽或无候选NPC。
该过程必须是确定性的，相同存档状态下结果必须完全一致。
- 分配结果生成后，应先写入NPC住房归属和床位占用状态，再开始NPC移动表现。

#### 床位归属与NPC移动表现

- 自动入住结算完成时，床位归属立即确定。
- 床位归属确定后，对应NPC再通过正常寻路和行为流程前往房屋。
- NPC尚未实际走到房屋，不影响床位已被占用的逻辑状态。
- NPC暂时无法寻路时，不取消已经确定的床位归属。
- 暂时无法寻路的床位不得再次分配给其他NPC。
- NPC寻路、移动动画、进入房屋和睡眠表现属于行为表现层，不改变自动入住的确定性排序结果。
- 本规则不提供传送、强制瞬移或卡路脱困机制。
- 长时间寻路失败、永久不可达和传送处理仍属于后续边界规则，本轮不作裁决。

#### 房屋失效与重新分配
- 房屋被拆除、摧毁或因其他既有规则失去居住功能时，该房屋内所有入住NPC立即失去该房屋床位。
- 这些NPC进入待重新分配状态。
- 系统检查其他有效房屋的空床：
  - 有空床时，重新按照全部合资格无床位NPC的创建顺序分配。
  - 无空床时，未分配NPC进入"无家可归"状态，无家可归的恐惧值影响沿用房屋.md既有规则。

#### 手动分配优先
- 玩家仍可通过房屋详情中的床位操作手动调整NPC住房。
- 玩家手动完成的有效分配优先保留。
- 自动入住不得主动驱逐已有住户。
- 自动入住不得为优化距离、属性、年龄或工作地点而重新洗牌。
- 自动结算只填补当前空床位。
- 手动调整后释放出的床位，可在该次操作完成后用于下一次自动入住结算。

#### 与工作和生育系统的边界
- 入住房屋不会自动分配职业或工作建筑。
- NPC已有工作不影响入住优先级。
- NPC换房不会取消已有工作。
- 自动入住规则不得写入3.3工作分配逻辑。
- 当空床位出现时，先完成现有无床位成年NPC的自动入住结算，再由生育系统检查是否仍存在满足其规则的空闲床位。

### 寿命与死亡
- [x] 是否有老年/自然死亡机制：是
  - [x] 有寿命且可通过后期科技/物品延长
  - [x] 基础平均寿命：30 游戏日（30日×20分钟=10小时现实时间）
  - [x] 寿命浮动范围：±10 游戏日（即 20~40 游戏日）
  - [x] **体质属性影响寿命**：每10点体质 +2 游戏日寿命（体质100→+20日，体质0→-0日）
  - [x] **疗愈所效果延长寿命**：寿命消耗 -30%（详见疗愈所单页）
  - [x] **社会科技Lv1**：基础寿命 +5 游戏日
  - [x] 延长寿命的方式汇总：
    - 商店道具「NPC寿命延长」：+10游戏日（300星辉）
    - 疗愈所建筑：NPC寿命消耗速度 -30%
    - 社会科技Lv1：基础寿命+5日
    - 体质属性：每10体+2日
    - 传奇特质「长生」：基础寿命+20日
    - 未来科技树解锁高级延寿科技

## Runtime住房权威规则（已实现）

- 只有存活、成年且未永久离开的NPC占用床位；婴儿不占床，昏迷不会失去已有住房。
- 房屋Lv1提供2个床位，每提升1级增加1个床位。
- 自动入住按NPC创建序、稳定ID排序；房屋按施工序、稳定ID排序；床位始终从最小编号开始。
- 玩家指定的有效房屋和床位会被保留，自动系统只填补空位，不覆盖有效手动选择。
- 房屋被拆除、摧毁、失稳、停用或床位失效时，居民立即释放并按同一固定顺序重新安置。
- 无可用床位的合格成年NPC进入权威无家可归集合；该状态进入存档、StateHash、诊断和服务器同步。
- 城市视角只读取汇总数据；建筑剖面中的NPC、库存和物流动画属于表现层，不参与住房裁决。

## Runtime基础生存需求（已实现）

- 生存需求按游戏日集中结算：成年人每日食物0.5、水0.5；婴儿每日食物0.25、水0.5。
- 小数需求使用四分之一单位整数余数跨日结转，不使用浮点数，不因Tick切分方式产生漂移。
- 只消费全局库存中的未锁定资源；已被生产或运输锁定的食物和水不能被生存系统抢用，建筑局部库存也不会被隔空抽取。
- 每次结算保存需求量、实际消费量、缺口量、余数和连续短缺天数，并产生服务器权威事件。
- 暂停不推进Tick，因此不会结算；倍速和大步长Tick按跨过的每个游戏日边界逐日结算。
- 旧存档从读档时刻之后的下一个完整日开始，不追溯补扣历史消耗。
- 当前短缺只形成权威状态与警报依据，不直接造成死亡、减速或属性惩罚；这些效果须在后续生命周期裁决后接入。
- 寻路、取餐和进食动画属于建筑剖面表现层，不参与资源扣除裁决。

## Runtime年龄、成长与自然死亡（已实现）

- 婴儿阶段精确持续2.5游戏日；到达边界后转为成年，并立即参与成年住房自动分配。
- 成年寿命在20至40游戏日内，由世界种子与NPC稳定ID确定性生成；同一存档、服务器和重连结果一致，不受集合遍历顺序影响。
- 生命周期保存当前阶段已过Tick、成年寿命、成年转换Tick和死亡Tick；暂停不衰老，倍速按实际推进Tick加速。
- 单次大步长可以连续跨过成长与自然死亡，事件按真实发生Tick排序；拆分推进与一次推进的年龄、死亡和生存消耗结果一致。
- 自然死亡保留NPC历史记录并设置非存活状态，同时立即释放岗位、住房和无家可归状态。
- 旧存档中的NPC从读档时刻以当前阶段年龄0开始，不追溯衰老，也不会加载后立刻死亡。
- 当前寿命尚未应用体质、疗愈所、社会科技、商店延寿或传奇特质修正；这些效果必须在相应权威字段落地后统一接入。
- 葬礼、墓碑、生平面板和死亡动画属于后续表现与内容层，不影响死亡裁决。
