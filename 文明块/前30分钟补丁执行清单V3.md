## 二、唯一时间记法

### 两列并行体系

| 列名 | 格式 | 示例 | 说明 |
|------|------|------|------|
| 累计游戏时间 | T+MM:SS | T+05:00 | 从放置第一座建筑开始累计的游戏分钟:秒。暂停时不增加 |
| 游戏日时间 | Dx HH:MM | D1 05:00 | 每个游戏日20分钟, D1 00:00 = T+00:00 |

### 换算关系

| 累计游戏时间 | 游戏日时间 | 说明 |
|-------------|-----------|------|
| T+00:00 | D1 00:00 | 放置第一座建筑，世界启动 |
| T+05:00 | D1 05:00 | 白天前半段 |
| T+09:00 | D1 09:00 | 黄昏预告 |
| T+10:00 | D1 10:00 | 正式入夜 |
| T+20:00 | D1 20:00 = D2 00:00 | 日出，第二日开始 |
| T+25:00 | D2 05:00 | 第二日白天 |

### 预计现实时间

| 倍速 | 关系 | 说明 |
|------|------|------|
| 1倍速 | 现实时间 = 累计游戏时间 | 无暂停下，T+10:00 = 现实10分钟 |
| 2倍速 | 现实时间 = 累计游戏时间 ÷ 2 | T+10:00 = 现实5分钟 |
| 暂停 | 累计游戏时间不增加，现实时间继续 | 暂停5分钟后再恢复→T+10:00对应现实15分钟 |

暂停玩家不得设置固定现实时间触发事件。所有游戏内事件绑定累计游戏时间（T+XX:XX），暂停时不触发。

| 问题 | 原文档答案 | 依据 |

| 夜晚施工是否继续 | 【原文档未明确定义】 | - |
| NPC夜间休息是否会停止施工 | NPC默认"晚上进入休息/睡眠状态" | 1.3 |

### 新增规则

【新增规则】并发施工规则
1. 可以同时存在多个在建施工项目
2. 每个施工项目需要至少1名独立NPC作为施工工人
3. 同一NPC不能同时参与多个施工项目
4. 没有施工工人的施工项目进度暂停
5. 玩家可以手动将任意空闲NPC重新分配为施工工人
6. 施工进度不会因停工而清空或减少
7. 日班NPC在夜间停止施工（默认行为，服从1.3昼夜规则）
8. 夜班NPC可以在夜间施工
9. 施工不依赖自然光，只依赖NPC班次

### 对流程的影响

- 玩家放置房屋(T+00:00)后，可立即用另一名空闲NPC放置农田
- 房屋和农田**并发施工**
- 房屋：T+00:00至T+05:00（5分钟）
- 农田：T+00:00至T+08:00（8分钟，享受首建加速）
- 两座建筑独立推进，互不影响

## 四、第一次真实产出验算

### 推荐路线的精确计算

**条件**：
- 房屋：T+00:00放置，T+05:00完成（加速）
- 农田：T+00:00放置，T+08:00完成（加速，并发施工）
- 分配工人：T+08:00即完成时分配（理想状况）
- 农田白天产出，夜间因光照暂停
- NPC为日班（默认）

**进度计算**：

| 时间区间 | 累计游戏时间 | 状态 | 累计进度(食物) |
|---------|-------------|------|--------------|
| 农田施工中 | T+00:00至T+08:00 | 施工中，无产出 | 0 |
| 分配工人→入夜 | T+08:00至T+10:00 | 白天生产，2分钟×0.2/分钟 | 0.4 |
| 第一夜 | T+10:00至T+20:00 | 夜间，光照不足→暂停 | 冻结在0.4 |
| 日出→产出 | T+20:00至T+23:00 | D2白天，3分钟×0.2/分钟 | 0.4+0.6=1.0 |

**第一次产出时刻**：T+23:00（D2 03:00）

**验算**：4食物/日 ÷ 20分钟 × 累计有效生产时间(13分钟白天) = 2.6食物 → 产出2单位，第二次产出约在T+28:00（D2 08:00）

### 验收标准

【推荐路线下，第一次真实资源产出不晚于T+25:00。】

## 五、完整连续生产规则

### 1. 通用公式

【新增规则】所有持续生产建筑为每种产出维护一个建筑级累计进度值（浮点数）。

```
每游戏分钟进度 = 该产出的每日基础产量 ÷ 20 × 所有有效工人效率叠加
```

**工人效率叠加规则**：
- 所有有效工人的效率值累加后作为乘数
- 第1位工人100%，第2位+95%，第3位+90%...（沿用3.3顺序递减规则）
- 进度达到≥1.0时触发一次产出，进度扣除1.0，余数保留

### 2. 各建筑适用表

| 建筑 | 产出1 | 产出1日产量 | 产出2 | 产出2日产量 | 输入消耗 | 适用进度 |
|------|-------|-----------|-------|-----------|---------|---------|
| 农田 | 食物 | 4/工人 | - | - | 0.5水/日(灌溉) | ✅ 单一产出 |
| 水井 | 水 | 4/工人 | - | - | 无 | ✅ 单一产出 |
| 树场 | 木材 | 3/工人 | - | - | 无 | ✅ 单一产出 |
| 采掘场 | 矿石 | 2/工人 | 石头 | 1/工人 | 无 | ✅ **双产出** |
| 熔炉房 | 金属 | 1→1/工人 | - | - | 1矿石+1燃料/单位金属 | ✅ 单一产出+输入 |
| 研究站 | 科技点 | 1/工人 | - | - | 0.5食+0.5水/工人/日 | ✅ 单一产出+输入 |

### 3. 多产物建筑处理

【新增规则】采用**统一配方进度**方案：
- 每种产物的完整配方绑定为一个整体进度
- 当进度达到1.0时，同时产出配方中的所有资源
- 示例(采掘场)：进度累计(2矿石+1石头)/日÷20 = 0.15/分钟。进度达1.0时同时产出2矿石+1石头
- 产出一组的间隔：约6.67分钟
- 每日总产出：与6.1一致（矿石8+石头4）

**选择理由**：进度条简单（一个建筑一根进度条），UI干净，与建筑内NPC工作动画匹配。

### 4. 输入资源消耗规则

【新增规则】
1. 输入资源在**产出触发时刻**一次性扣除
2. 产出触发时(进度≥1.0)，检查所需输入资源是否充足
3. 输入不足 → 该次产出不触发，进度冻结在1.0位置
4. 输入恢复后，立即触发产出（无需重新累计）
5. 已累计的进度不清零，不销毁
6. 扣料顺序：优先从建筑本地缓存扣，不足时从全局仓库扣
7. 严禁先产出后欠账

### 5. 缓存满处理

【新增规则】
1. 建筑本地缓存满时，产出触发但不生成资源→进度冻结在即将产出的位置(≥1.0但不触发)
2. 不销毁资源，不继续累计（防止无限隐藏产出）
3. 腾出缓存空间后，立即触发被抑制的产出

### 6. 进度保留规则

| 事件 | 进度处理 |
|------|---------|
| NPC加入 | 建筑进度保留，新工人的效率在加入时点开始计入 |
| NPC离开 | 建筑进度保留，离开工人的效率从离开时点停止 |
| 更换NPC | 进度保留不变 |
| 暂停(玩家) | 冻结，恢复后继续 |
| 停产(条件不足) | 冻结，恢复后继续（输入不足时冻结在≥1.0位置） |
| 拆除建筑 | 未完成的小数进度清除 |

## 六、夜间生产与夜班兼容规则

### 逐建筑核查

| 建筑 | 夜间是否受环境限制 | 依据 | 有夜班工人时 | Lv3自动运转时 |
|------|------------------|------|------------|-------------|
| **农田** | ✅ **是**——依赖阳光，夜晚产出暂停 | 1.3+农田详情 | 需太阳灯覆盖才可产出。无太阳灯时即使有夜班工人也不产出 | 50%效率，但仍需太阳灯解除环境限制 |
| **水井** | ❌ **否**——无环境限制 | 1.3"持续运行建筑可继续运作" | ✅ 夜班水工可正常生产 | ✅ 50%效率自动运转 |
| **树场** | ❌ **否**——无环境限制 | 同上 | ✅ 夜班樵夫可正常生产 | ✅ 50%效率自动运转 |
| **采掘场** | ❌ **否**——无环境限制 | 同上 | ✅ 夜班矿工可正常生产 | ✅ 50%效率自动运转 |
| **熔炉房** | ❌ **否**——无环境限制 | 同上 | ✅ 夜班铁匠可正常生产 | ✅ 50%效率自动运转 |
| **研究站** | ❌ **否**——无环境限制 | 同上 | ✅ 夜班研究者可正常生产 | ✅ 50%效率自动运转 |

### 关键结论

- 只有**农田、农舍**因阳光要求在夜间停产（除非太阳灯覆盖）
- **水井、树场、采掘场、熔炉房、研究站**——只要有人值守即可继续运行
- 日班NPC夜间默认不工作（睡觉），所以这些建筑在没有夜班工人时夜间自然停产
- 第一夜新手无怪物，玩家可以安心探索夜间机制
- 玩家可以在第一夜将某个NPC设为夜班，观察夜班生产

### 对时间轴的影响修正

在推荐路线中：
- 农田T+08:00完工、分配工人
- 农田在T+08:00至T+10:00（2分钟）白天生产：累计0.4进度 → ✅
- 水井/树场如果建造，夜间有夜班工人时可持续产出

### 证据等级说明

- 【原文明确】：直接可从原文档引用的规则
- 【可推导】：由多条原文规则组合可得的结论
- 【设计推断】：原文未明确定义，基于游戏设计一致性的推断
- 【本次新增限制】：原文没有，本次设计新增的规则

### 堆叠预览判定原则

【原规则明确化】堆叠预览的判定改为：
- 不是按建筑种类硬编码
- 而是对每个具体位置执行**完整的合法性验证**（承重+光照+通行+尺寸+接地）
- 仅通过全部验证的位置才显示淡蓝色可建造网格
- 当前阶段，房屋顶部通过全部验证，农田水井树场顶部不通过

**这样做的好处**：
- 如果后续设计确认水井/树场可以堆叠，修改规则后网格自动显示
- 不需要修改V3中的硬编码列表
- 与正式堆叠规则一致

| 2人无住所是否会产生惩罚 | 【原文档未明确定义无住所的具体惩罚】 | - |
| 恐惧值上升条件 | 3.4只定义"玩家击杀+15"等，未写"无住所" | 3.4 |
| 房屋详情"无家可归状态" | "房屋被拆除/损坏导致床位不足时，多余NPC进入'无家可归'状态（恐惧值持续上升）" | 房屋详情 |

### 结论

【原文明确】房屋详情明确写了：因拆除/损坏导致床位不足时，NPC进入"无家可归"状态，恐惧值上升。但初始床位不足（NPC从未入住过）是否触发该效果，**原文档未明确定义**。

### 新增规则

【新增规则】初始无住所处理：
1. 房屋建成且自动入住后，剩余的NPC标记为"无住所"
2. **初始阶段（D1 D2两日）**：无住所NPC不触发恐惧值上升——给玩家宽裕时间建第二座房屋
3. **从D3 00:00开始**：如果NPC仍无住所，按照正式规则触发恐惧值(每日+X)
4. 该延迟惩罚的规则独立于自动入住逻辑

**独立PATCH**：不混入自动入住PATCH，而是单独一个PATCH处理。
