# 任务UI设计

### 入口与布局
- [x] 任务入口位置：主界面快捷键Q / 顶部功能栏按钮
- [x] 界面整体布局：
  - 左侧分类标签 + 中间任务列表 + 右侧任务详情与奖励预览
- [x] 分类标签：主线 / 日常 / 周常 / 里程碑 / 随机事件 / 已完成

### 任务追踪
- [x] **无常驻追踪UI** — 保持画面简洁
  - 仅在任务面板（Q键）中查看所有任务

### 任务列表显示
- [x] 单条任务显示内容：任务名称、进度条、奖励图标、状态标记
  - 状态：🔵进行中 / 🟢可领取 / ⚪未接取 / ✅已完成
- [x] 进行中任务是否显示实时进度数值：是
- [x] 已完成任务是否保留显示：是，保留至当前游戏日结束

### 交互规则
- [x] 点击任务可展开详情：完整目标描述、奖励明细、倒计时（限时任务）
- [x] 可领取任务高亮显示，点击「领取奖励」按钮直接到账
- [x] 限时任务显示剩余时间，倒计时结束自动标记失败
- [x] 日常/周常任务：点击"接取"按钮激活，每日手动接取

### 主线任务更新展示分级

#### 普通进度原位更新
- 主线任务普通进度变化（如收集数量从2/10变为3/10、建筑施工进度变化、NPC工作次数增加、连续生产计数增加、非最终子目标更新）只更新任务追踪栏和任务面板中的数值。
- 不得弹出任务窗口、显示居中弹窗、暂停游戏、抢夺键盘或鼠标焦点、每次计数变化都显示提示卡、播放大型任务完成演出。
- 允许使用数字平滑刷新、轻微高亮、非阻断式进度动画、任务追踪栏中的短暂强调。同一逻辑批次内的多次进度变化必须合并刷新。

#### 新主线任务或新阶段通知
- 新主线任务或主线阶段首次激活时，显示一次非模态任务通知卡。
- 通知卡至少包含：标题"主线任务已更新"、任务或阶段名称、简短目标摘要、"查看任务"按钮、"关闭"按钮。
- 不暂停游戏、不打开全屏任务面板、不自动切换玩家当前界面、不抢夺当前输入焦点、不遮挡建造确认/拖拽/工作分配等关键操作。同一任务阶段只自动提示一次。点击"查看任务"才打开任务面板。点击"关闭"只关闭通知，不改变任务状态。通知关闭后仍在任务日志和未读标记中保留。重复上报同一阶段事件不得重复弹出。

#### 阶段完成通知
- 阶段完成时使用比普通更新更明显的非模态完成卡，但仍不得自动暂停。
- 完成卡至少包含：标题"主线阶段完成"、完成的阶段名称、获得或待领取内容摘要、"查看详情"按钮、"关闭"按钮。
- 不得自动领取奖励、自动进入下一任务详情页、自动打开全屏面板、自动跳转镜头、自动关闭玩家正在操作的面板。任务奖励和领取规则继续服从现有任务系统。

#### 不可逆选择的确认窗口
- 只有任务确实需要玩家做出不可逆选择时，才允许使用确认窗口。但不得自动弹出选择窗口。
- 正确流程：先显示非模态任务通知→在任务追踪栏和任务面板标记"需要选择"→玩家主动点击"做出选择"→此时才打开确认窗口。确认窗口只处理该选择，不把普通任务更新改成阻断式弹窗。

#### 任务通知去重
- 每个任务通知事件必须具有稳定事件键，由任务标识、阶段标识、事件类型、阶段转换序号或唯一业务事件组合而成。不得依赖通知文案去重。
- 同一事件重复上报时：只保留一条通知；已展示过则不得自动重播；已关闭则保持已读或已关闭状态；读档不得再次弹出；任务面板仍保留正确任务状态。

#### 任务通知队列
- 同一时刻最多展示一张主线任务通知卡。多个任务通知同时产生时，按以下顺序：1.需要玩家选择；2.主线阶段完成；3.新主线或新阶段；4.其他任务信息。同级按首次有效事件时间排序。
- 任务通知不得覆盖正在展示的教程提示。优先级关系：阻断式系统错误>必须确认的安全警告>教程提示>主线任务通知>普通信息通知。教程提示结束后，任务通知再继续展示。

#### 存档与读档
- 存档至少保留：哪些任务阶段通知已经展示、哪些通知仍未读、哪些通知已关闭、当前排队的合法任务通知、需要玩家选择但尚未处理的任务状态。
- 读档后：不重播已经展示过的普通阶段通知；未读内容通过任务按钮角标和任务日志保留；不在加载画面弹出任务通知；不连续补弹旧任务历史；需要选择的任务保持等待，但必须由玩家主动打开。

#### 旧存档初始化
- 旧存档缺少通知状态时，根据当前任务业务状态初始化。已完成的旧阶段不得逐条补弹。只保留当前有效主线阶段。当前阶段可以在任务按钮上显示一次未读标记。不自动弹出全部历史主线通知。

### 主线任务追踪教程节点
- 教程标识：tutorial.mainquest.tracker
- 教程名称：主线任务追踪说明
- 前置条件：存在首个有效主线任务或主线阶段
- 触发条件：玩家首次获得主线任务，并且任务追踪栏已经可用
- 完成条件：玩家主动打开过一次任务面板
- 展示目标：任务追踪栏或任务按钮
- 展示方式：非模态锚点提示
- 普通关闭后的重触发：玩家关闭并再次打开主界面，且从未打开任务面板时
- 允许跳过：是
- 不得：自动打开任务面板、自动完成主线任务、自动领取奖励、因主线进度每次变化而重复触发
