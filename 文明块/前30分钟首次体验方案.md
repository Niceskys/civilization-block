# 前30分钟首次体验方案
> 阶段四：首次体验与目标引导详细方案
> 基于原始设计文档 + 问题诊断 + 机制裁决
> 本阶段只处理玩家第一次进入游戏后的前30分钟

---

## 第一部分：当前版本前30分钟流程还原

> 以下严格依据原始文档，不补充文档中不存在的初始资源或教程步骤。
> 【原文档未定义】标记表示文档中未明确描述的内容。

### 时间单位说明
- 1游戏日 = 20现实分钟（1.3时间系统规则）
- 前30分钟 ≈ 前1.5游戏日

---

### 当前流程还原表

#### 0—1分钟：游戏启动与创建

| 项目 | 内容 |
|------|------|
| 玩家拥有的资源（按普通难度） | 初始成年NPC 4名、食物30、水20、木材50、石头25、星辉100（6.3难度选项） |
| 初始建筑 | 无已建造建筑。初始解锁可建造：房屋、农田、水井、树场（2.1建筑通用规则） |
| 初始地皮 | 4块地皮（B2/B3/C2/C3），坐标固定（1.0世界生成） |
| 系统要求玩家做什么 | 进入游戏后无强制要求。无自动弹出的指引。【原文档未定义前30秒自动行为】 |
| 玩家如何知道下一步 | 主线任务1"安营扎寨"自动触发：建造1座房屋+1座农田，奖励50星辉（10.2任务清单） |
| 能体验到的核心特色 | 3D方块世界视野、斜45°俯角、资源栏显示核心资源 |
| 可能流失原因 | 看到空荡荡的4块地皮+4个NPC，不知道要做什么 |

#### 1—3分钟：第一个操作

| 项目 | 内容 |
|------|------|
| 玩家拥有的资源 | 同上。4名NPC全部为"空闲"状态（无工作分配） |
| 系统要求玩家做什么 | 无自动要求。主线任务1在任务面板中（Q键打开），无常驻追踪UI（10.3任务UI设计） |
| 玩家如何知道下一步 | 主线任务1触发时"弹出一次通知提示"（10.3），但通知内容未定义持续时间。玩家需要按Q打开任务面板查看详情 |
| 能体验到的核心特色 | 可自由旋转视角、鼠标右键平移、滚轮缩放 |
| 可能流失原因 | 不知道按B打开建造面板、不知道按Q查看任务、面对空地图不知从何开始 |

#### 3—5分钟：首次建造

| 项目 | 内容 |
|------|------|
| 玩家拥有的资源 | 假设玩家按B打开了建造面板，看到分类Tab（全部/生产/居住/防御/功能/科技/装饰/规划）（5.2建造系统UI） |
| 系统要求玩家做什么 | 选择建筑→确认建造。建造预览显示绿色/红色虚影 |
| 玩家如何知道下一步 | 建造面板中点击建筑卡片→底部显示建造成本/时长/所需工人（5.6 UI原型） |
| 能体验到的核心特色 | 首次放置建筑的3D虚影、建造时半透明进度条环绕 |
| 可能流失原因 | 8个分类Tab让玩家困惑（5.8已识别此问题）、不知道需要工人参与施工 |

#### 5—10分钟：等待建造+分配工作

| 项目 | 内容 |
|------|------|
| 玩家拥有的资源 | 假设建造了1座房屋（0.5游戏日=10分钟）+ 1座农田（1游戏日=20分钟） |
| 系统要求玩家做什么 | 建造完成后：点击建筑→分配NPC到建筑工作槽位（3.3工作分配逻辑） |
| 玩家如何知道下一步 | 【原文档未定义建筑完成后的自动引导】。任务面板中任务1进度可能显示"0/1房屋 0/1农田" |
| 能体验到的核心特色 | 建筑建造动画、NPC自动走到建筑开始工作（如果被分配） |
| 可能流失原因 | 房屋10分钟建造、农田20分钟建造——前10分钟玩家大部分时间在等待。NPC虽然自动做低效杂活（25%效率），但玩家看不到明显效果 |

#### 10—15分钟：首次NPC交互

| 项目 | 内容 |
|------|------|
| 玩家拥有的资源 | 假设房屋已建成（5分钟时完工）+ 农田仍在建造中 |
| 系统要求玩家做什么 | 分配NPC入住房屋（点击房屋→分配NPC） |
| 玩家如何知道下一步 | 【原文档未定义】NPC入住是自动的还是手动的。3.1只写"NPC分配至房屋后自动占用1床位"，未写如何分配 |
| 能体验到的核心特色 | NPC名称、性格标签可见、NPC在房屋中出现 |
| 可能流失原因 | NPC详情面板信息过多（属性柱形图7项、职业、社交、轨迹等），新玩家被信息淹没（5.3已有此问题） |

#### 15—20分钟：首次资源产出+昼夜变化

| 项目 | 内容 |
|------|------|
| 玩家拥有的资源 | 假设农田已建成（如果玩家在第5分钟建造，此时应已完工）。第1个游戏日结束 |
| 系统要求玩家做什么 | 夜晚到来。如果有怪物（30%概率/夜），需面对 |
| 玩家如何知道下一步 | 日/夜指示器显示黄昏→夜晚（1.3时间系统） |
| 能体验到的核心特色 | 昼夜交替视觉变化（10秒过渡）、建筑窗格暖光、夜晚氛围 |
| 可能流失原因 | 如果农田还没建完（玩家如果第8分钟才建造），第一个夜晚时可能没有任何食物产出。4名NPC消耗2食物/日，初始30食物可支撑15天，短期内不会饿死，但玩家不知道这个数据 |

#### 20—30分钟：第二游戏日+首次管理决策

| 项目 | 内容 |
|------|------|
| 玩家拥有的资源 | 应该有1座房屋+1座农田+一些资源消耗。主线任务1完成后获得50星辉 |
| 系统要求玩家做什么 | 任务1完成后→任务2"生计之源"自动触发：建造1口水井+人口达到4人（10.2） |
| 玩家如何知道下一步 | 任务2触发通知（如果第一个任务被完成） |
| 能体验到的核心特色 | 生产链雏形（农田产食物、水井产水） |
| 可能流失原因 | 任务2需要"人口达到4人"——但初始已有4名NPC，所以只需要再建1口水井。但如果玩家还没分配NPC入住房屋，NPC可能未全部入住。"人口"的定义在【原文档中未明确是"已入住NPC"还是"总NPC数量"】。玩家可能因为概念不清而困惑 |

### 当前流程核心问题总结

| 问题 | 严重程度 | 原因 |
|------|---------|------|
| 前5分钟不知道按什么按钮 | P0 | 无自动引导，快捷键提示条需要前10游戏日才能自动显示（5.8） |
| 建筑等待时间过长（农田20分钟） | P0 | 20分钟等待后才有第一个食物产出 |
| 主线任务1弹窗后无后续指引 | P1 | 弹窗消失后玩家需要自己记住按Q查看 |
| NPC管理和建造管理是分开的面板 | P1 | 建好建筑后不知道要去NPC面板分配工人 |
| 任务面板无常驻追踪 | P1 | 10.3明确写"无常驻追踪UI"，玩家需要主动打开任务面板 |
| 8个分类Tab的建造面板复杂 | P1 | 5.2原始设计问题，5.8已提出优化 |
| 30分钟内没有堆叠相关内容 | P2 | 第一个堆叠相关任务（任务5）需要"拥有3座以上建筑"，前30分钟可能达不到 |

## 第二部分：三个不同的首次体验方向

### 方案A：建设驱动型

#### 1. 前30分钟完整体验概述
玩家进入游戏后的核心驱动力是"让城市变大的可见变化"。前30分钟的设计围绕"每3分钟城市外观就有一次明显变化"展开。

- 0-2分钟：进入游戏，视角自动从远拉近（展示4块地皮的全貌→聚焦到中心地块）
- 2-5分钟：建造面板默认打开、默认选中"生产"分类Tab、高亮推荐第一个建筑（房屋）。推荐建筑不是任务，不弹窗，只在建造面板中有一个微弱的"推荐"标签
- 5-10分钟：房屋以加速速度完成（首次建造速度为正常的2倍）。房屋完成后自动弹出一个NPC入住动画（NPC走进房屋、窗口亮灯）
- 10-15分钟：推荐第二个建筑（农田），建造过程中玩家可以旋转视角观察农田的生长区/播种区/收获区的3D模型预览
- 15-20分钟：农田完成后，自动产出第一批食物（少量启动产出一即建成时自带2食物）
- 20-25分钟：推荐堆叠——在房屋上方放置第二层建筑。展示半透明虚影预览堆叠效果
- 25-30分钟：完成2层堆叠，城市出现"多层"轮廓。结束时显示"你的城市已经有了2层建筑"的成就记录

#### 2. 最早展示独特性时间点
第5分钟：首次看到3D建筑建造动画+建筑内部结构预览

#### 3. 玩家第一次有意义选择
第8分钟：选择在房屋上面建什么（水井还是树场还是第二座房屋）

#### 4. 第一次看到NPC生命感
第7分钟：NPC走进房屋，窗口亮灯

#### 5. 第一次接触堆叠
第22分钟：在房屋上方放置第二层建筑

#### 6. 第一次感受到压力
第28分钟：第一次夜晚降临，看到夜间城市灯光效果（无怪物入侵的情况下）

#### 7. 30分钟结束时留下的未完成目标
- 农田只有1座，想再建1座扩大食物生产
- 2层堆叠，想试试能不能建到第3层
- 4名NPC只分配了部分工作，还有空闲NPC需要安排

#### 8. 优点
- 城市视觉变化快：每3-5分钟可见新建筑出现
- 堆叠特色在30分钟内展示
- 建造操作学习曲线平缓（先学放建筑，再学堆叠）
- 与其他沙盒建造游戏体验一致，上手门槛低

#### 9. 风险
- NPC存在感弱：玩家注意力在建筑上，NPC变成了背景
- 城市变大但缺乏"为什么"的动机
- 前30分钟可能感觉像"放置游戏"而非"管理游戏"
- 若玩家对纯建造不感兴趣，可能觉得"只是搭积木"

#### 10. 开发成本
中。需要：
- 首次建造加速（临时数值调整）
- 建造面板默认推荐标签（UI逻辑调整）
- 首次堆叠引导提示（已有堆叠系统，只需增加推荐标记）
- NPC入住窗口亮灯动画（已有建筑3D视觉规范中的"窗口透出暖光"）

#### 11. 对原文档改动范围
- 5.2建造系统UI：增加"推荐"标签和默认Tab逻辑
- 2.1建筑通用规则：增加"首次建造时间减半"的新手保护
- 2.2堆叠交互规则：在建造面板中增加堆叠预览的推荐标记
- 10.2任务清单：任务1拆分为子步骤提示（非任务形式）

### 方案B：NPC驱动型

#### 1. 前30分钟完整体验概述
玩家进入游戏后的核心驱动力是"认识并管理这群NPC"。前30分钟围绕"认识每个NPC→给他们分配合适的工作→看到他们工作的成果"展开。

- 0-2分钟：进入游戏，视角聚焦在4名NPC身上。他们各自站在4块地皮中心，头顶显示名称+性格标签
- 2-5分钟：系统通过左键点击一个NPC→弹出简约信息卡片（只显示名字、性格、一个属性推荐值，不显示全部7维属性）。提示"点击建筑→选择NPC→分配工作"
- 5-10分钟：建造1座房屋，指定某个NPC入住。房屋建成时该NPC自动走向房屋，窗口亮灯，NPC名字旁出现"🏠已入住"标记
- 10-15分钟：建造1座农田，系统推荐"力量最高"或"体力最高"的NPC去当农民。若不按推荐分配，系统不阻止
- 15-20分钟：被分配到农田的NPC在工作时，玩家可以双击进入建筑内部视图，看到NPC的播种→浇水→等待→收割→搬运完整动画循环
- 20-25分钟：第一个游戏日结束前，通过一条社交消息（"张三和李四在农田边聊了一下午"）展示NPC社交系统
- 25-28分钟：夜晚降临，NPC返回房屋睡觉，窗口透出暖光。玩家可以点击房屋看到NPC"躺床闭眼(zzZ气泡)"的动画
- 28-30分钟：第二游戏日开始时，NPC自动起床走出房屋，开始新一天的工作。玩家看到他们的日常轨迹

#### 2. 最早展示独特性时间点
第2分钟：看到4个有名有姓、有性格标签的NPC站在城市中

#### 3. 玩家第一次有意义选择
第6分钟：选择把哪个NPC分配到哪座建筑

#### 4. 第一次看到NPC生命感
第2分钟：NPC不是静止的——他们在原地有轻微空闲动画（站立/环视/走动）

#### 5. 第一次接触堆叠
30分钟内不展示堆叠（堆叠被推迟到第30-40分钟）

#### 6. 第一次感受到压力
第25分钟：如果某座建筑没有分配工人，系统通过"NPC站在建筑外无所事事"的视觉呈现（非UI警告），让玩家注意到"有人没活干"

#### 7. 30分钟结束时留下的未完成目标
- 还有NPC没有分配工作
- 想探索建筑内部看NPC工作动画
- 想知道明天下班后NPC会聊什么

#### 8. 优点
- NPC存在感极强：玩家从第1分钟就开始认识NPC
- 情感连接建立快：有名字、有性格、有行为的NPC
- 建筑内部视图的丰富动画在30分钟内就展示给玩家
- 社交系统的首次体验印象深刻

#### 9. 风险
- 建造速度慢：建造退居次要位置
- 堆叠在30分钟内完全未展示——核心特色被延迟
- 如果玩家对NPC管理不感兴趣，前30分钟会感觉无聊
- 信息密度高：需要一次性了解名字/性格/属性/工作分配等多个概念
- 初始4个NPC同时出现可能信息过载

#### 10. 开发成本
中高。需要：
- NPC初始位置和空闲动画设计（已有NPC模型，但初始聚集需要额外逻辑）
- 简约NPC信息卡片（替代当前的全属性面板，需要新的UI组件）
- NPC自动入住房屋动画（已有窗口亮灯，需要"走向房屋"的寻路动画）
- 建筑外部显示"该建筑有工人"的标记（已有红色感叹号用于停产，需要反向标记）
- 社交消息在前30分钟就要触发（当前社交事件概率默认为每日判定，需要确保首次触发）

#### 11. 对原文档改动范围
- 3.1属性与生命周期：增加"首次点击NPC时只看简约卡片"的渐进式信息呈现
- 3.3工作分配逻辑：增加"推荐NPC"的UI标记（基于属性权重）
- 3.8NPC社交事件系统：确保前1.5游戏日内至少触发1条社交消息
- 5.4NPC管理UI：增加简约卡片模式作为默认状态
- 5.3建筑详情UI：外部需要显示NPC是否在建筑内工作的状态

### 方案C：事件驱动型

#### 1. 前30分钟完整体验概述
玩家进入游戏后的核心驱动力是"准备→等待→看到结果"的微循环。前30分钟由2-3个微型事件周期构成。

**第一周期（0-12分钟）：基础建设**
- 0-2分钟：进入游戏，视角自动展示城市全貌。提示语极简："你的第一批居民需要住所和食物"
- 2-6分钟：建造1座房屋（加速，8分钟完成）。建造过程中地面出现建造进度圈
- 6-10分钟：房屋建成。推荐建造1座农田。此时系统通过HUD下方的进程条暗示"下次日出前需要食物"
- 10-12分钟：继续建造

**第二周期（12-22分钟）：第一次微型事件**
- 12-15分钟：第一个游戏日黄昏，天空开始变色。系统提示"夜晚即将来临，你的人口需要安全的住所"
- 15-17分钟：夜晚降临。无怪物入侵（新手保护）。但可以听到环境音效（风声/虫鸣），看到NPC房屋窗口亮灯
- 17-20分钟：夜晚期间，玩家可以做"查看NPC是否都已入住房屋"的检查操作。如果还有NPC没入住，房屋附近会出现一个微弱的灰色问号
- 20-22分钟：日出。游戏进入第二日。显示简单的"昨日简报"：NPC全部存活、食物储备充足

**第三周期（22-30分钟）：规模扩展**
- 22-25分钟：推荐建造第二座农田或水井，展示"同种建筑相邻有加成"的预览
- 25-28分钟：建议在上面堆叠第二层（建一个太阳灯或另一个小建筑），展示堆叠预览
- 28-30分钟：完成2层堆叠。如果玩家完成了所有推荐建造，城市从2座建筑变成了3-4座建筑，出现了"小社区"的轮廓

#### 2. 最早展示独特性时间点
第12分钟：第一次黄昏到夜晚的过渡，城市灯光亮起

#### 3. 玩家第一次有意义选择
第8分钟：建造水井还是第二座房屋还是树场

#### 4. 第一次看到NPC生命感
第15分钟：夜晚NPC回到房屋，窗口亮灯（间接展示）

#### 5. 第一次接触堆叠
第26分钟：推荐在房屋上方建第二层

#### 6. 第一次感受到压力
第12分钟：黄昏来临，系统暗示"夜晚前需要住所"（轻度压力，非惩罚性）

#### 7. 30分钟结束时留下的未完成目标
- 下一夜可能会出现怪物（第一次威胁即将到来）
- 想继续扩建城市看看能多高
- 想知道明天的简报会说什么

#### 8. 优点
- 节奏感强：有明确的小周期（准备→等待→结果）
- 昼夜系统作为天然的节奏划分器
- 微型事件让玩家有"应对感"但没有真正的惩罚
- 堆叠在第26分钟展示，不算太晚

#### 9. 风险
- 强制节奏感：如果玩家建造速度慢，可能赶不上"夜晚前建好房屋"的节奏
- "事件周期"设计可能太像教程关卡，降低沙盒自由感
- 缺乏NPC正面展示（NPC只在夜晚作为"回家的角色"出现）
- 如果玩家在第一个周期中建了太多建筑，后续节奏被打乱

#### 10. 开发成本
中。需要：
- 昼夜过渡的强化视觉（已有，但需要更明显的"即将入夜"标记）
- 微型事件触发器（基于时间+玩家状态的条件判断）
- "昨日简报"的简化版（已有每日结算系统，需要修改触发条件使其在第一个夜晚后自动出现极简版）
- 堆叠推荐标记（已有建造预览系统，增加推荐标记）

#### 11. 对原文档改动范围
- 1.3时间系统规则：增加"首次入夜"的特殊提示（非常驻）
- 5.7玩家留存与分享系统：每日结算简报在首次出现时改为极简版（只显示关键信息）
- 2.1建筑通用规则：首次建造时间减半（同方案A）
- 2.2堆叠交互规则：增加堆叠预览中的推荐标记

## 第四部分：推荐方案

### 推荐：方案A为主 + 方案B的NPC入门元素

**不采用三合一**：只从方案B提取一个元素——"首次点击NPC时展示简约信息卡片"。

#### 1. 为什么方案A最适合《文明块》

《文明块》在第三阶段裁决中确认的核心定位包括：
- 3D体素/方块式城市建造（第一）
- 建筑垂直堆叠与空间规划（第二）
- NPC居民持续生活和工作（第三）

**方案A直接展示前两个核心特色（建造+堆叠），且在30分钟内全部完成。** 方案B虽然在"NPC情感连接"上得分最高，但它牺牲了堆叠核心特色的展示时间。方案C的事件节奏与沙盒自由之间有矛盾。

#### 2. 为什么B和C不作为主方案

**方案B**：最大问题是堆叠特色在30分钟内完全未展示。如果一个玩家在首次体验中从未看到"可以把建筑堆起来"，游戏与普通平面建造游戏的区别就没有被传达。堆叠是《文明块》区别于同类游戏的最明显的视觉和机制特色。延迟展示意味着可能错失"这就是我要玩的游戏"的关键时刻。

**方案C**：事件周期作为首次体验的结构是可接受的，但它为"节奏"牺牲了"自由"。前30分钟是玩家建立"这是我的城市"所有权感的关键期。如果玩家感觉自己在按系统设定的步调走（即使系统只是推荐），他们可能不会产生"A城市是我的"的归属感。

#### 3. 最早展示核心卖点的时间

第5分钟：第一个建筑（房屋）的3D建造动画+完工时的建筑完整外观。此时玩家已经看到"我可以在3D方块世界中放建筑"。

#### 4. 如何避免变成强制教程

所有"推荐"都是视觉标记而非任务：
- 推荐建筑：建造面板中建筑卡片左上角显示微弱的"推荐"标签（半透明，不是红点）
- 玩家可以点击任何其他建筑，推荐标签不会被强化或闪烁
- 推荐标签在玩家完成推荐操作后自动消失
- 玩家连续3次忽略推荐，推荐标签自动淡出不再出现

#### 5. 如何避免变成任务清单游戏

- 无"任务面板"概念在前30分钟。主线任务1"安营扎寨"作为后台成就记录，不强制追踪
- 唯一可见的"目标指示"是建造面板的推荐标记——且同一时间最多1个
- 玩家建造了推荐以外的建筑，推荐自动更新"好像不需要这个了"并消失
- 不设置"完成主线任务1才能解锁主线任务2"的硬性链条
- 任务2"生计之源"作为里程碑在任务面板中，但不自动弹出

#### 6. 如何自然进入第30分钟后的沙盒

第30分钟结束时：
- 玩家建了2-3座建筑，其中1座在另一座上面（堆叠已展示）
- 知道如何操作建造面板（学会了B键和选择分类）
- 见过NPC走进建筑工作（看到了基础的NPC行为）
- 经历了第一个黄昏和夜晚（体验了昼夜系统）
- 没有"教学关卡结束"的提示——游戏继续运行，推荐标记逐渐减少，最终在10分钟后完全消失

## 第六部分：引导系统设计（不是任务系统）

### 三种提示类型的定义

#### 1. 教程提示（Teach Tip）

| 属性 | 内容 |
|------|------|
| 触发条件 | 特定操作首次可执行时（如首次可建造时→教按B） |
| UI位置 | 底栏快捷键提示条区域内（5.8已设计"快捷键提示条"） |
| 显示条件 | 仅在玩家尚未执行该操作时显示 |
| 隐藏条件 | 玩家执行该操作后永久消失 |
| 是否弹窗 | 否——集成在提示条中，不遮挡游戏画面 |
| 是否可关闭 | 可——提示条右上角有"×" |
| 是否影响自由建造 | 否——提示条在底栏上方，不阻挡3D视图 |
| 是否发放奖励 | 否——教程提示只是教操作，不给奖 |
| 是否在任务面板中出现 | 否 |
| 同时最多数量 | 1条（新提示出现前旧提示必须已消失或被执行） |
| 超时处理 | 连续3次被忽略（玩家做了其他操作）→该提示不再出现 |

**教程提示清单（仅前30分钟）：**
| 提示内容 | 触发条件 | 预期出现时间 |
|----------|---------|-------------|
| "[B] 建造" | 首次进入游戏且建造面板未打开 | 0-1分钟 |
| "左键放置建筑" | 首次进入建造模式 | 2-3分钟 |
| "点击建筑→分配工人" | 首次建筑完成 | 5-6分钟 |
| "滚轮缩放 右键旋转" | 首次进入游戏 | 0-1分钟（集成在提示条中） |

#### 2. 推荐行动（Suggestion）

| 属性 | 内容 |
|------|------|
| 触发条件 | 基于当前城市状态：缺少某类建筑时推荐 |
| UI位置 | 建造面板中对应建筑卡片的左上角（半透明"推荐"标签） |
| 显示条件 | 玩家当前没有该类型建筑或数量不足 |
| 隐藏条件 | 玩家建造了该建筑→标签消失；或玩家连续3次选择其他建筑→标签淡出 |
| 是否弹窗 | 否——仅在建造面板内 |
| 是否可关闭 | 可——玩家可忽略，标签在3次忽略后自动消失 |
| 是否影响自由建造 | 否——玩家可自由选择任何建筑 |
| 是否发放奖励 | 否——推荐不是任务，不给奖励 |
| 是否在任务面板中出现 | 否 |
| 同时最多数量 | 1个（建完推荐建筑后，系统根据新状态可能生成下一个推荐） |
| 超时处理 | 推荐标签在10分钟无操作后自动消失（玩家可以按B重新打开看到） |

**推荐行动清单（仅前30分钟）：**
| 推荐内容 | 触发条件 | 预期出现时间 |
|----------|---------|-------------|
| 建造房屋 | 玩家尚无房屋 | 1-2分钟 |
| 建造农田 | 玩家已有房屋但尚无农田 | 8-9分钟 |
| 分配工人到农田 | 农田建成但尚无工人 | 20-21分钟 |
| 建造水井 | 玩家已有房屋+农田但尚无水源 | 27-28分钟 |

#### 3. 里程碑记录（Milestone Record）

| 属性 | 内容 |
|------|------|
| 触发条件 | 城市自然发展达到特定状态 |
| UI位置 | HUD底部左侧（非弹窗），以文字通知形式淡入淡出（持续3秒后自动消失） |
| 显示条件 | 达成时自动显示 |
| 隐藏条件 | 3秒后自动消失。可在事件日志中永久查看 |
| 是否弹窗 | 否——文字通知不打断游戏 |
| 是否可关闭 | 不需关闭——自动消失 |
| 是否影响自由建造 | 否 |
| 是否发放奖励 | 是——达到里程碑时发放对应奖励（如首次建造房屋→50星辉） |
| 是否在任务面板中出现 | 是——里程碑记录在任务面板的"里程碑"分类中 |

**前30分钟内可能触发的里程碑：**
| 里程碑 | 触发条件 | 奖励 | 反馈等级 |
|--------|---------|------|---------|
| 建造第一座建筑 | 首次完成任何建筑 | 无（仅记录） | 轻反馈（通知文字） |
| 人口首次入住 | 首次分配NPC到房屋 | 无 | 轻反馈 |
| 首次分配工作 | 首次分配NPC到生产建筑 | 无 | 轻反馈 |
| 度过第一夜 | 第一个游戏日结束 | 无 | 轻反馈（日出自然过渡即为反馈） |
| 首份资源产出 | 首次有资源进入仓库 | 无 | 轻反馈（数值飘动） |

**里程碑与现有任务系统的关系：**
- 主线任务1-9保留为任务面板中的"引导序列"，但前30分钟内**不自动弹出任务通知**
- 玩家按下Q键仍可查看任务列表
- 主线任务的奖励照常发放（任务1完成→50星辉）
- 里程碑记录在任务面板中但不是任务形式——不追踪进度条，只有"已完成"标记

## 第十部分：需要项目负责人最终决定的问题

### 问题1：新手保护期的持续时间

| 选项 | 持续时间 | 保护内容 |
|------|---------|---------|
| A（推荐） | **前3游戏日**（60分钟现实时间） | 拆除全额返还、重新分配无冷却、怪物概率0% |
| B | 前1游戏日（20分钟） | 同上 |
| C | 前5游戏日（100分钟） | 同上 |

**差异**：A覆盖前60分钟，足够玩家完成2-3次建造循环。B仅覆盖20分钟，可能不够新手适应。C的100分钟过长，玩家可能形成依赖。

**对玩家体验的影响**：
- A：保护期内玩家可以自由试错，建立信心；60分钟后自然进入正式规则
- B：新手可能在第1天的40分钟里面临怪物和资源管理双重压力
- C：玩家可能过度依赖保护，100分钟后感到"突然变难"

**对开发成本的影响**：无差异。保护期只是在配置文件中设置一个参数。

**我的推荐**：**A（前3游戏日/60分钟）**。覆盖首次完整昼夜循环+第二次昼夜循环的部分，给足试错空间。

**推荐理由**：大多数新玩家在60分钟内会完成2-4次建造循环，足够他们理解核心玩法。3游戏日后恢复正式规则，不会形成依赖。

### 问题2：推荐标记的消失条件

| 选项 | 消失条件 |
|------|---------|
| A | 玩家执行推荐操作后消失 |
| B（推荐） | 玩家执行推荐操作后消失；或玩家连续3次选择其他建筑，推荐标签自动淡出且不再出现 |
| C | 玩家执行推荐操作后消失；或30分钟时间到后所有推荐标签统一消失 |

**差异**：B允许玩家通过主动选择其他方向来"拒绝"推荐，C是硬性时间截止。

**对玩家体验的影响**：
- A：如果玩家始终不跟随推荐，推荐标签永久显示，变成"红点"式骚扰
- B：给予玩家"拒绝推荐"的控制权，同时防止推荐变成永久UI
- C：30分钟后所有推荐消失，但此时如果玩家仍需要引导则缺乏支持

**对开发成本的影响**：A最低（简单的条件判断），B稍高（需要记录忽略次数），C最低。

**我的推荐**：**B**。尊重玩家自由的同时，防止推荐变成永久UI元素。

**推荐理由**：沙盒游戏的核心是自由。推荐应该被玩家"拒绝"而非"忍受"。如果玩家连续3次选择其他建筑，说明他们有自己明确的目标，系统应该退让。

### 问题3：首次建造加速的范围

| 选项 | 加速范围 |
|------|---------|
| A（推荐） | 仅第一座建筑（任何类型）建造时间减半 |
| B | 前3座建筑建造时间减半 |
| C | 不加速，保持原建造时长 |

**差异**：A仅加速第一座（让玩家尽快看到第一个完成建筑），B加速前3座（快速建立2-3座建筑的城市轮廓），C无加速。

**对玩家体验的影响**：
- A：第一座建筑（如房屋）5分钟完成，后续建筑（农田）保持20分钟。玩家在等待第二座建筑时有"等待"的体验
- B：前3座建筑都快速完成，城市在15分钟内就有3座建筑，但等待的概念被弱化
- C：第一座建筑原10分钟（房屋），玩家在10分钟内只有一个建筑在建，城市感觉"发展慢"

**对开发成本的影响**：无差异。都是修改一个加速参数。

**我的推荐**：**A**。仅加速第一座建筑，让玩家尽快体验"建筑完成"的正反馈，但第二座建筑开始让玩家体验真实的建造节奏。

**推荐理由**：第一座建筑加速是给玩家"第一次成功"的体验。后续建筑恢复正式节奏，让玩家理解建造需要时间。如果所有建筑都加速，玩家会误以为游戏节奏很快，后续感到落差。

### 问题4：前30分钟内是否需要展示堆叠

| 选项 | 堆叠展示方式 |
|------|------------|
| A（推荐） | 不强制展示。当玩家在地皮上建造第3座建筑时，建造面板中出现"试试在上面再建一层"的推荐标签 |
| B | 通过一个微型事件在25分钟时自动触发（NPC走到建筑顶层向上看→触发提示"你的NPC想看看上面能不能建一层"） |
| C | 前30分钟内不展示堆叠，推迟到30-45分钟 |

**差异**：A是纯可选推荐（仅在建造面板中），B是事件触发（有NPC动画），C是完全不展示。

**对玩家体验的影响**：
- A：玩家可能看到推荐标签，也可能忽略。如果忽略了，堆叠核心特色在首次体验中就缺失了
- B：NPC动画自然地引导玩家尝试堆叠，但需要额外的动画设计
- C：堆叠在30分钟后展示，适合"前30分钟聚焦基础建设"的设计思路

**对开发成本的影响**：
- A：低——只需要在建造面板中增加一个条件判断标签
- B：中——需要NPC"向上看"的新动画
- C：无成本

**我的推荐**：**A**。推荐标签是可选的，不强制。如果玩家看到了并尝试，30分钟内就能体验堆叠；如果没看到或忽略，堆叠会在后续的沙盒阶段自然被发现。

**推荐理由**：堆叠是《文明块》的核心特色，但不应在首次体验中强制展示。A方案保留了展示的可能性，同时尊重玩家的自由节奏。

### 问题5：每日结算简报在首次出现时的形态

| 选项 | 首次简报内容 |
|------|-------------|
| A（推荐） | 极简版：仅显示"✅ 所有NPC存活"+"当前资源量"，无对比/无战斗报告。淡入浮层，背景不虚化 |
| B | 完整版：与正式版一致，显示产出总览+战斗报告+城市动态+关键预警 |
| C | 不弹出简报，第一个夜晚结束后静默过渡到第二天 |

**差异**：A展示核心信息但不干扰，B给新玩家信息过载，C零干扰但错失"结算时刻"的仪式感。

**对玩家体验的影响**：
- A：玩家知道"昨晚没人死""现在有多少资源"，信息足够但不冗余
- B：新手在第一次结算时面对大量数据（战斗报告为空、城市动态为"无事件"），3秒后自动关闭，信息基本没被吸收
- C：游戏从第一天直接进入第二天，玩家意识和不到"一天结束了"的节点感

**对开发成本的影响**：
- A：需要新增一个"极简简报"UI模板
- B：复用现有每日结算系统，零成本
- C：修改触发逻辑，零成本

**我的推荐**：**A**。极简简报提供必要的"一日回顾"节点感，但不冗余。

**推荐理由**：第一个夜晚是玩家首次经历"一天结束"。极简简报建立了"每日结算"的仪式，但内容精简到新手能吸收的程度。从第二日开始，恢复为完整版简报。

## 第十一部：分报告总结

### 设计原则确认

| 原则 | 本方案中的体现 |
|------|--------------|
| 玩家因为"想继续建城市"而留存 | 30分钟结束时城市从空地变为了2-3座建筑的小定居点 |
| 玩家因为"想看NPC之后怎样"而留存 | 已分配NPC有名字、有性格、有工作动画、有社交消息 |
| 玩家因为"想完成建设计划"而留存 | 有未分配的NPC、未建造的资源建筑、可扩展的空间 |
| 玩家因为"想尝试不同布局"而留存 | 建造自由保留，推荐不强制，拆除在新手期零成本 |
| 不使用登录奖励/限时奖励/红点 | 所有提示均为非弹窗、可忽略、无惩罚 |
| 不使用强制日常/随机抽奖/错过惩罚 | 无日常任务、无限时奖励、无错过惩罚 |
| 不使用连续弹窗 | 同一时间最多1个推荐标记，教程提示集成在提示条中 |
