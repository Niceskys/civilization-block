# 垃圾机制

### 垃圾产生
- [x] 垃圾产生来源：生产建筑运行、NPC死亡
- [x] 单位时间垃圾产生量：每座运行中的建筑产生 **1 垃圾/游戏日** + 每名NPC产生 **0.5 垃圾/游戏日**
  - 建筑闲置（无工人）不产生垃圾
  - NPC死亡时额外产生 1 垃圾
- [x] 总垃圾产生公式：运行建筑数 + NPC数×0.5 / 游戏日
- [x] 垃圾处理参考：1名清洁工可处理约 5-8 座运行建筑的垃圾产出

### 垃圾不处理的负面效果
- [x] 垃圾未处理时：每日占用全局存储空间（垃圾=存储类资源，上限500）
- [x] 垃圾堆积超过200：NPC满意度每日 -1%（全员工作效率 -1%，可累积至 -20%）
- [x] 垃圾堆积超过400：NPC满意度每日 -2%，疾病概率 +5%/日
- [x] 垃圾满500后：溢出垃圾丢弃，但负面效果持续累积
- [x] 垃圾清零后：满意度负面效果立即消除，疾病概率次日恢复

### 垃圾处理场
- [x] 处理模式：4种模式可手动切换
  1. **直接销毁**：垃圾直接清除，无产出，处理速度最快
  2. **资源回收**：垃圾转化为少量随机资源（木材 / 石头 / 金属之一）
  3. **堆肥制肥**：垃圾转化为肥料，可加速农田产出
  4. **燃料提炼**：垃圾转化为燃料，可提升熔炉房效率
- [x] 垃圾处理场单周期处理量：**6 垃圾/游戏日**（1工人基础值）
- [x] 各模式处理效率与产出数值：

| 处理模式 | 处理速度 | 每1垃圾产出 | 备注 |
|---------|---------|------------|------|
| A-销毁 | 10 垃圾/日 | 无 | 最快清除，无收益 |
| B-回收 | 6 垃圾/日 | 回收1随机资源(木材40%/石头40%/金属20%) | 资源循环 |
| C-堆肥 | 6 垃圾/日 | 产出1肥料（可使1座农田下次产出+50%）+ 每5垃圾额外产出1染料 | 农业增益+染料副产 |
| D-提炼 | 6 垃圾/日 | 产出0.3燃料 | 工业增益 |

### 存储规则
- [x] 垃圾可堆积：是
- [x] 垃圾堆积上限：500（全局仓库内）
- [x] 垃圾超出上限后：溢出自动丢弃
- [x] 垃圾存储不影响其他资源的分类上限

### 与存储告警联动
- [x] 垃圾达400时触发存储黄色告警（同全局存储80%）
- [x] 垃圾达500时触发存储红色告警（同全局存储90%）
- [x] 详见 **4.3 存储规则** § 存储告警系统

## Runtime处理模式状态（2026-06-20）

- 垃圾处理场已按单槽互斥配方实现A/B/C/D四模式；运行不消耗燃料，避免模式D形成燃料自启动矛盾。
- A以`10垃圾/1游戏日`整批销毁且明确允许无产出；B以`6垃圾/1游戏日`执行6次40%木材、40%石头、20%铁锭的确定性加权回收。
- C以`6垃圾 -> 6肥料/1游戏日`实现；染料副产等待4.1新增正式资源ID后再迁移。
- D采用最小整数批次`10垃圾 -> 3燃料/5/3游戏日`，长期严格等价于每垃圾0.3燃料、每日处理6垃圾。
- 加权结果使用世界种子、批次稳定ID与投掷序号计算，服务器拥有最终裁决；协议升级17，未新增存档状态字段。

## Runtime产生与阈值状态（2026-06-20）

- 每个游戏日集中结算：每座至少有一名有效工人的可运行建筑产生1垃圾，每名存活且未永久离开的NPC产生0.5垃圾；婴儿同样计入人口垃圾。
- NPC的0.5单位使用整数半单位余数跨日结转，不使用浮点；NPC死亡时立即额外产生1垃圾，并与死亡事件保持确定顺序。
- 垃圾同时受500分类上限和全局共享容量约束；无法存入的部分丢弃，但累计产生量、累计丢弃量和溢出事件仍保留用于诊断。
- 垃圾超过200时每日累计1%满意度惩罚，超过400时每日累计2%，上限20%；超过400另保存5%疾病概率增量。
- 垃圾清零后满意度惩罚立即清除；疾病概率增量在下一次日结算恢复。当前只保存权威效果状态，尚未接入NPC满意度、生产效率与疾病判定。
- 状态支持存档迁移、StateHash、诊断及服务器同步；存档升级2.6，协议升级18。

## Runtime负面效果接入（2026-06-21）

- NPC保存基础满意度，当前默认100%；界面和后续行为读取“基础满意度 - 垃圾累计惩罚”，因此垃圾清零不会破坏NPC原有满意度。
- 垃圾累计惩罚已同时接入批次生产与持续生产；两条生产链均使用万分比整数余数，不使用浮点，也不会因切分Tick产生长期精度损失。
- 批次完成或取消时清除该批次不足1工作Tick的余数，避免旧批次进度泄漏到下一批；持续生产则跨Tick保留余数。
- 垃圾超过400时，每个日结算对当日存活NPC执行一次5%疾病暴露判定；结果由权威随机种子、结算Tick和NPC稳定ID确定，并发布稳定事件。
- 当前疾病接口只记录暴露数、触发数和触发NPC，不擅自定义病程、效率惩罚、治疗时长或死亡率；这些数值需在疾病规则定稿后接入。
- 状态、效率余数与疾病统计支持存档、诊断、StateHash及服务器同步；存档升级2.7，协议升级19。

## Runtime肥料闭环（2026-06-21）

- 模式C产出的肥料可通过权威命令施用于有工人的可运行农田，每次消耗1份，效果结束前不可叠加。
- 施肥时锁定当前工人对应的一个完整基础日产周期，并对该覆盖量额外产出50%食物；后续换人不会改变已锁定覆盖量。
- 覆盖量与半单位奖励余数进入存档、诊断、StateHash和服务器同步；暂停、换速及Tick切分不改变总奖励。
- 存档升级2.8，协议升级20。
