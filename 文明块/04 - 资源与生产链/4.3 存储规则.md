# 资源存储规则

### 存储体系（三层结构）
- [x] **建筑本地缓存**：每个生产建筑有独立存储空间
  - 建筑产出先存入自身缓存
  - 各建筑本地存储上限：
    - 农田/水井/树场/房屋/太阳灯：20 单位
    - 采掘场/熔炉房/农舍/垃圾处理场：30 单位
- [x] **全局仓库**：所有资源共享的存储池
  - 建筑本地缓存满后，溢出资源自动进入全局仓库
  - 全局仓库基础上限：2000 容量
- [x] **仓库建筑扩容**：可建造"仓库"建筑扩展全局存储上限
  - 每个仓库增加全局存储容量：+200 容量

### 资源分类存储上限（全局仓库内细分）
| 资源类型 | 单类上限 | 备注 |
|---------|---------|------|
| 食物 | 1000 | 超出部分溢出处理 |
| 水 | 1000 | 超出部分溢出处理 |
| 木材 | 800 |  |
| 石头 | 800 |  |
| 矿石 | 500 |  |
| 金属 | 400 |  |
| 燃料 | 300 |  |
| 垃圾 | 500 | 单独处理（见 4.4） |
| 科技点 | 无上限 | 非存储资源 |

### 存储联动规则
- [x] 跨层建筑资源是否自动连通：否（需建造管道连接上下层间资源传输）
- [x] 资源运输是否有时间/成本消耗：是
  - 跨层资源运输（通过管道）每层有 1 秒传输延迟
  - 同一层内资源流转无延迟
- [x] **资源溢出后的处理逻辑**（超出全局上限）：
  - 玩家可选两种策略（在设置面板切换）：
    1. **自动销毁**（默认）：溢出资源直接丢弃，不做任何保留
    2. **暂停生产**：当某类资源到达上限后，产出该资源的建筑自动暂停（显示"📦 存储已满"）
  - 切换策略即时生效

### 存储告警系统
- [x] 全局存储使用率达 80%：HUD资源栏变黄色 + 闪烁提示
- [x] 全局存储使用率达 90%：HUD资源栏变红色 + 持续闪烁 + 顶部弹出"⚠️ 存储空间不足"横幅
- [x] 任一类资源达到单类上限：该类资源图标显示红色边框

### 资源消耗优先级
- [x] 建造/生产消耗资源时，优先从哪个存储池扣减：
  - 优先从全局仓库扣减
  - 全局仓库不足时，从建筑本地缓存补充

## 生产资源锁定与产物缓冲

### 输入资源锁定
批次开始时锁定输入资源，锁定资源继续占用原存储容量，不得被其他建筑、任务、商店或建造系统使用。多种输入必须全部成功锁定，否则整次启动失败。取消批次时解除锁定，资源恢复可用。不得把锁定资源同时计入可用库存、可交易库存或可建造库存。

### 锁定资源的库存统计
UI必须区分库存总量、已锁定、当前可用。锁定资源不计入可用库存。

### 锁定存储失效
如果锁定资源所在的存储建筑被摧毁、拆除或失效，先按存储失效规则处理；锁定资源被成功迁移则锁定关系随资源迁移；锁定资源依法损失时当前生产批次取消。

### 单批产物缓冲
每个生产槽拥有最多容纳一个完整批次产物的逻辑缓冲。该缓冲不是普通仓库，不能存放玩家随意放入的资源，不能被其他生产建筑直接取用。必须进入存档，不允许覆盖旧产物。

### 满仓时的产物处理
合法存储空间不足时产物保留在缓冲中，状态进入产物待转移。生产停止，连续模式保持开启但不得启动下一批。不得采用停在99%、覆盖已有资源、静默删除产物或把产物无限堆在建筑中。

### 整批转移
默认采用整批转移。只有合法存储能够接收完整批次时才转移。不得拆分为部分入库、部分留缓冲，除非配方或存储规则明确允许拆分。

### 建筑拆除与摧毁
主动拆除前必须确认有待转移产物时禁止拆除，要求先处理存储空间，不得静默删除产物。被摧毁时优先尝试将完整批次产物转入合法存储；无法完整转入则按资源损失规则处理，不得复制产物。无法迁移的产物视为随建筑损失。不新增地面掉落系统。

## Runtime存储实现状态（2026-06-19）

- 全局资源栈和建筑本地资源栈均保存`Amount`与`LockedAmount`；可用量恒为两者之差。
- 建筑本地库存使用共享总容量，不按资源类型分别扩容；输入锁仍占用原库存容量。
- 生产输入查找顺序为全局库存优先、本地库存补足，全部输入规划成功后才一次性写入锁。
- 产物转移按稳定资源ID顺序规划本地与全局去向；任一剩余产物无法容纳时，整批不发生变更。
- 读档会校验锁不大于总量、本地总量不超过容量，以及生产批次声明的锁与库存实际锁完全一致；损坏存档明确拒绝。
- 普通拆除被活动批次、产物缓冲或非空本地库存阻止；事故摧毁会取消批次、解锁输入，并尝试把本地库存与产物缓冲整体迁入全局库存，否则记录明确损失事件。

## Runtime最小物流实现状态（2026-06-20）

- 支持显式创建建筑到建筑、建筑到全局仓库、全局仓库到建筑的单资源运输任务。
- 创建任务时原子检查源可用量和目标未预约容量；成功后源资源增加锁，目标增加入库预约，失败时两者均不改变。
- 运输完成前资源仍属于源库存并占用源容量，目标预约占用目标可接收容量；其他生产或运输不得重复使用锁与预约。
- 同层建筑运输及建筑与抽象全局仓库之间在下一次权威Tick结算；跨层建筑必须存在启用的权威路线，每跨一层延迟一Tick。
- 玩家取消、端点不可运行或路线停用会确定性取消任务并完整释放锁与预约，不消费资源。
- 活动运输阻止普通拆除；事故摧毁先取消相关任务和路线，再执行建筑库存迁移或损失规则。
- 正式管道需先施工；完工后创建仅允许下层向上层、仅允许建造时所选资源的路线。一根管道同一时刻最多维持一个自动运输任务，自动批量可配置或关闭。
- 管道施工或运行期间，端点临时失稳、停用会暂停施工或取消在途任务，恢复运行后继续；端点实际摧毁会清理管道、路线、任务、锁和预约。
- 管道是专用连接设施状态，不占用整数整格`Solid`空间，也不提供承重；其0.5格表现位置由后续连接设施空间阶段处理。
- 运输任务、路线、管道、管道施工、锁、预约与序列进入当前存档2.0、StateHash和联机协议11；读档会核对端点、方向、资源、任务锁与预约完全一致。
- 当前未实现复杂物流优先级、搬运工、道路寻路、吞吐升级和运输动画。

## Runtime持续产出缓存（2026-06-20）

- 基础持续产物同样遵守“本地优先、全局补充”的存储顺序，并扣除物流入库预约占用的容量。
- 全部目标容量已满时，只允许保存下一次形成的1个整数单位作为待转移产物；随后停止生产，不允许无限隐藏缓存。
- 腾出容量后先转移待处理产物，再恢复进度；待处理产物存在时普通拆除必须拒绝。

## Runtime全局共享容量（2026-06-20）

- 全局库存使用2000单位基础共享容量；每座完工仓库增加200。共享容量与资源栈的分类上限同时裁决，任一上限不足都不得入库。
- 建筑施工期间不提前提供容量；完工时增加容量。正常拆除前同时计算现有资源与全部入库预约，若剩余容量不足则拒绝拆除。
- 事故摧毁仓库时减少容量但不静默删除现有资源；允许库存暂时超过共享容量，诊断产生可恢复警告，所有新增全局入库在释放容量前停止。
- 拆除退款同样占用拆除后的共享容量；无法容纳的部分继续按既有退款溢出规则记录，不绕过容量限制。
- 状态进入存档2.5、StateHash和联机协议16；旧存档迁移获得2000基础共享容量。
