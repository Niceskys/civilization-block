# 时间系统规则

### 基础时间设定
- [x] 1 个游戏日 = 20 现实分钟
- [x] 是否有昼夜交替：是
  - [x] 昼夜时长比例：各占 1/2（白天10分钟 + 夜晚10分钟）
  - [x] 昼夜对生产的影响：
    - 依赖阳光的建筑（农田、农舍）在夜晚产出暂停（有太阳灯覆盖则不受影响）
    - 持续运行建筑（熔炉房、采掘场等）可继续运作，前提是有工人值守
  - [x] 昼夜对NPC行为的影响：
    - 默认：白天工作，夜晚进入休息/睡眠状态
    - **夜班机制**：玩家可将NPC设为"夜班"（夜晚工作，白天睡觉）
    - **Lv3自动化**：建筑升至Lv3后解锁"自动运转"，夜晚无工人时以 50% 效率运作
  - [x] 昼夜对安全的影响：
    - 夜晚可能生成怪物攻击建筑（详见 7.2 怪物与防御系统）
- [x] 游戏时间控制档位：暂停 / 1倍速 / 2倍速
- [x] 暂停时是否停止所有生产与NPC行为：是
- [x] 游戏时间是否可倒退：否

### 新存档初始状态与首次启动
- 本规则只适用于新创建的存档。
- 新存档首次进入游戏世界时，时间控制状态默认为"暂停"。
- 初始暂停期间，游戏时钟不推进；所有依赖游戏时间推进的生产、施工、NPC行为和事件计时均不推进。
- 玩家仍可观察世界、移动相机、打开界面和进行建筑放置操作。
- 开局暂停期间经过的现实时间不计入游戏内时间，也不计入前30分钟体验流程的计时。
- 当玩家第一次成功确认一个合法建筑的放置，并成功创建该建筑的施工任务时，触发一次"首次时间启动"。
- 首次时间启动触发后，若世界仍处于新存档初始暂停状态，则自动切换为1倍速。
- 若玩家在首次放置前已经手动启动了游戏时间，则首次建筑放置不得覆盖玩家当前选择的时间档位。
- 放置位置不合法、取消放置、只显示建筑虚影或打开建造面板，均不得触发首次时间启动。
- 首次时间启动每个存档最多触发一次；后续放置建筑不得自动改变时间档位。
- 玩家在自动启动后仍可正常使用暂停、1倍速和2倍速控制。
- 读取已有存档时恢复该存档保存的时间状态，不重新套用"新存档默认暂停"。

### 昼夜详细规则
- [x] 夜晚是否需要照明建筑维持部分运作：是（太阳灯覆盖范围可维持农田/农舍夜间产出）
- [x] 是否有季节变化影响昼夜比例：否（EA版本无季节，但有天气系统影响生产效率，详见 1.6 随机事件系统）

### 第一夜战斗保护时间边界
- 新存档D1（第一个游戏日）的唯一夜晚阶段定义为第一夜。
- 第一夜的时间边界：D1正式入夜时刻（D1 10:00）起，至D1夜晚结束并切换到D2白天的时刻（D1 20:00）止，不包含结束时刻。
- 正式范围：D1 10:00 <= 当前游戏时间 < D1 20:00。
  - D1 10:00整已属于第一夜保护范围。
  - D1 20:00整第一夜保护结束，游戏时间切换为D2 00:00。
- 游戏暂停时，时间不推进，因此不会提前结束第一夜。
- 1倍速和2倍速只改变现实等待时间，不改变第一夜的游戏时间边界。
- 存档、退出和读档不得刷新或重新开始第一夜。
  - 读取处于第一夜时间区间内的存档时，保护继续有效。
  - 读取D2及以后时间点存档时，不得重新获得第一夜保护。
- 第一夜保护应由权威的游戏日和昼夜阶段判定，不得仅依赖可被重置的临时UI状态。
- 第一夜期间敌对怪物生成系统跳过整套生成流程，具体规则详见 7.2 怪物与防御系统 § 第一夜无怪物保护。

- [x] 天气系统影响昼夜产出：阴天/阴雨/暴风雨对农田减产，详见 **1.6 随机事件系统** § 天气
