# 时间系统规则

### 基础时间设定
- [x] 1 个游戏日 = 20 现实分钟
- [x] 是否有昼夜交替：是
  - [x] 昼夜时长比例：各占 1/2（白天10分钟 + 夜晚10分钟）
  - [x] 昼夜对生产的影响：
    - 依赖阳光的建筑（农田、农舍）在夜晚产出暂停（有太阳灯覆盖则不受影响）
    - 持续运行建筑（熔炉房、采掘场等）可继续运作，前提是有工人值守
    - **Lv3自动运转**：达到Lv3且无活动工人的建筑可在夜晚按50%效率自动运转，具体行为规则以《2.1 建筑通用规则》「Lv3自动运转」章节为唯一权威源
  - [x] 昼夜对NPC行为的影响：
    - 默认：白天工作，夜晚进入休息/睡眠状态
    - **夜班机制**：玩家可将NPC设为"夜班"（夜晚工作，白天睡觉）
  - [x] 昼夜对安全的影响：
    - 夜晚可能生成怪物攻击建筑（详见 7.2 怪物与防御系统）
- [x] 游戏时间控制档位：暂停 / 1倍速 / 2倍速
- [x] 暂停时是否停止所有生产与NPC行为：是
- [x] 游戏时间是否可倒退：否

### Runtime权威Tick换算

- Runtime权威时间单位为游戏Tick，`1 Tick = 1游戏秒`，`1游戏日 = 86,400 Tick`。
- 1倍速每现实秒推进72 Tick，因此20现实分钟对应1游戏日；2倍速每现实秒推进144 Tick；暂停推进0 Tick。
- 结构宽限、施工、生产、事件与崩塌队列只读取服务器`SimulationTick`，不得读取客户端系统时间。
- 存档保存当前Tick和各截止Tick；退出期间不补算，读档后从原进度继续。
- 联机由服务器决定本次推进Tick，客户端只显示服务器状态；网络延迟不得额外扣除宽限时间。

### Runtime接入状态（2026-06-28）

- 当前Runtime已提供`DayNightCycle`纯派生规则，由`SimulationTick`计算白天/夜晚阶段。
- 该派生规则不保存第二套昼夜状态，不新增存档字段，不升级协议。
- 当前Runtime已把昼夜阶段接入农田夜晚停产裁决：夜晚农田必须有有效太阳灯完整覆盖才会推进生产。
- 当前Runtime尚未把昼夜阶段接入NPC作息、夜班、怪物生成、太阳灯燃料消耗、UI提示或光照渲染。
- 后续系统接入昼夜阶段时，必须读取Runtime权威Tick或派生结果，不得读取客户端系统时间或由UI自行判断昼夜。

### 新存档初始状态与首次启动
- 本规则只适用于新创建的存档。
- 新存档首次进入游戏世界时，时间控制状态默认为"暂停"。
- 初始暂停期间，游戏时钟不推进；所有依赖游戏时间推进的生产、施工、NPC行为和事件计时均不推进。
- 玩家仍可观察世界、移动相机、打开界面和进行建筑放置操作。
- 开局暂停期间经过的现实时间不计入游戏内时间，也不计入前30分钟体验流程的计时。
- 当玩家第一次成功确认一个合法建筑的放置，并成功创建该建筑的施工任务时，触发一次"首次时间启动"。
- 首次时间启动触发后，若世界仍处于新存档初始暂停状态，则自动切换为1倍速。
- 若玩家在首次放置前已经手动启动了游戏时间，则首次建筑放置不得覆盖玩家当前选择的时间档位。
- 放置位置不合法、取消放置、只显示建筑虚影或打开建造面板，均不得触发首次时间启动。
- 首次时间启动每个存档最多触发一次；后续放置建筑不得自动改变时间档位。
- 玩家在自动启动后仍可正常使用暂停、1倍速和2倍速控制。
- 读取已有存档时恢复该存档保存的时间状态，不重新套用"新存档默认暂停"。

### 昼夜详细规则
- [x] 夜晚是否需要照明建筑维持部分运作：是（太阳灯覆盖范围可维持农田/农舍夜间产出）
- [x] 是否有季节变化影响昼夜比例：否（EA版本无季节，但有天气系统影响生产效率，详见 1.6 随机事件系统）

### 第一夜战斗保护时间边界
- 新存档D1（第一个游戏日）的唯一夜晚阶段定义为第一夜。
- 第一夜的时间边界：D1正式入夜时刻（D1 10:00）起，至D1夜晚结束并切换到D2白天的时刻（D1 20:00）止，不包含结束时刻。
- 正式范围：D1 10:00 <= 当前游戏时间 < D1 20:00。
  - D1 10:00整已属于第一夜保护范围。
  - D1 20:00整第一夜保护结束，游戏时间切换为D2 00:00。
- 游戏暂停时，时间不推进，因此不会提前结束第一夜。
- 1倍速和2倍速只改变现实等待时间，不改变第一夜的游戏时间边界。
- 存档、退出和读档不得刷新或重新开始第一夜。
  - 读取处于第一夜时间区间内的存档时，保护继续有效。
  - 读取D2及以后时间点存档时，不得重新获得第一夜保护。
- 第一夜保护应由权威的游戏日和昼夜阶段判定，不得仅依赖可被重置的临时UI状态。
- 第一夜期间敌对怪物生成系统跳过整套生成流程，具体规则详见 7.2 怪物与防御系统 § 第一夜无怪物保护。

### 第一夜新手保护
第一夜是新手战斗保护的时间阶段规则。第一夜定义为新存档D1（第一个游戏日）的唯一夜晚阶段。时间边界：D1 10:00 <= 当前游戏时间 < D1 20:00。

第一夜保护开始时点：D1 10:00（第一个夜晚阶段正式开始）。保护结束时点：D1 20:00（第一夜结束，进入D2白天），或第一夜被游戏时间正式跨越后。

第一夜保护不延长到第二夜。保护在第一夜开始前确定，并持续整个第一夜。不得在第一夜中途因保存、退出、读档、游戏倍速变化、跨越现实日期或修改UI设置而提前结束。

存档与读档：第一夜内保存，读档后仍处于相同游戏时间，第一夜保护继续。第一夜结束后保存，读档后第一夜保护保持结束，不得重新获得保护。旧存档缺少保护状态时，根据当前游戏日和昼夜阶段判断：第一夜尚未结束时恢复保护，已进入第二夜或之后时标记保护已完成。

不补刷、不累积：不在日出时补生成被跳过的怪物，不累积欠下的怪物波次，不在第二夜追加第一夜的怪物数量。

具体怪物和敌袭规则引用7.2 § 第一夜保护。本文件只定义时间边界，不定义怪物种类、概率、数量或掉落。

### Lv3自动运转的时间边界

进入夜晚边界时，生产系统重新检查符合条件建筑的自动运转资格；进入白天边界时，生产系统结束夜间自动运转资格并重新检查正常工人生产。

Lv3要求、适用建筑、50%效率、批次推进、输入输出、连续生产、满仓及存档规则，以《2.1 建筑通用规则》的「Lv3自动运转」章节为唯一权威源。
