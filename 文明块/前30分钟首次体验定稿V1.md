> 阶段五：逻辑校准与基线定稿
> 基于《前30分钟首次体验方案》的矛盾审计与修正

## 一、上一阶段方案矛盾清单

### 矛盾1：初始地皮状态不一致

| 上一方案中的表述 | 问题 |
|----------------|------|
| "4块地皮已解锁"（0-1分钟行） | ✅ |
| "地皮显示'未勘探'标记"（0-1分钟行） | ❌ 未勘探地皮不能建造（1.1地皮基础规则） |
| "玩家2-3分钟可以直接放置房屋" | ❌ 未勘探地皮不可建造 |

**根源**：1.1明确要求"未勘探的地皮不可建造"，勘探消耗1空闲工人+1游戏日。
**结论**：初始地皮若未勘探，前30分钟无法建造任何建筑，流程完全失效。

### 矛盾2：时间轴基准混用

| 事件 | 上一方案标注的"现实时间" | 对应游戏时间 | 矛盾 |
|------|------------------------|-------------|------|
| 进入游戏 | 0分钟 | 游戏未启动（暂停） | 未说明时间从何时开始计算 |
| 首次建造确认 | 3-4分钟 | ？ | 未说明是否从此时开始推进时间 |
| 第一次黄昏 | 14-16分钟 | ？ | 与"1游戏日=20分钟"的关系不明确 |
| 第一次日出 | 25-27分钟 | ？ | 如果从3分钟开始计算，到25分钟=22分钟≈1.1日，与"1日=20分钟"矛盾 |

**根源**：未区分"游戏时间"和"现实时间"的推进关系。

### 矛盾3：堆叠展示时间点与触发条件冲突

| 要求 | 上一方案设计 | 问题 |
|------|------------|------|
| "堆叠在第22分钟展示" | ✅ 有展示 | ❌ 但"需要第3座建筑后触发"，前30分钟内可能达不到 |
| "不强制玩家堆叠" | ✅ 推荐标签 | ❌ 推荐标签仅在建造面板中，玩家若不打开面板就看不到 |
| 堆叠核心特色需要早于15分钟 | ❌ 未满足 | 22分钟展示违反了"早于15分钟"的要求 |

**根源**：堆叠展示依赖"玩家建造了3座建筑"这个条件，但前30分钟正常节奏只能建2-3座。

### 矛盾4：生产反馈时间不可控

| 设计 | 问题 |
|------|------|
| "第一座建筑5分钟完成"（加速） | ✅ |
| "农田20分钟完成"（正式时长） | ❌ 如果玩家在第8分钟建造农田，农田将在第28分钟完成。前30分钟内可能没有食物产出 |
| "食物+2"（启动产出） | ❌ 6.1中无此规则。建筑建成时不自动附带资源。这是虚假产出 |
| "20-25分钟：第一次资源产出" | ❌ 如果农田在第8分钟建造（加速一倍），真实完成时间约在第18分钟（非加速需到第28分钟），"食物+2"无依据 |

**根源**：生产建筑按日结算，且建筑建成时无"启动资源"。

### 矛盾5：NPC入住方式模糊

| 设计 | 问题 |
|------|------|
| "手动选择NPC入住" | 增加操作量 |
| "分配NPC后窗口亮灯" | ✅ 视觉反馈好 |
| 原文档中NPC入住机制 | 1.1地皮基础规则未说明NPC入住是自动还是手动 |
| 但玩家需要管理多个NPC入住 | 前10分钟要求新玩家同时学建造+分配工作+分配入住，操作过载 |

**根源**：原文档未定义NPC是自动入住还是手动分配。

### 矛盾6：新手保护统一时长不合理

| 保护类型 | 上一方案 | 问题 |
|---------|---------|------|
| 拆除全额返还 | 前3游戏日 | 操作保护可以较长 |
| 重新分配无冷却 | 前3游戏日 | 同上 |
| 怪物概率0% | 前3游戏日 | ❌ 玩家60分钟看不到怪物，防御系统教学延迟过长 |
| 无危机事件 | 前3游戏日 | 但1.6原规则是前30日无危机，差异需说明 |

**根源**：将不同类型的保护绑定了相同时长，未根据各自目的分层。

### 矛盾7："强制自动打开建造面板"违反自由原则

| 设计 | 问题 |
|------|------|
| "30秒后建造面板自动打开一次" | ❌ 打断玩家观察，与沙盒自由冲突 |
| "不关闭，玩家可手动关闭" | ❌ 即使可关闭，自动打开已经是强制行为 |

**根源**：以"确保玩家看到建造面板"为名侵犯了玩家的主动探索权。

## 二、锁定后的设计原则

### 核心原则

1. **初始地皮直接可用**：初始4块地皮在新存档开始时均为已解锁、已完成基础勘探、可直接建造。后续新地皮继续使用正式勘探规则。
2. **时间由玩家行为触发**：世界默认为暂停状态。玩家确认放置第一座建筑后，世界自动切换为1倍速。玩家可随时手动暂停。
3. **堆叠必须在15分钟内展示**：堆叠提示不依赖玩家建造数量，而是通过建造模式中的视觉预览自然展示。
4. **生产反馈必须真实**：不赠送虚假资源。第一次资源增长必须来自正式生产结算。
5. **经验习得先于概念理解**：先让玩家做（建造/分配），再让玩家理解（堆叠/职业）。不要求玩家理解全部概念后再操作。
6. **逐层认知负荷管理**：前10分钟只学"放建筑"和"看资源"；前20分钟学"管理NPC"；前30分钟接触"堆叠"概念。
7. **保护分层**：操作保护（长）> 战斗保护（短）> 危机保护（独立）。
8. **提示可忽略，不打扰**：无自动打开面板、无红点、无弹窗式教程。所有提示均在底栏或建造面板内部。

## 三、堆叠首次展示规则

### 初始建筑堆叠兼容性检查

| 初始建筑 | 可作底层 | 可建在上层 | 首局展示是否适合 | 原规则依据 |
|---------|---------|-----------|----------------|-----------|
| 房屋 | ✅ 承重值3，重量1，可承载上层 | ✅ | ✅ 适合作为堆叠展示的底层 | 2.2堆叠交互规则：重量=建造成本÷10，房屋10木材→重量1。承重值3，可承载上层 |
| 农田 | ✅ 承重值3，重量1 | ✅ | ✅ 适合 | 同上。重量=5木材+1水→(5×1)÷10=0.5→1重量 |
| 水井 | ✅ 承重值3，重量10石→(10×2)÷10=2 | ✅ | ⚠️ 重量2，承重3，可承载重量1的建筑 | 同上 |
| 树场 | ✅ 承重值3，重量10木材→1 | ✅ | ✅ 适合 | 同上 |

**结论**：所有4种初始解锁建筑均可作为底层承载上层建筑。堆叠展示可行。

### 堆叠展示的具体规则

| 属性 | 内容 |
|------|------|
| 触发条件 | 玩家第一座建筑完成后，再次进入建造模式（按B） |
| 展示方式 | 当玩家鼠标移过已建建筑顶部时，建筑顶部显示淡蓝色半透明可建造网格（与地面网格同色系但亮度略高） |
| 首次吸附提示 | 当建筑虚影首次吸附到建筑顶部位置时，底栏提示条显示一次："建筑也可以建在其他建筑上方"——持续5秒后自动消失，不弹窗，不记录 |
| 是否强制 | 否——玩家可以继续选择地面位置放置建筑 |
| 是否重复提示 | 仅显示一次。若玩家已堆叠过建筑，不再显示 |
| 后续行为 | 堆叠后正常走承重验证规则。如承重不足，显示红色虚影+"承重不足"原因 |

### 原规则一致性验证

- ✅ 承重验证保留（下层承重值≥上层总重量）
- ✅ 重量计算规则不变（建造成本÷10）
- ✅ 堆叠对齐规则不变（允许悬空）
- ✅ 高度惩罚规则不变（第8层起有效）
- ✅ 仅新增"顶部网格预览"的视觉辅助，不改变堆叠机制

## 四、前期建造节奏比较与裁决

### 三个方案对比

| 项目 | 方案A：前两座加速 | 方案B：基础建筑永久缩短 | 方案C：阶段反馈 |
|------|-----------------|---------------------|--------------|
| 前5分钟反馈 | 房屋5分钟完成，产出可见 | 房屋更短（3-4分钟），更快见产出 | 房屋仍10分钟，但建造过程有阶段视觉变化 |
| 前15分钟能否完成生产建筑 | ✅ 若前5分钟建房屋+立即建农田（加速），约15分钟可完成 | ✅ 更快完成 | ⚠️ 取决于正式建造时长是否缩短 |
| 是否误导正式游戏节奏 | 轻度——前两座加速后恢复正式节奏，落差主要在"等待农田"上 | ⚠️ 基础建筑永久快于进阶建筑，玩家以为所有建筑都这么快 | ✅ 建造等待期间有内容可看，不觉得"在等" |
| 后续节奏落差 | 第3座建筑恢复到正式时长，落差明显 | 中后期建筑时长恢复正式，落差更大 | 落差最小（因为等待期间有内容） |
| 开发成本 | 低——数值调整 | 低——数值调整 | 中——需要新增建造阶段动画和反馈 |
| 与原数值兼容度 | ✅ 仅调整倍率，不改变基础公式 | ⚠️ 改变6.1建筑数值表的基础建筑时长 | ⚠️ 需要新增"建造阶段"系统 |

### 裁决

**采用方案A**：前两座基础建筑加速。

理由：
- 方案B会误导玩家以为所有建筑都这么快，且永久缩短基础建筑时间改变了全局数值一致性
- 方案C的"阶段反馈"增加开发成本且与现有建造系统（从半透明到实体的渐变已经存在阶段感）重叠
- 方案A只需在2.1建筑通用规则中增加一条："进入存档后的前2座建筑（无论类型）建造时间减半"
- 落差在第一座农田完成后即被"第一次产出正反馈"填补，玩家不会持续感受落差

### 验收条件验证

| 条件 | 方案A结果 |
|------|----------|
| 第一座建筑在确认放置后5分钟游戏时间内完成 | ✅ 房屋原0.5日→加速后0.25日=5分钟 |
| 第一座生产建筑在进入存档后15分钟游戏时间内完成 | ✅ 若房屋在第3分钟确认，5分钟完成。第8-9分钟开始建农田(加速1日→0.5日=10分钟)，第18-19分钟完成 |
| 第一次真实资源产出在进入存档后20分钟游戏时间内发生 | ✅ 农田约第18-19分钟完成。分配工人后需等待完整1个游戏日才能产出？→见第五节 |

## 五、基础生产结算规则

### 原文档检查

| 问题 | 原文档答案 | 依据 |
|------|-----------|------|
| 农田4食物/工人/日是每日统一结算还是连续产出 | 【原文档未明确定义】6.1只写"4食物/工人/游戏日"，未说明结算方式 | 6.1建筑数值表 |
| 第一次产出需要等待完整游戏日还是一个生产子周期 | 【原文档未定义】 | - |
| NPC中途加入工作如何计算产出 | 【原文档未定义】 | - |
| NPC中途离开如何计算 | 【原文档未定义】 | - |
| 暂停和倍速如何影响生产 | 暂停时停止所有生产（1.3时间系统规则）。倍速下生产按比例加速 | 1.3时间系统规则 |
| 资源进入建筑缓存还是直接进入全局仓库 | 先存入建筑本地缓存（最大20单位），满后溢出进入全局仓库 | 4.3存储规则 |
| 仓库容量已满时第一次产出如何处理 | 建筑缓存未满（缓存上限20），直接存入缓存 | 4.3存储规则 |

### 最小补充方案（新增规则）

为确保前20分钟可见真实产出，新增以下规则：

**【新增规则】农业生产按生产周期结算而非按日结算**
- 1游戏日 = 24个生产刻（每生产刻≈50秒游戏时间，对应现实时间≈50秒在1倍速下）
- 农田每次生产刻产出 = 4食物 ÷ 24 ≈ 0.167食物/生产刻/工人
- UI显示：在建筑详情面板中显示"产出进度：X/24"进度条
- 资源实际进入建筑缓存的时刻：每满1个整数单位时一次性进入
- 首次产出时刻：约6个生产刻后（约5分钟游戏时间）产出第1单位食物

**【新增规则适用范围】**
- 仅适用于基础生产建筑：农田、水井、树场（后续建筑沿用此周期规则）
- 不影响现有日消耗机制（NPC消耗仍按日计算，在每日结束时一次性扣除）
- 不影响6.1中的"4食物/游戏日"总产出量

**验收条件验证**：
- 第一次资源产出在农田建成并分配工人后约5分钟游戏时间内发生 → ✅ 满足

## 六、NPC居住与工作分配规则

### 自动入住规则

**【新增规则】NPC自动入住**
- 第一座房屋建成后，无住所NPC自动入住，排序规则：
  1. 优先分配给已参与施工的NPC（该NPC已和这栋建筑建立关联）
  2. 其次按剩余寿命从低到高排序（寿命短的NPC优先获得住所保护）
  3. 最后按NPC编号顺序
- 自动入住不改变NPC的职业或状态
- 房屋满员后，多余的NPC持续处于"无住所"状态，但不会触发恐惧值惩罚（新手保护期内）

### 更换住所入口

- 点击房屋 → 详情面板 → 显示"已入住NPC"列表 → 点击NPC头像 → 选择"更换住所" → 选择目标房屋
- 新增"当前无住所"的NPC在NPC列表中有一个标记

### 工作分配规则

| 属性 | 内容 |
|------|------|
| 推荐依据 | 根据该职业的3个偏好属性权重，计算每个NPC的"适配值"（后台计算，不显示公式）。在空闲NPC列表中按适配值从高到低排序 |
| 推荐显示 | 在空闲NPC列表中，前2名推荐NPC的名字旁显示"✓推荐"标签（绿色小字，非弹窗） |
| 是否强制 | 否——玩家可以选择任何NPC，不限于推荐 |
| 首次推荐时机 | 农田建成后，第一次打开工人分配面板时 |
| 显示的信息 | NPC姓名 + 性格标签 + 适配值(用星级表示，1-5星) + 前1项最高属性（如"力量82"） |
| 不显示的信息 | 全部7维属性、社交关系、日常轨迹、职业列表 —— 这些在首次分配时隐藏 |

### NPC无工作时的可见行为

- 空闲NPC在地皮公共区域行走、站立、与其他NPC短暂交谈
- 以25%效率随机帮助附近有工人的建筑（3.3原规则保留）
- 玩家可通过NPC列表中"空闲"筛选查看

### 玩家最终决定规则

- 玩家可以不分配工人到建筑。建筑处于"空闲"状态，无产出，顶部显示正常图标（非停产红色感叹号——生产建筑建好后未分配工人和资源耗尽停产是两种状态，应使用不同图标）

## 七、分层新手保护规则

### 保护层设计

| 保护层 | 保护内容 | 持续时间 | 理由 |
|-------|---------|---------|------|
| **A层：操作试错保护** | 拆除全额返还、NPC重新分配无冷却 | **前5游戏日**（≈100分钟） | 操作熟练需要时间。5日足以覆盖3-4次完整建造循环 |
| **B层：战斗保护** | 怪物生成概率0%、无精英怪物 | **前1游戏日**（20分钟，即第一夜） | 确保玩家先体验夜景，第二夜起进入教学战斗 |
| **C层：危机保护** | 无瘟疫/虫灾/火灾等大型危机事件 | **前10游戏日**（≈200分钟） | 1.6原规则为前30日无危机，此处保留但缩短以让玩家更早看到内容。完整危机规则自第11日起生效 |

### 与原规则的兼容性

| 原规则 | 调整说明 | 标记 |
|-------|---------|------|
| 1.6危机事件"前30日无危机" | 缩短为前10日无危机 | 【原规则调整】 |
| 7.2怪物生成"第1-5日1-3只" | 改为第1-2日0只（第一夜无怪），第3日起恢复生成概率 | 【原规则调整】 |
| 6.3难度选项 | 新增"新手保护期"参数，默认随难度变化 | 【新增规则】 |
| 2.1拆除返还75% | 前5日改为100%返还，第6日起恢复75% | 【原规则调整】 |
| 3.3工作分配取消冷却 | 前5日重新分配无冷却，第6日起恢复正式规则 | 【原规则调整】 |

## 八、教程与推荐状态机

### 教程提示状态机

| 状态 | 进入条件 | 显示内容 | 退出条件 | 是否永久记录 |
|------|---------|---------|---------|------------|
| 等待显示 | 玩家可执行某操作且尚未被执行过 | 提示文本进入待显示队列 | - | 否 |
| 显示中 | 上一条提示已退出（或首次进入游戏）且队列非空 | 底栏提示条显示文本 | 玩家执行了对应操作；或玩家点击提示条"×"；或30秒内无响应 | 是（记录"已显示"） |
| 已完成 | 玩家执行了对应操作 | 不显示 | - | 是 |

## 九、唯一逐分钟时间轴

### 基准说明

- **游戏时间起点**：玩家确认放置第一座建筑时，游戏时间从第1日0:00开始，世界自动切换为1倍速
- **时间单位**：1游戏日 = 20现实分钟（1倍速下）
- **暂停**：玩家可随时按Space暂停。暂停时游戏时间不推进，生产停止
- **2倍速**：使用2倍速时，20分钟的游戏日缩短为10分钟现实时间。首次体验预设为1倍速，玩家可自行调整

## 十三、仍需项目负责人决定的3个问题

### 问题1：农业生产结算方式

| 选项 | 内容 |
|------|------|
| **A：按生产刻结算（本报告采用）** | 1日=24刻，每刻产出4÷24≈0.167食物，积累到整数时入仓。首次产出约5分钟游戏时间后 |
| B：按日统一结算 | 每日结束时一次性产出4食物。首次产出需等待完整1个游戏日后 |
| C：按半日结算 | 1日=2个半日，每个半日产出2食物。首次产出约10分钟游戏时间后 |

**对玩家体验的影响**：
- A：玩家在20-25分钟可见第一次产出，反馈快；但0.167/刻的连续小数在UI中较难展示
- B：玩家需要等待完整1日（约20分钟）才看到第一次产出，前30分钟几乎看不到产出
- C：介于两者之间，10分钟可见产出，但改变了6.1中的每日产出总量

**对开发成本的影响**：
- A：需要新增"生产刻"系统（只是一个int计数器，无UI只需后端逻辑）
- B：零成本（复用现有日结算逻辑）
- C：需要修改结算周期参数

**我的推荐**：**A**。5分钟即可见到第一次产出，确保前30分钟内有真实的正反馈。

**推荐理由**：B方案会导致前30分钟玩家可能完全看不到资源产出，失去"建造→回报"的核心循环。A方案只增加了生产刻计数器，开发成本极低，且与6.1总产出量一致（4/日）。不赠送虚假资源。

### 问题2：第二夜的怪物教学安排

| 选项 | 内容 |
|------|------|
| **A：第二夜仅预警无入侵** | 第二夜(第2游戏日)屏幕边缘泛红微光3秒后消退，提示"你感到外面有动静"，但无实际怪物。第三夜起开始出现普通怪物 |
| B：第二夜出现1-2只最低级怪物 | 第2个夜晚生成1-2只普通怪物(HP为普通×0.5)，建筑被攻击后耐久减5(极低)，被NPC目击后触发教程"怪物出现了！" |
| C：推迟到玩家建造哨塔后 | 不按游戏日计算，而是玩家建造哨塔并分配守卫后才开始生成怪物 |

**对玩家体验的影响**：
- A：给玩家"此处有威胁"的预期，但不造成实际损失。节奏平稳
- B：第二夜即有小规模战斗，但HP减半的怪物极容易被击败。有助于玩家理解防御系统
- C：如果玩家一直不建哨塔，怪物永远不会出现，防御系统可能被完全跳过

**对开发成本的影响**：
- A：低——仅增加一条预警动画，复用现有红光效果
- B：中——需要特殊低难度怪物
- C：低——条件判断

**我的推荐**：**B**。第二夜出现1-2只弱化怪物，让玩家在前30-40分钟内就能接触战斗系统。

**推荐理由**：A虽然更安全但弱化了"夜晚有威胁"的认知。玩家在第一夜无事后可能以为永远安全。B用极低风险的1-2只弱化怪物展示了"这里确实有危险"，同时不会造成损失。HP减半的怪物可以被NPC的25%效率杂活消灭或自然在日出时消失。

### 问题3：拆除返还率的"新手期"和"正式"是否应加入难度选项

| 选项 | 内容 |
|------|------|
| **A：随难度变化** | 简单难度下新手保护期更长或返还率更高；极难难度下新手保护期更短或无保护 |
| B：保护期独立于难度 | 所有难度下新手保护期相同（前5日100%返还） |
| C：所有难度统一75%返还，无新手期优惠 | 不设置新手保护期 |

**对玩家体验的影响**：
- A：极难模式下玩家更需要试错空间，但若减少保护期可能让极难更不公平
- B：统一保护期比较简单，但极难玩家可能觉得"简单和极难的保护一样"不公平
- C：新玩家在极难模式下的一次错误可能致命，流失风险高

**对开发成本的影响**：三者差异极小（均为一个参数配置）

**我的推荐**：**A**，但保持所有难度的返还率在前5日一致（100%），仅在新手保护期时长上区分。

**推荐理由**：
- 拆除返还率100%对所有难度一视同仁——新手试错的权利不应因难度选择而被剥夺
- 仅新手保护期时长随难度变化：简单前7日、普通前5日、困难前3日、极难前1日
- 这样既保证了所有玩家有试错空间，又让高难度玩家更快进入正式规则
