# 堆叠交互规则

> 本文件是光照遮挡、水平相邻空间判定和跨层资源连通的唯一规则权威源。堆叠承重与崩塌见1.2；建筑基础数值和相邻加成数值见6.1。

### 光照遮挡
- [x] 光照遮挡判定标准：正上方1格内存在任何实体建筑即完全遮挡（含非标准尺寸建筑）
- [x] 太阳灯生效是否需要消耗资源：是
  - 消耗：1 燃料/游戏日
- [x] 太阳灯最多向下覆盖层数：3 层（含自身所在层）
- [x] 太阳灯水平覆盖范围：本格 + 相邻4格（上下左右十字形）
- [x] **多层太阳灯是否效果叠加**：是（2个太阳灯可覆盖6层深度，效果累加不衰减）

### 相邻加成/惩罚
- [x] 水平相邻加成（同层）：

#### 同种建筑水平相邻加成
- 相邻加成的完整数值和上限以6.1建筑数值表 § 同种建筑专业化组合为唯一权威源。
- 只计算同层上下左右四个正交方向。
- 对角和上下层不计算。
- 每座建筑独立计算自己的邻居。
- 移动、重新放置、完成施工、拆除或摧毁后立即刷新。
- 与同种数量倍率的组合公式和倍率数值见6.1；数量统计与序位规则见2.1。

- [x] 采掘场相邻熔炉房：双方+20%效率
- [x] 水井相邻农田：双方+15%效率
- [x] 哨塔相邻（连线围成防御圈）：每座+10%攻击范围
- [x] **上下层相邻加成**：暂不设上下层相邻加成（EA阶段可通过管道实现资源连通）

### 跨层资源连通
- [x] 底层建筑与上层建筑的资源是否自动连通：否
  - 默认情况下，每层建筑资源独立
  - 需建造"管道"建筑才能在上下层之间自动传输资源
- [x] 管道连接规则：
  - 一根管道连接上下相邻2层建筑（输入口在下层，输出口在上层）
  - 多层堆叠需逐层铺设管道
  - 管道不占用建筑工人槽位
  - 管道方向一旦建好不可更改（需拆除重建）

### 管道建筑
- 功能：连接同一块地皮的上下层，使资源自动流通
- 建造成本与承重字段：见6.1建筑数值表
- 建造条件：需在上下层之间有空位（管道尺寸0.5×0.5×2，不占用全格）
- 连接范围：仅连接直接相邻的上下2层

### Runtime连接设施实现状态（2026-06-20）
- 管道使用专用连接设施生命周期，不作为普通整格`Solid`建筑放入占用网格；这避免在半格连接空间规则定稿前伪造整数占地。
- 建造时必须指定同一地皮内、水平占地有重叠且垂直直接相邻的下层与上层建筑，并选择唯一资源类型。
- 施工完成后生成固定的“下层输入→上层输出”路线；反向运输或其他资源类型必须拒绝。
- 每根管道可启用或关闭自动运输并设置单次批量；同一管道最多存在一个在途自动任务。
- 端点临时失稳或停用仅暂停施工/自动运输，恢复后继续；端点被实际摧毁时，管道及路线一并失效。
- 相邻两层的每次运输具有1游戏秒权威延迟；表现动画不得改变库存结算时点。
