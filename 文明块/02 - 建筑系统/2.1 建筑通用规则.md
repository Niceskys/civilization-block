# 建筑通用规则

### 建造规则
- [x] 所有建筑建造是否消耗资源：是
- [x] 建筑建造是瞬间完成还是需要时长：需要施工时长
- [x] 建造是否需要工人参与：是（需分配工人进行施工）
- [x] 可消耗额外资源加速建造（如消耗双倍资源，建造时间减半）
- [x] 具体建造时长数值：见 6.1 建筑数值总表

#### 首次合法放置的时间启动触发
- 建筑通过地皮、承重、空间及其他合法性检查，并由玩家确认放置、成功创建施工任务后，视为一次"成功确认建筑放置"。
- 每个存档中的第一次成功确认建筑放置，向时间系统发送一次首次启动触发。
- 放置失败、取消放置、建筑虚影预览和仅打开建造面板均不触发。
- 时间状态、自动启动条件及存档规则统一由《1.3 时间系统规则》维护，本文件不得重复维护完整时间逻辑。

#### 新存档首次建造加速
- 每个存档拥有两项彼此独立的一次性施工加速：首次房屋加速、首次基础生产建筑加速。
- 首次房屋加速仅适用于该存档第一次建造的房屋，施工时长由0.5游戏日（10游戏分钟）缩短为0.25游戏日（5游戏分钟）。
- 首次基础生产建筑加速适用于该存档第一次建造的农田、水井或树场，三类建筑共享一次机会，施工时长由1游戏日（20游戏分钟）缩短为0.4游戏日（8游戏分钟）。
- 房屋加速与基础生产建筑加速互不要求固定先后顺序，使用其中一项不会消耗另一项。
- 基础生产建筑加速被农田、水井或树场中的任意一类使用后，另外两类建筑恢复正常施工时长。
- 第2座及后续房屋使用正常施工时长。
- 基础生产建筑共享加速机会被使用后，后续农田、水井和树场均使用正常施工时长。
- 防御、科技、功能、装饰及其他进阶建筑不享受本规则。
- 存档分别记录`first_home_bonus_used`和`first_basic_prod_bonus_used`。
- 各建筑正常时长和首次加速时长统一以《6.1 建筑数值表》为数值权威来源。

#### 首建加速机会状态
- 每个首建加速机会使用以下三种状态之一：未使用、已预约、已消耗。
- 首座房屋加速机会和首座基础生产建筑加速机会各自独立维护。
- 基础生产建筑类别中，农田、水井、树场共用一次基础生产建筑加速机会。
- 现有持久化字段`first_home_bonus_used`和`first_basic_prod_bonus_used`可继续表示最终"已消耗"结果；程序实现还必须保存当前是否存在预约、预约对应的施工任务实例以及预约对应的建筑类别。

#### 预约与消耗时点
- 首建加速机会只在以下条件全部满足后预约：建筑放置位置已通过全部合法性检查、玩家已最终确认建造、建造资源已成功扣除或锁定、合法施工任务已创建、该类别机会处于未使用、当前没有其他任务预约同一类别机会。
- 预约与施工任务创建必须属于同一原子流程。
- 不得在仅打开建造面板、选择建筑图标、拖动建筑虚影、位置非法、玩家尚未确认、资源不足或施工任务尚未创建时提前预约。
- 如果施工任务创建失败，机会保持未使用。
- 首建加速机会在预约施工任务成功完成并正式转为可用建筑实例时正式消耗。
- 完成事件中必须原子执行：完成建筑施工、建筑转为可用状态、将对应机会标记为已消耗、清除施工任务预约关系、保存最终状态、刷新相关UI。
- 不得在施工任务刚创建、工人开始施工、施工进度首次增加、施工进度达到50%、玩家保存游戏或退出游戏时提前标记已消耗。

#### 取消、失败与摧毁
- 预约任务在成功完工前发生以下终止事件时释放预约，机会恢复为未使用：玩家主动取消施工、建筑放置任务被系统判定失败、在建建筑被敌对攻击彻底摧毁、地皮失效导致施工任务永久终止、建筑实例或施工任务因正式异常处理被删除、其他明确标记为"任务永久失败"的终止事件。
- 恢复时必须：清除预约施工任务标识、将机会恢复为未使用、不保留原任务施工进度、不把机会标记为已消耗、刷新建造UI、保存恢复后的状态。
- 首建加速本身免费，因此不存在首建加速资源退款。建造资源的返还继续使用现有取消、失败或摧毁规则，本轮不修改建造资源返还比例。

#### 暂停和临时阻塞
- 以下状态不会释放预约：游戏暂停、施工暂时缺少工人、施工暂时缺少后续资源、工人寻路暂时失败、施工任务被手动暂停、施工速度暂时为0、存档、退出游戏、读档、切换游戏倍速、打开或关闭其他UI、在建建筑受到伤害但尚未被彻底摧毁。
- 上述情况下，预约继续绑定原施工任务。
- 不得让玩家通过暂停、缺料或读档复制首建加速机会。

#### 并行施工
- 同一类别机会处于"已预约"时，允许玩家创建其他同类别施工任务；其他任务使用正常施工时间，不得共享或复制预约，不得使用同一首建加速机会。
- 预约任务终止并释放机会后，已经创建且按正常时间运行的其他任务不在中途自动改为首建加速，不追溯修改既有任务的总施工时间或已完成进度。
- 下一项在机会恢复后新创建的符合条件任务可以预约机会。
- 首建加速资格以"首个成功预约该机会的施工任务"为目标，不以UI列表顺序或最终完成先后重新分配。

#### 同一时刻事件顺序
- 如果预约任务在同一逻辑时刻同时满足"施工完成"和"被摧毁"，先结算施工完成事件，再处理完成后的建筑受损或摧毁事件。机会标记为已消耗，不因刚完成后被摧毁而恢复。
- 如果施工尚未达到完成状态便被摧毁，任务失败，释放预约并恢复机会。

#### 与额外资源加速的关系
- 首建加速与额外资源施工加速禁止同时作用于同一施工任务。
- 免费首建加速优先于额外资源施工加速。
- 符合首建加速资格的施工任务自动应用首建加速，不提供放弃首建加速的选项，不允许改用额外资源施工加速，不显示可叠加选项，不扣除额外加速资源，不计算双重加速，不产生额外资源加速状态。
- 对于不享受首建加速的普通施工任务，额外资源施工加速继续按现有正式规则使用，资源种类不变，资源消耗数量不变，施工时间效果不变。
- 额外资源加速的原有数值：消耗双倍建造资源，施工时间减半。其他加速不得与首建加速叠加。

#### 存档要求
- 存档必须保存：两类机会是否已消耗、当前是否存在预约、预约的施工任务实例、预约对应的建筑类型或类别、预约任务的施工进度、该任务使用的最终施工时间、是否启用了额外资源施工加速。
- 读档后已预约任务继续使用原加速施工时间，不重复预约，不重复扣除任何资源，不恢复已经消耗的机会，不因读档而清除预约，不重新选择另一项施工任务。
- 如果读档校验发现机会显示已预约但预约施工任务已经不存在，且对应机会尚未被标记为已消耗，则按异常状态修复为机会恢复未使用。如果机会已经标记为已消耗，不得因预约任务记录缺失而恢复机会。

#### 非法双加速状态修复
- 如果旧存档、错误实现或异常状态出现同一施工任务同时拥有首建加速和额外资源施工加速，统一修复为：保留首建加速、移除额外资源施工加速、按原投入数量返还额外加速资源、施工时间恢复为该任务对应的首建加速时间。
- 返还遵循既有资源精度和容量规则。
- 显示非模态修复提示："已保留首次建造加速，并返还重复投入的加速资源"。
- 如果资源存储已满，返还处理沿用现有溢出资源规则。

### 耐久与维护
- [x] 建筑是否有耐久度/损耗报废机制：是
  - 耐久随使用时间自然损耗
  - 随机灾害/事件/怪物攻击可能导致建筑损坏
  - 损坏建筑需消耗资源维修
- [x] 维修规则：消耗同种建造资源，维修量为原建造成本的 30%
  - 例：农田建造成本=5木材+1水，维修需 2木材+1水
  - 维修后耐久恢复至满值

### 拆除规则
- 正常拆除返还75%建造资源。
- 返还基数使用该建筑施工时实际支付的资源快照；额外资源加速造成的双倍投入计入实际支付，后续定义或模组改价不得改变已建建筑的返还基数。
- 每种资源分别按`实际支付量 × 75%`计算并向下取整，禁止因四舍五入产生额外资源。
- 返还受当前仓库容量约束；事件必须同时记录应返还量、实际入库量和未入库溢出量，后续溢出资源系统可消费该记录，当前Runtime不把溢出量强行写入满仓。
- 旧存档没有实际支付快照时，兼容回退为拆除时当前建筑定义的基础成本；该回退仅用于旧档，所有新施工必须保存实际支付快照。
- 结构崩塌、战斗摧毁等非正常拆除不返还建造资源。
- 免费重新放置不属于拆除，不产生资源返还。
- 不存在前5日或前三座建筑拆除100%返还规则。

### 升级与操作
- [x] 建筑是否可以升级：是
  - 可多级升级（每升1级提升产出）
  - 升级后建筑外观发生变化
  - 升级消耗资源，需工人参与施工
  - **Lv3解锁自动运转**（详见下方「Lv3自动运转」专节）
  - [x] 各级升级数值：见 6.1 建筑数值总表
- [x] 建筑是否可被移动：是（消耗时间，需工人搬运）
  - 建筑普通移动需要消耗时间，并由工人执行搬运。
  - 每个存档最先完成的3座玩家建筑，分别拥有一次完成后10游戏分钟内的同地皮免费重新放置机会。
  - 免费重新放置属于新手试错保护，不替代普通移动。
- [x] 同一建筑是否可重复建造：是（可重复，同种建筑数量倍率见下方章节）

### 同种建筑数量倍率与专业化

#### 同种建筑定义
- 同种建筑以正式建筑类型标识判断。
- 以下属于同种：同一个建筑类型的不同实例；同一建筑类型的不同等级；同一建筑类型经过移动或免费重新放置后的实例。
- 以下不属于同种：农田与农舍；采掘场与矿脉挖掘点；哨塔与城墙；不同建筑类型但功能相近的建筑；建筑升级前后被错误当成两个实例。
- 建筑升级、普通移动和免费重新放置不得改变建筑类型、完成序号或同种序位基础记录。

#### 数量倍率
同种数量倍率的完整数值与永久下限由6.1建筑数值表维护。本节只维护统计范围、序位和适用规则。

#### 统计范围
- 当前存档内全部玩家拥有的有效建筑，跨地皮统一统计。
- 不得按每块地皮、每个堆叠、每个屏幕区域、每个生产网络或每层高度分别计算。
- 同种建筑数量按以下条件统计：玩家实际建造、已完成施工、建筑实例仍然存在、尚未被拆除或彻底摧毁、已正式转为可用建筑。
- 以下不进入统计：建筑虚影、尚未完成的施工任务、已取消或失败的施工任务、地图预生成建筑、遗迹、场景装饰物、已拆除建筑、已彻底摧毁建筑、升级过程本身、维修任务。
- 以下状态仍然保留在统计中：缺少工人、缺少输入资源、暂停生产、夜间停产、正在维修、耐久不足但尚未摧毁、正在普通移动、正在免费重新放置、玩家手动关闭自动运转。

#### 有效建筑与序位
- 同种建筑按原始施工完成游戏时刻从早到晚排序。
- 同一游戏时刻完成时，按施工任务创建序号从小到大确定完成顺序。
- 完成顺序必须写入存档，存档和读档后顺序不变。
- 移动、升级、维修、停产和改名不改变完成顺序。

#### 拆除和摧毁后的重排
- 当某座建筑被拆除或彻底摧毁时，该建筑从有效同种建筑集合中移除。
- 剩余建筑按原始完成顺序重新连续编号，后续建筑可以向前递补序位。
- 对应数量倍率立即重新计算。
- 新建建筑完成后排在当前有效同种建筑集合的最后。

#### 倍率适用范围
- 同种数量倍率只影响使用"建筑效率""生产效率"或"运转效率"参与计算的动态输出，包括但不限于：资源生产速度或产量、加工进度、研究效率、治疗效率、其他明确引用建筑效率的运转结果。
- 同种数量倍率不直接改变：建造成本、施工时长、升级成本、维修成本、最大耐久、当前耐久、承重值、建筑尺寸、工人槽位、床位数量、存储容量、输入资源配方、攻击范围、基础攻击伤害、建筑解锁条件、首次建造加速、免费重新放置资格。
- 如果某种建筑没有使用"建筑效率"参与任何运转计算（如纯容量或纯结构建筑），则记录其同种序位，但不因此改变床位、容量或结构数值。

#### 与相邻加成的组合
- 同种数量倍率与同种水平相邻加成采用乘算。
- 正式公式和相邻加成数值以6.1建筑数值表 § 同种建筑专业化组合为唯一权威源。
- 同种数量倍率和同种相邻加成不得互相覆盖或二选一；二者先形成专业化组合倍率，再作为建筑最终效率计算中的一个独立因素。
- 相邻加成的空间判定规则详见 2.2 堆叠交互规则 § 同种建筑水平相邻加成。

#### 存档要求
- 至少需要持久保存：建筑类型标识、建筑实例标识、原始施工完成游戏时刻、施工任务创建序号、建筑是否有效存在。

### Lv3自动运转

本节是Lv3自动运转机制的唯一规则权威源。时间系统只负责提供昼夜边界和触发时点，具体行为均以本节为准。

#### 适用范围

Lv3自动运转适用于所有达到Lv3且属于生产类或功能类的正式建筑。防御建筑、装饰建筑、纯容量建筑（如仓库）不因Lv3自动获得生产能力。实际适用建筑以6.1建筑数值表的建筑类型和生产能力标识为准，本轮不扩大或缩小具体范围。

#### 解锁条件

建筑达到Lv3后解锁自动运转资格。不得改为Lv2、Lv4或科技解锁。若建筑详情明确设置了高于Lv3的特殊自动运转条件，以详情为准，但不得低于Lv3。

#### 夜晚触发

进入夜晚后，符合条件的Lv3建筑在没有活动生产工人时，可以进入自动运转。

「没有活动生产工人」是指：
- 当前没有能够正常执行该建筑生产工作的工人；
- 夜班工人正在正常工作时，不启用无人自动运转；
- 不因工人暂时寻路中的单帧状态反复切换；
- 具体工人有效性继续遵守现有工作分配规则。

自动运转不得和正常工人生产同时叠加。

#### 50%效率

自动运转效率：基础生产速度的50%。

50%计算规则：
- 50%作用于未来生产工作量推进速度，不把一批固定产物直接减半；
- 不改变配方输入数量；
- 不改变批次固定输出数量；
- 不改变资源品质；
- 不改变怪物、防御或其他非生产系统；
- 不叠加工人等级、职业、性格或岗位适配加成；
- 不因为夜晚占半个游戏日而再次额外乘以50%。

计算公式：
```
自动运转生产速度 = 建筑当前不含工人加成的合法基础生产速度 × 50%
```

如果建筑还有独立、正式、适用于无人运转的建筑倍率，则按现行倍率顺序计算。

#### 与夜班工人的关系

如果建筑有正常工作的夜班工人：
- 使用正常工人生产模式；
- 不启用无人自动运转；
- 不得在正常产量之外再叠加50%。

夜班工人离岗后，建筑在满足其他条件时，才可切换为自动运转。

白天开始后：
- 自动运转状态结束；
- 重新按正常工人和建筑状态判断；
- 不补发夜间未完成产量；
- 不把夜间50%进度重算为白天100%进度。

#### 与连续生产状态机的关系

自动运转只替代当前批次推进所需的活动工人条件。不会绕过其他生产规则。必须继续遵守：
- 当前配方有效；
- 批次开始时锁定完整输入；
- 批次完成时消耗输入；
- 不预锁多批资源；
- 连续生产逐批检查；
- 满仓进入产物待转移；
- 产物未转移前不开始下一批；
- 建筑停用时不生产；
- 建筑被摧毁时按现有规则终止；
- 输入不足时等待；
- 没有合法配方时不生产。

自动运转不代表无输入生产、无消耗生产、无仓储限制生产或离线凭空结算。

#### 批次进度

夜晚开始时已有活动批次：
- 保留原批次、保留输入锁定、保留已完成工作量；
- 无正常夜班工人且满足自动运转条件时，以50%速度继续；
- 不创建第二个批次；
- 不重复锁定输入。

夜晚开始时没有活动批次：
- 只有连续生产已开启且满足全部输入条件时，才可自动启动新批次；
- 单批模式不得因进入夜晚自动创建新批次；
- 输入不足则保持现有等待规则。

白天开始时：
- 保留当前批次和进度；
- 恢复正常生产条件判断；
- 不重置或按比例转换进度。

#### 暂停条件

即使建筑达到Lv3，以下情况仍不得自动运转：
- 建筑被手动停用；
- 建筑不可运行；
- 建筑正在拆除；
- 建筑已经摧毁；
- 当前没有有效配方；
- 输入不足且没有活动批次；
- 产物缓冲已满；
- 处于产物待转移；
- 正式建筑规则明确禁止该时段运行；
- 自动运转功能被该建筑的明确特殊规则禁用。

#### 存档与读档

必须保存或可由已有状态稳定恢复以下内容：
- 建筑等级；
- 当前是否满足自动运转资格；
- 当前昼夜阶段；
- 当前生产批次、已完成工作量、输入锁定；
- 连续生产设置；
- 产物缓冲；
- 暂停或等待原因。

读档不得：
- 重复开始批次；
- 重复锁定输入；
- 重复生成产物；
- 把50%自动效率重复乘两次；
- 将白天错误恢复为夜间自动运转；
- 将夜班工人生产和自动运转同时恢复。

自动运转状态优先通过「建筑等级 + 当前时间阶段 + 工人状态 + 建筑运行状态」重新校验，不应只信任孤立的UI布尔值。

### 前三座建筑免费重新放置

#### 资格确定
- 每个新存档中，按建筑完成施工并正式转为可用建筑的顺序，确定最先完成的3个玩家建筑实例。
- 第1个完成的建筑实例获得一次免费重新放置机会；第2个获得一次；第3个获得一次。
- 第4个及后续完成的建筑不获得本机会。
- 三个建筑分别拥有各自独立的一次机会；一个建筑使用或失去机会不影响另外两个。
- "最先完成"以完成施工事件为准，不以开始建造顺序、放置建筑虚影顺序、扣除资源顺序或建造面板点击顺序为准。
- 取消或失败的施工任务、尚未完成的在建建筑、建筑升级、建筑维修、地图预生成建筑、遗迹、装饰场景物或其他非玩家施工建筑不计入前三座。
- 对已有建筑执行移动或重新放置不计入前三座完成顺序。
- 如果多个建筑在同一游戏时刻完成，按施工任务创建序号从小到大确定完成顺序。

#### 有效期限与机会消耗
- 每个合资格建筑的有效期限为建筑完成时刻起10游戏分钟。
- 使用游戏时间，暂停时期限不推进；1倍速正常推进；2倍速下仍消耗10游戏分钟。
- 存档和读档后必须保持原完成时刻、截止时刻及剩余时间；退出游戏不会刷新期限。
- 到达截止时刻时仍允许确认；超过截止时刻后机会失效，操作失败且不得移动。
- 以下行为不消耗机会：点击重新放置按钮但随后取消；只进入建筑虚影预览；选择非法位置；合法性检查失败；未确认最终目标位置；操作过程中关闭面板；存档或读档；游戏暂停。
- 只有同时满足目标合法、玩家最终确认、建筑成功移动到目标位置、建筑状态正确保存才消耗机会。
- 成功后该建筑的免费机会永久变为"已使用"。

#### 重新放置范围和合法性
- 免费重新放置只能把建筑移动到当前建筑所在的同一地皮内。
- 不得跨地皮移动、移动到未拥有地皮、改变建筑所属地皮、移动整块地皮或同时移动多个建筑。
- 必须重新验证：地皮归属、地皮可用、空间占用、建筑碰撞、承重、堆叠层数、建筑尺寸、光照、通行、地形及其他A级正式规则规定的合法性条件。
- 合法位置显示绿色虚影，非法位置显示红色虚影并标明原因。
- 免费重新放置只移动被选中的建筑，不携带上方、下方或相连的其他建筑。
- 如果被选建筑当前支撑其他建筑，且移动会使被支撑建筑失去合法承重，则目标位置视为非法，不得完成移动。

#### 状态保留
- 成功重新放置后必须保留：建筑类型、等级、当前耐久值、最大耐久值、已入住NPC、已分配工人、建筑内部库存、输入/输出资源、当前生产进度、自动运转状态、升级解锁状态、自定义名称及其他与建筑实例绑定的持久状态。
- 建筑唯一实例标识必须保持不变。
- 不得重新创建全新建筑实例、重置耐久、清空进度、清空库存、驱逐住户、取消工人分配、重置升级等级或重置首次加速状态。

#### 战斗伤害限制
- 合资格建筑在免费机会使用前，一旦受到任何有效战斗伤害（伤害数值大于0），免费机会立即永久失效。
- 被攻击但最终伤害为0不算有效战斗伤害。
- 战斗伤害包括敌对怪物、敌对单位或正式战斗系统产生的伤害；自然耐久损耗、正常生产损耗和非战斗维护不属于战斗伤害。
- 修复建筑不会恢复已失效的免费机会；读档不能清除战斗伤害失效状态。
- 机会失效后建筑仍可使用普通移动规则。

#### 与普通移动和拆除的区别
- 免费重新放置：每存档最先完成的3座各一次、完成后10游戏分钟内有效、同一地皮、不消耗资源、不需要工人、不消耗时间。免费机会已使用、超时或失效后，建筑仍可使用普通移动。
- 普通移动：消耗时间，需要工人搬运。
- 免费重新放置不属于拆除，不返还资源。
- 正常拆除返还75%建造资源。

#### 存档状态
- 系统必须持久保存：全局玩家建筑完成计数、建筑完成顺序、建筑完成游戏时刻、是否属于前三座、免费重新放置截止游戏时刻、是否已使用、是否因战斗伤害失效。
