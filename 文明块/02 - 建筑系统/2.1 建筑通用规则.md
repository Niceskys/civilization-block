# 建筑通用规则

### 建造规则
- [x] 所有建筑建造是否消耗资源：是
- [x] 建筑建造是瞬间完成还是需要时长：需要施工时长
- [x] 建造是否需要工人参与：是（需分配工人进行施工）
- [x] 可消耗额外资源加速建造（如消耗双倍资源，建造时间减半）
- [x] 具体建造时长数值：见 6.1 建筑数值总表

#### 首次合法放置的时间启动触发
- 建筑通过地皮、承重、空间及其他合法性检查，并由玩家确认放置、成功创建施工任务后，视为一次"成功确认建筑放置"。
- 每个存档中的第一次成功确认建筑放置，向时间系统发送一次首次启动触发。
- 放置失败、取消放置、建筑虚影预览和仅打开建造面板均不触发。
- 时间状态、自动启动条件及存档规则统一由《1.3 时间系统规则》维护，本文件不得重复维护完整时间逻辑。

#### 新存档首次建造加速
- 每个存档拥有两项彼此独立的一次性施工加速：首次房屋加速、首次基础生产建筑加速。
- 首次房屋加速仅适用于该存档第一次建造的房屋，施工时长由0.5游戏日（10游戏分钟）缩短为0.25游戏日（5游戏分钟）。
- 首次基础生产建筑加速适用于该存档第一次建造的农田、水井或树场，三类建筑共享一次机会，施工时长由1游戏日（20游戏分钟）缩短为0.4游戏日（8游戏分钟）。
- 房屋加速与基础生产建筑加速互不要求固定先后顺序，使用其中一项不会消耗另一项。
- 基础生产建筑加速被农田、水井或树场中的任意一类使用后，另外两类建筑恢复正常施工时长。
- 第2座及后续房屋使用正常施工时长。
- 基础生产建筑共享加速机会被使用后，后续农田、水井和树场均使用正常施工时长。
- 防御、科技、功能、装饰及其他进阶建筑不享受本规则。
- 存档分别记录`first_home_bonus_used`和`first_basic_prod_bonus_used`。
- 各建筑正常时长和首次加速时长统一以《6.1 建筑数值表》为数值权威来源。

### 耐久与维护
- [x] 建筑是否有耐久度/损耗报废机制：是
  - 耐久随使用时间自然损耗
  - 随机灾害/事件/怪物攻击可能导致建筑损坏
  - 损坏建筑需消耗资源维修
- [x] 维修规则：消耗同种建造资源，维修量为原建造成本的 30%
  - 例：农田建造成本=5木材+1水，维修需 2木材+1水
  - 维修后耐久恢复至满值

### 拆除规则
- 正常拆除返还75%建造资源。
- 免费重新放置不属于拆除，不产生资源返还。
- 不存在前5日或前三座建筑拆除100%返还规则。

### 升级与操作
- [x] 建筑是否可以升级：是
  - 可多级升级（每升1级提升产出）
  - 升级后建筑外观发生变化
  - 升级消耗资源，需工人参与施工
  - **Lv3解锁自动运转**：建筑升至Lv3后，夜晚无工人时自动以50%效率继续运作
  - [x] 各级升级数值：见 6.1 建筑数值总表
- [x] 建筑是否可被移动：是（消耗时间，需工人搬运）
  - 建筑普通移动需要消耗时间，并由工人执行搬运。
  - 每个存档最先完成的3座玩家建筑，分别拥有一次完成后10游戏分钟内的同地皮免费重新放置机会。
  - 免费重新放置属于新手试错保护，不替代普通移动。
- [x] 同一建筑是否可重复建造：是（可重复，同种建筑数量倍率见下方章节）

### 同种建筑数量倍率与专业化

#### 同种建筑定义
- 同种建筑以正式建筑类型标识判断。
- 以下属于同种：同一个建筑类型的不同实例；同一建筑类型的不同等级；同一建筑类型经过移动或免费重新放置后的实例。
- 以下不属于同种：农田与农舍；采掘场与矿脉挖掘点；哨塔与城墙；不同建筑类型但功能相近的建筑；建筑升级前后被错误当成两个实例。
- 建筑升级、普通移动和免费重新放置不得改变建筑类型、完成序号或同种序位基础记录。

#### 数量倍率
| 当前有效同种建筑序位 | 同种数量倍率 |
| -------- | -----: |
| 第1座      |   100% |
| 第2座      |   100% |
| 第3座      |    90% |
| 第4座及以后   |    80% |
- 前两座同种建筑不受效率衰减。
- 第3座开始出现轻微规模衰减。
- 第4座及以后统一保持80%作为同种数量倍率的永久下限。
- 该下限只约束同种数量倍率，不是最终建筑效率下限；其他系统的负面状态仍可使最终效率低于80%。

#### 统计范围
- 当前存档内全部玩家拥有的有效建筑，跨地皮统一统计。
- 不得按每块地皮、每个堆叠、每个屏幕区域、每个生产网络或每层高度分别计算。
- 同种建筑数量按以下条件统计：玩家实际建造、已完成施工、建筑实例仍然存在、尚未被拆除或彻底摧毁、已正式转为可用建筑。
- 以下不进入统计：建筑虚影、尚未完成的施工任务、已取消或失败的施工任务、地图预生成建筑、遗迹、场景装饰物、已拆除建筑、已彻底摧毁建筑、升级过程本身、维修任务。
- 以下状态仍然保留在统计中：缺少工人、缺少输入资源、暂停生产、夜间停产、正在维修、耐久不足但尚未摧毁、正在普通移动、正在免费重新放置、玩家手动关闭自动运转。

#### 有效建筑与序位
- 同种建筑按原始施工完成游戏时刻从早到晚排序。
- 同一游戏时刻完成时，按施工任务创建序号从小到大确定完成顺序。
- 完成顺序必须写入存档，存档和读档后顺序不变。
- 移动、升级、维修、停产和改名不改变完成顺序。

#### 拆除和摧毁后的重排
- 当某座建筑被拆除或彻底摧毁时，该建筑从有效同种建筑集合中移除。
- 剩余建筑按原始完成顺序重新连续编号，后续建筑可以向前递补序位。
- 对应数量倍率立即重新计算。
- 新建建筑完成后排在当前有效同种建筑集合的最后。

#### 倍率适用范围
- 同种数量倍率只影响使用"建筑效率""生产效率"或"运转效率"参与计算的动态输出，包括但不限于：资源生产速度或产量、加工进度、研究效率、治疗效率、其他明确引用建筑效率的运转结果。
- 同种数量倍率不直接改变：建造成本、施工时长、升级成本、维修成本、最大耐久、当前耐久、承重值、建筑尺寸、工人槽位、床位数量、存储容量、输入资源配方、攻击范围、基础攻击伤害、建筑解锁条件、首次建造加速、免费重新放置资格。
- 如果某种建筑没有使用"建筑效率"参与任何运转计算（如纯容量或纯结构建筑），则记录其同种序位，但不因此改变床位、容量或结构数值。

#### 与相邻加成的组合
- 同种数量倍率与同种水平相邻加成采用乘算。
- 正式公式：专业化组合倍率 = 同种数量倍率 ×（1 + 同种水平相邻总加成）。
- 同种数量倍率和同种相邻加成不得互相覆盖或二选一；二者先形成专业化组合倍率，再作为建筑最终效率计算中的一个独立因素。
- 相邻加成的空间判定规则详见 2.2 堆叠交互规则 § 同种建筑水平相邻加成。

#### 存档要求
- 至少需要持久保存：建筑类型标识、建筑实例标识、原始施工完成游戏时刻、施工任务创建序号、建筑是否有效存在。

### 前三座建筑免费重新放置

#### 资格确定
- 每个新存档中，按建筑完成施工并正式转为可用建筑的顺序，确定最先完成的3个玩家建筑实例。
- 第1个完成的建筑实例获得一次免费重新放置机会；第2个获得一次；第3个获得一次。
- 第4个及后续完成的建筑不获得本机会。
- 三个建筑分别拥有各自独立的一次机会；一个建筑使用或失去机会不影响另外两个。
- "最先完成"以完成施工事件为准，不以开始建造顺序、放置建筑虚影顺序、扣除资源顺序或建造面板点击顺序为准。
- 取消或失败的施工任务、尚未完成的在建建筑、建筑升级、建筑维修、地图预生成建筑、遗迹、装饰场景物或其他非玩家施工建筑不计入前三座。
- 对已有建筑执行移动或重新放置不计入前三座完成顺序。
- 如果多个建筑在同一游戏时刻完成，按施工任务创建序号从小到大确定完成顺序。

#### 有效期限与机会消耗
- 每个合资格建筑的有效期限为建筑完成时刻起10游戏分钟。
- 使用游戏时间，暂停时期限不推进；1倍速正常推进；2倍速下仍消耗10游戏分钟。
- 存档和读档后必须保持原完成时刻、截止时刻及剩余时间；退出游戏不会刷新期限。
- 到达截止时刻时仍允许确认；超过截止时刻后机会失效，操作失败且不得移动。
- 以下行为不消耗机会：点击重新放置按钮但随后取消；只进入建筑虚影预览；选择非法位置；合法性检查失败；未确认最终目标位置；操作过程中关闭面板；存档或读档；游戏暂停。
- 只有同时满足目标合法、玩家最终确认、建筑成功移动到目标位置、建筑状态正确保存才消耗机会。
- 成功后该建筑的免费机会永久变为"已使用"。

#### 重新放置范围和合法性
- 免费重新放置只能把建筑移动到当前建筑所在的同一地皮内。
- 不得跨地皮移动、移动到未拥有地皮、改变建筑所属地皮、移动整块地皮或同时移动多个建筑。
- 必须重新验证：地皮归属、地皮可用、空间占用、建筑碰撞、承重、堆叠层数、建筑尺寸、光照、通行、地形及其他A级正式规则规定的合法性条件。
- 合法位置显示绿色虚影，非法位置显示红色虚影并标明原因。
- 免费重新放置只移动被选中的建筑，不携带上方、下方或相连的其他建筑。
- 如果被选建筑当前支撑其他建筑，且移动会使被支撑建筑失去合法承重，则目标位置视为非法，不得完成移动。

#### 状态保留
- 成功重新放置后必须保留：建筑类型、等级、当前耐久值、最大耐久值、已入住NPC、已分配工人、建筑内部库存、输入/输出资源、当前生产进度、自动运转状态、升级解锁状态、自定义名称及其他与建筑实例绑定的持久状态。
- 建筑唯一实例标识必须保持不变。
- 不得重新创建全新建筑实例、重置耐久、清空进度、清空库存、驱逐住户、取消工人分配、重置升级等级或重置首次加速状态。

#### 战斗伤害限制
- 合资格建筑在免费机会使用前，一旦受到任何有效战斗伤害（伤害数值大于0），免费机会立即永久失效。
- 被攻击但最终伤害为0不算有效战斗伤害。
- 战斗伤害包括敌对怪物、敌对单位或正式战斗系统产生的伤害；自然耐久损耗、正常生产损耗和非战斗维护不属于战斗伤害。
- 修复建筑不会恢复已失效的免费机会；读档不能清除战斗伤害失效状态。
- 机会失效后建筑仍可使用普通移动规则。

#### 与普通移动和拆除的区别
- 免费重新放置：每存档最先完成的3座各一次、完成后10游戏分钟内有效、同一地皮、不消耗资源、不需要工人、不消耗时间。免费机会已使用、超时或失效后，建筑仍可使用普通移动。
- 普通移动：消耗时间，需要工人搬运。
- 免费重新放置不属于拆除，不返还资源。
- 正常拆除返还75%建造资源。

#### 存档状态
- 系统必须持久保存：全局玩家建筑完成计数、建筑完成顺序、建筑完成游戏时刻、是否属于前三座、免费重新放置截止游戏时刻、是否已使用、是否因战斗伤害失效。

### 解锁方式
- [x] 建筑解锁方式：
  - 初始直接解锁（基础建筑）：房屋、农田、水井、树场
  - 其余建筑通过商店购买或任务奖励解锁
- [x] 商店解锁的建筑是否为永久权限：是
