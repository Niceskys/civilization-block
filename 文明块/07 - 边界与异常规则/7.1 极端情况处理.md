# 边界与异常规则

### 资源与生产边界
- [x] 资源不足时，建筑行为：暂停生产（显示"🚫 资源不足"）
- [x] 工人不足时，生产效率计算方式：按工人数量线性衰减（有1人则50%效率，无人则停产）
- [x] 光照条件不满足时，农田：完全停产（有太阳灯覆盖则恢复生产）

### 地皮与堆叠边界
- [x] 地皮全部解锁完毕后的后续玩法：继续扩展堆叠层数 + 优化效率 + 科技树研究 + 随机事件应对
- [x] 堆叠达到层数上限后的提示与处理：显示"⚠️ 已达最大堆叠层数" + 建议横向扩展或升级承重科技
- [x] 地陷事件导致地皮不可用：需修复后恢复，详见 **1.6 随机事件系统** § 地陷

### NPC人口边界
- [x] NPC人口上限：50人（性能考虑，好感度矩阵50×50=2500条）
- [x] 达到上限后：流浪NPC无法加入，显示"城镇人口已满"
- [x] NPC全体死亡处理：按难度分4档（见下方）
- [x] NPC社交边界：城镇仅1人时好感度系统不生效，详见 **3.7 NPC好感度系统** § 边界

### 全体NPC死亡处理（按难度）
- [x] 简单难度：免费救援2名基础NPC（流浪者加入事件）+ 商店可购买更多NPC
- [x] 普通难度：免费救援2名基础NPC（无商店购买选项）
- [x] 困难难度：无免费救援，仅可通过商店用星辉购买NPC恢复
- [x] 极难难度：直接游戏结束，需新开档

### 前三座建筑免费重新放置边界

#### 完成顺序判定
- 每个新存档按建筑完成施工并正式转为可用建筑的顺序确定前三座。
- 取消或失败的施工任务、尚未完成的在建建筑、建筑升级、维修、地图预生成建筑、遗迹、装饰场景物或其他非玩家施工建筑不计入前三座。
- 如果多个建筑在同一游戏时刻完成，按施工任务创建序号从小到大确定完成顺序。

#### 超时与暂停
- 每个合资格建筑的有效期限为建筑完成时刻起10游戏分钟。
- 游戏暂停时，10游戏分钟期限不推进。
- 1倍速正常推进；2倍速下仍消耗10游戏分钟。
- 超过截止时刻后机会失效，操作失败且不得移动。

#### 非法位置和取消
- 点击重新放置按钮但随后取消、只进入建筑虚影预览、选择非法位置、合法性检查失败、未确认最终目标位置、操作过程中关闭面板、存档或读档、游戏暂停均不消耗机会。
- 只有同时满足目标合法、玩家最终确认、建筑成功移动到目标位置、建筑状态正确保存才消耗机会。

#### 战斗伤害
- 合资格建筑在免费机会使用前，一旦受到任何有效战斗伤害（伤害数值大于0），免费机会立即永久失效。
- 被攻击但最终伤害为0不算有效战斗伤害。
- 修复建筑不会恢复已失效的免费机会。
- 读档不能清除战斗伤害失效状态。
- 机会失效后建筑仍可使用普通移动规则。

#### 拆除与摧毁
- 建筑被拆除或摧毁后机会不转移，第4座建筑不递补。
- 正常拆除返还75%建造资源。
- 不存在前5日或前三座建筑拆除100%返还规则。
- 免费重新放置不属于拆除，不产生资源返还。

#### 堆叠与依赖建筑
- 免费重新放置只移动被选中的建筑，不携带其上方、下方或相连的其他建筑。
- 如果被选建筑当前支撑其他建筑，且移动会使被支撑建筑失去合法承重，则目标位置视为非法，不得完成移动。

#### 存档与读档
- 存档必须保存：全局玩家建筑完成计数、建筑完成顺序、建筑完成游戏时刻、是否属于前三座、免费重新放置截止游戏时刻、是否已使用、是否因战斗伤害失效。
- 读档不得重新计算前三座。
- 读档不得刷新倒计时。

### 首建加速生命周期与双加速异常

#### 施工任务创建失败
- 首建加速机会在施工任务成功创建时预约。如果施工任务创建失败，机会保持未使用。
- 预约与施工任务创建必须属于同一原子流程。

#### 取消和永久失败
- 预约任务在成功完工前发生以下终止事件时释放预约，机会恢复为未使用：玩家主动取消施工、建筑放置任务被系统判定失败、在建建筑被敌对攻击彻底摧毁、地皮失效导致施工任务永久终止、建筑实例或施工任务因正式异常处理被删除、其他明确标记为"任务永久失败"的终止事件。
- 恢复时必须清除预约施工任务标识，将机会恢复为未使用，不保留原任务施工进度，不把机会标记为已消耗。
- 首建加速本身免费，建造资源返还继续使用现有取消、失败或摧毁规则。

#### 暂停与临时阻塞
- 以下状态不会释放预约：游戏暂停、施工暂时缺少工人、施工暂时缺少后续资源、工人寻路暂时失败、施工任务被手动暂停、施工速度暂时为0、存档、退出游戏、读档、切换游戏倍速、打开或关闭其他UI、在建建筑受到伤害但尚未被彻底摧毁。
- 预约继续绑定原施工任务。

#### 在建建筑被摧毁
- 施工尚未达到完成状态时被摧毁，任务失败，释放预约并恢复机会。

#### 同时完成与摧毁
- 如果预约任务在同一逻辑时刻同时满足"施工完成"和"被摧毁"，先结算施工完成事件，再处理完成后的建筑受损或摧毁事件。机会标记为已消耗，不因刚完成后被摧毁而恢复。

#### 并行施工
- 同一类别机会处于"已预约"时，允许玩家创建其他同类别施工任务；其他任务使用正常施工时间。
- 预约任务终止并释放机会后，已经创建且按正常时间运行的其他任务不在中途自动改为首建加速。

#### 存档与读档
- 存档必须保存：两类机会是否已消耗、当前是否存在预约、预约的施工任务实例、预约对应的建筑类型或类别、预约任务的施工进度、该任务使用的最终施工时间、是否启用了额外资源施工加速。
- 读档后已预约任务继续使用原加速施工时间，不重复预约，不重复扣除任何资源，不恢复已经消耗的机会，不因读档而清除预约。

#### 预约任务记录丢失
- 如果读档校验发现机会显示已预约但预约施工任务已经不存在，且对应机会尚未被标记为已消耗，则按异常状态修复为机会恢复未使用。
- 如果机会已经标记为已消耗，不得因预约任务记录缺失而恢复机会。

#### 非法双加速状态
- 如果旧存档、错误实现或异常状态出现同一施工任务同时拥有首建加速和额外资源施工加速，统一修复为：保留首建加速、移除额外资源施工加速、按原投入数量返还额外加速资源、施工时间恢复为该任务对应的首建加速时间。
- 返还遵循既有资源精度和容量规则。
- 如果资源存储已满，返还处理沿用现有溢出资源规则。

### 存档规则
- [x] 自动存档间隔：每 10 现实分钟自动存档
- [x] 手动存档：支持（玩家可随时手动保存）
- [x] 存档槽位：5 个（3个自动轮替 + 2个手动槽位）
- [x] 读档后是否完全还原所有状态：是
