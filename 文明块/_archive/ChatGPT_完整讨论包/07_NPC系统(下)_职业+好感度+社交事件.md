# 职业系统通用规则

### 核心效率规则（基础设定）
- [x] 无职业普通NPC：所有工作效率基准为 100%
- [x] 职业匹配：NPC从事对应职业的工作时，工作效率 × 200%（翻倍）
- [x] 职业不匹配：NPC从事非对应职业的工作时，工作效率 × 75%
- [x] 职业加成与属性加成的计算方式：**相乘**
  - 最终效率 = 职业匹配倍率 × 属性适配倍率 × 性格修正
  - 详见 **3.3 工作分配逻辑** § 职业效率加成规则

### 职业获取规则
- [x] 职业确定时机：玩家手动分配
- [x] 初始可选择职业数量：8种（全部可用，部分需建筑解锁后触发）
- [x] 高级职业解锁条件：科技解锁 / 商店购买建筑后可用
- [x] 每名NPC最多拥有 3 个职业
  - 多职业规则：
    - 第1职业：满效率（匹配时 ×200%）
    - 第2职业：效率递减至 ×150%（匹配时）
    - 第3职业：效率递减至 ×125%（匹配时）
    - 同一时间只能从事 1 个职业的工作
- [x] 学习新职业的消耗/条件：消耗50星辉 + 学习时间1游戏日（期间不能工作）

### 转职规则
- [x] NPC是否可以更换职业：是（玩家手动操作）
- [x] 转职是否有冷却时间：是，冷却时长：2 游戏日
- [x] 转职是否消耗资源/货币：是，消耗：50 星辉/次
- [x] **转职后原职业是否保留：是**（已学职业永久保留，NPC最多积累3个职业）

### 职业与属性关联
- [x] 属性是否影响职业适配度：是（每职业有3偏好属性，按权重计算适配倍率）
- [x] 高属性NPC是否可解锁特殊职业：否（所有职业均可通过学习获得）
- [x] 职业是否会随工作经验升级：是 → 科技树社会Lv5解锁"大学"后，NPC可自学提升职业等级

# 职业详情清单

> 效率规则：职业匹配对应建筑 = ×200%，不匹配 = ×75%，无职业 = 100%
> 每名NPC最多学习3个职业，第1职业满效率，第2职业×150%，第3职业×125%
> 最终效率 = 职业匹配倍率 × 属性适配倍率 × 性格修正（相乘计算）

### 基础职业（8种）

| 职业名称 | 适配建筑 | 核心属性偏好 | 解锁条件 | 备注 |
|----------|----------|-------------|----------|------|
| 农民 | 农田、农舍 | 体力 > 智力 > 幸运 | 初始解锁 | 食物生产核心 |
| 矿工 | 矿场 | 力量 > 体力 > 幸运 | 初始解锁 | 矿石产出 |
| 樵夫 | 树场 | 力量 > 敏捷 > 体力 | 初始解锁 | 木材产出 |
| 铁匠 | 熔炉房 | 智力 > 力量 > 创造力 | 解锁熔炉房后可用 | 矿石→金属加工 |
| 水工 | 水井 | 体力 > 敏捷 > 智力 | 初始解锁 | 供水保障 |
| 清洁工 | 垃圾处理场 | 体力 > 敏捷 > 幸运 | 解锁垃圾处理场后可用 | 垃圾处理 |
| 守卫 | 哨塔 | 力量 > 体力 > 敏捷 | 解锁哨塔后可用 | 夜间防御战斗 |
| 研究者 | 研究站 | 智力 > 创造力 > 社交 | 解锁研究站后可用 | 科技点产出 |
| 管理者 | 无特定建筑 | 社交 > 智力 > 创造力 | 初始解锁 | 特殊通用职业 |

### 属性适配倍率公式
- [x] 每职业有3个偏好属性，按50%/30%/20%权重计算适配倍率
- [x] **计算公式**：适配倍率 = 0.5×(属性1/50) × 0.5 + 0.3×(属性2/50) × 0.3 + 0.2×(属性3/50) × 0.2
  - 简化公式：适配倍率 = (属性1×0.005) + (属性2×0.0018) + (属性3×0.0008)
  - 平均值50的NPC → (50×0.005)+(50×0.0018)+(50×0.0008) = 0.25+0.09+0.04 = 0.38 → 适配倍率≈1.0
  - 高属性80的NPC → (80×0.005)+(70×0.0018)+(60×0.0008) = 0.40+0.126+0.048 = 0.574 → 适配倍率≈1.51
  - 低属性30的NPC → (30×0.005)+(20×0.0018)+(10×0.0008) = 0.15+0.036+0.008 = 0.194 → 适配倍率≈0.51
- [x] 适配倍率与职业匹配倍率**相乘**
  - 例：研究者匹配×2 × 属性适配1.51 × 勤奋性格1.15 = 3.47倍

### 管理者职业详情
- 管理者不需要绑定特定建筑
- 效果：当管理者处于空闲状态时，**周围相邻地皮**上的建筑效率 +10%
- 多名管理者效果可叠加（上限3名，即最多+30%）
- 管理者不占用建筑工人槽位

### 职业与属性适配
| 职业 | 最适配属性（权重排序） | 属性影响 |
|------|----------------------|----------|
| 农民 | 体力 > 智力 > 幸运 | 高体力=更久持续工作，高智力=更高暴击概率 |
| 矿工 | 力量 > 体力 > 幸运 | 高力量=更多矿石产出，高体力=耐疲劳 |
| 樵夫 | 力量 > 敏捷 > 体力 | 高力量=更多木材，高敏捷=更快砍伐 |
| 铁匠 | 智力 > 力量 > 创造力 | 高智力=更高金属转化率，高创造力=概率双倍产出 |
| 水工 | 体力 > 敏捷 > 智力 | 高体力=稳定供水，高敏捷=更快打水 |
| 清洁工 | 体力 > 敏捷 > 幸运 | 高敏捷=更快处理速度，高幸运=回收稀有资源概率 |
| 守卫 | 力量 > 体力 > 敏捷 | 高力量=更高伤害，高体力=更持久作战 |
| 研究者 | 智力 > 创造力 > 社交 | 高智力=科技点产出速度，高创造力=概率暴击产出 |
| 管理者 | 社交 > 智力 > 创造力 | 高社交=更大影响范围，高智力=更高加成幅度 |

# NPC好感度系统
> 方案B：简单好感度关系 — 一起工作+好感，拥挤住房-好感，影响效率±15%

---

## 一、好感度基础

### 数值范围
- [x] 每对NPC之间存在独立的双向好感度：0 ~ 100
- [x] 默认初始好感度：30（中立友好）
- [x] 好感度显示在NPC详情面板中（头像旁 ❤️数值）

### 好感度档位效果
| 档位 | 范围 | 效果 |
|------|------|------|
| 🤝 友好 | 60-100 | 同建筑工作效率 +15%（两人在同一建筑工作时触发） |
| 😐 中立 | 30-59 | 无加成也无惩罚 |
| 😞 冷淡 | 10-29 | 同建筑工作效率 -10% |
| 😡 敌对 | 0-9 | 同建筑工作效率 -15%，有概率一方辞职 |

---

## 二、好感度增减规则

### 增加途径
| 事件 | 增加值 | 备注 |
|------|--------|------|
| 同建筑工作满1游戏日 | +2 | 每对同建筑NPC每日触发1次 |
| 同房屋居住满1游戏日 | +3 | 居住同房屋的NPC每日+3 |
| 一方被怪物攻击后被另一守卫救下 | +15 | 一次性 |
| 商店道具「社交茶会」使用 | +20（全体互加） | 消耗：3食物+3水 |

### 减少途径
| 事件 | 减少值 | 备注 |
|------|--------|------|
| 拥挤住房（房屋无空位） | -3/日 全体 | 床位不足时，每NPC每日减3（对屋内所有人） |
| 一方死亡（怪物/寿命/事故） | -20 | 对该NPC所有熟人的好感一次性暴跌 |
| 失业超过3游戏日 | -5/日 | 对未分配工作的NPC，与其熟人的好感日降5 |

---

## 三、好感度对建筑分配的影响
- [x] 玩家分配工人时，UI面板显示该NPC与目标建筑中已有工人的好感度平均值
  - 显示为颜色标记：绿色(友好) / 黄色(中立) / 红色(敌对)
- [x] 好感度高(≥60)的NPC优先自动配对至同建筑（自动工作分配逻辑）
- [x] 敌对关系(≤9)的NPC不会被自动分配到同建筑

---

## 四、社交UI
- [x] NPC详情面板底部新增"社交圈"区域：
  - 显示该NPC的Top5好友（按好感度排序）
  - 每个好友显示：头像 + 姓名 + 好感度数值 + 颜色标记
  - 点击好友头像→跳转该好友的NPC详情面板
- [ ] NPC关系图谱全视图：👉 EA期间补充（显示所有NPC关系网络）

---

## 五、与恐惧值联动
- [x] NPC好感度最高的朋友死亡时：额外触发全体恐惧值 +5
- [x] NPC好感度最高的朋友被玩家击杀：该NPC好感度清零 + 全体恐惧值额外 +10

---

## 六、边界规则
- [x] 城镇只有1个NPC时：好感度系统不生效（无配对对象）
- [x] NPC上限50人时：好感度存储矩阵 50×50 = 2500条，性能可接受
- [x] NPC死亡后：其所有好感度数据清空（不保留历史）

# NPC社交事件系统
> 方案C：文字小剧场 + 功能事件链

---

## 一、每日社交消息

- [x] 每游戏日自动生成1-2条NPC社交短消息
- [x] 消息显示在事件日志中（白色/粉色文字标记）
- [x] 消息基于NPC实时状态生成（好感度、职业、性格、所在建筑类型）

### 消息池示例
| 类型 | 消息示例 | 触发条件 |
|------|---------|---------|
| 🗣️ 闲聊 | "🧑‍🌾张三和👩‍🌾李四在农田边聊了一下午收成" | 同建筑工作好感≥40 |
| 💕 好感 | "👷王五今天给💧赵六带了一壶水" | 好感度单方≥60另一方向≥40 |
| 🎉 庆祝 | "👩‍🔧所有人围着喷泉庆祝福利的生产线！" | 当日某项产出创纪录 |
| 😤 摩擦 | "🧑‍🌾张三抱怨👩‍🌾李四干活太慢" | 同建筑工作效率差距>50% |
| 🎵 独处 | "🛡️守卫钱七在哨塔上哼了一整夜的歌" | 守卫夜班+孤独性格 |
| ☕ 休息 | "🧑‍🔬研究者孙八端着咖啡在屋顶看夕阳" | 研究站休息阶段+乐观性格 |

### 生成规则
- [x] 优先选择好感度最高/最低的NPC对生成消息
- [x] 消息自动写入事件日志（不弹窗，不打断游戏）
- [x] 玩家可在事件日志面板筛选"社交"标签

---

## 二、功能事件链

### 事件类型与效果

| 事件名 | 触发条件 | 概率 | 效果 | 持续时间 |
|--------|---------|------|------|---------|
| 💑 恋爱 | 一对异性NPC好感度均≥80 + 同建筑或同房屋≥5日 | 每对每10日判定一次，概率15% | → 双方同居（合并床位，1床位住2人）→ 生育冷却-1日 | 永久(直到一方死亡) |
| 💔 分手 | 恋爱NPC好感度降至<50 | 降至时立即 | → 恢复独立床位，双方心情-10(恐惧值+5临时) | 即时 |
| 😤 吵架 | 同建筑NPC好感度一方<20另一方<30 | 每5日判定一次，概率20% | → 其中一方辞职（自动变为空闲，该建筑出现空位）→ 双方好感-10 | 即时 |
| 🍽️ 聚餐 | 同房屋NPC≥3人且食物库存≥10 | 每3日判定一次，概率30% | → 全体该房屋NPC好感+5，食物消耗+2 | 即时 |
| 🎂 生日 | NPC出生日（每年一次，每30日） | 固定触发 | → 该NPC↓：全员好感+3，该NPC当日效率+50% | 1游戏日 |
| 🏅 表彰 | 某NPC连续工作≥10日无休息 | 第10日固定触发 | → 全体好感+2，该NPC当日效率+20% | 1游戏日 |
| 🚶 散步 | 空闲NPC好感≥60的另一NPC | 每日判定5% | → 双方好感+3 | 即时 |

### 恋爱/同居规则
- [x] 恋爱NPC入住同一房屋后：1个床位可住2人（合并居住）
- [x] 恋爱双方工作效率+10%（同居加成）
- [x] 恋爱状态在NPC详情面板显示：💕 恋人（对方名字）
- [x] 一方死亡时：另一方悲伤（效率-20%，持续3日）

### 生日规则
- [x] 每个NPC出生时记录其出生日（游戏日序号）
- [x] 每30游戏日触发一次生日（30天=1年）
- [x] 生日自动弹出小通知横幅"🎂 XXX 今天生日！"
- [x] 生日当天该NPC工作效率+50%，全员好感+3
- [x] NPC死亡后不再庆祝生日

### 事件通知
- [x] 功能事件触发时：弹出对应横幅（粉色💕/红色😤/绿色🍽️/金色🎂）
- [x] 纯社交消息写入事件日志不弹窗
- [x] 事件日志添加"社交"筛选标签
