# UI玩家留存+交互优化规范 V2（重构版）

## 核心定位
UI = 情绪驱动 + 行为引导 + 世界反馈系统

## 1. 第一印象系统
- 开局30秒强制危机事件
- 单点操作引导（一个UI→一个行为→一个结果）
- UI初始为“混乱但可读状态”

## 2. UI情绪系统
- 安全（绿色）
- 压力（黄色）
- 崩溃（红色）

UI必须随世界变化而变化

## 3. 核心循环UI
- 生产反馈
- NPC请求
- 危机事件

每10~30秒必须有一次UI刺激

## 4. 离线反馈系统
- 世界演化回放
- NPC变化记录
- 资源变化总结

## 5. UI状态机
由世界状态驱动：
稳定 / 压力 / 危机 / 灾变 / 重构
