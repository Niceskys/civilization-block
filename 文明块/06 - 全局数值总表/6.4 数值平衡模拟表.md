## 1. 模拟目标与适用范围

本表根据当前A级正式源（6.1建筑数值表、6.3难度选项、7.2怪物与防御系统、4.2生产链路总览、4.3存储规则、9.2商品清单等），重新校验全局数值平衡。目标：

1. 验证四档难度开局是否都能维持最低食物和水；
2. 计算各难度初始生存所需最少工人和剩余劳动力；
3. 检验基础生产链吞吐量、燃料链路冷启动风险和仓储阻塞边界；
4. 评估同种建筑效率递减对扩建收益的影响；
5. 检查星辉收入结构、防御解锁条件和自适应支援托底效果；
6. 明确区分正式数值、模拟假设和随机结果，删除虚构精确日期。

**本表不新增正式规则**，所有模拟假设均明确标记。

## 3. 时间、工作与日产量口径

**正式口径**（来自1.3）：

- 1游戏日 = 20现实分钟
- 白天 = 10现实分钟
- 夜晚 = 10现实分钟

**正式日产量解释**（正式输入）：

- `X资源/工人/游戏日`：一名符合条件的工人在默认白班工作制度下，完成一个完整游戏日结算后产生的正式日产量。
- 默认白班已经是正式日产量的基准工作制度，不得再次乘以50%。

**夜班规则**（模拟假设）：

- 夜班改变工作发生的时间段，同一NPC不得因从白班切换夜班而获得双倍日产量。
- 同一NPC一个游戏日仍只计算一份正式日产能力。
- 只有正式源明确允许不同NPC进行独立白班和夜班并分别产生完整工作量时，才能模拟双班。当前基准模拟不采用双班加产。
- 夜班只作为运营方式说明，不作为额外免费产量。

**自动化建筑**（模拟假设）：

- 只有正式源明确写明无工人或自动运行，才可在无工人时产出（如Lv3解锁50%自动运转）。
- 基准模拟不把普通生产建筑按全天自动运行计算。

## 4. 通用计算公式

### 4.1 NPC需求

```
每日总需求_R = 人口 × 单NPC基础日需求_R × 难度消耗倍率
```

### 4.2 单建筑产量

```
建筑日产量 = 分配工人数 × 单工人正式日产量 × 难度资源产出倍率 × 同种建筑倍率 × 其他明确正式效率倍率
```

不得额外乘以「白天占全天的50%」。

### 4.3 资源净变化

```
每日净变化 = 总产出 - NPC消耗 - 配方投入 - 建造投入 - 维修投入 - 商店出售量 - 其他正式消耗
```

一次性建造成本和每日流量必须分开显示。

### 4.4 劳动力

```
可用劳动力 = 成年可工作NPC - 食物岗位 - 水岗位 - 其他生产岗位 - 建设岗位 - 防御岗位 - 暂时不可工作NPC
```

人口增长缺少确定时间线时，不得凭空增加劳动力。

### 4.5 同种建筑倍率

使用6.1正式规则：

| 当前有效同种建筑序位 | 数量倍率 |
| - | -: |
| 第1座 | 100% |
| 第2座 | 100% |
| 第3座 | 90% |
| 第4座及以后 | 80% |

前两座同种建筑不受效率衰减。第4座及以后统一保持80%作为同种数量倍率的永久下限。

### 4.6 建造时间

```
实际建造时间 = 基础建造时间 × 难度建造时长倍率 × 正式建造加速修正
```

必须区分：房屋独立首次加速（0.5→0.25日）、农田/水井/树场共享一次首次加速（1→0.4日）、普通后续建造。

### 4.7 敌袭期望值

```
普通敌袭期望怪物数/夜 = 敌袭触发概率 × 该日数区间怪物数量的期望值 × 自适应支援倍率
```

若数量分布没有正式概率，使用区间中点作为敏感性假设，标记为「均匀分布敏感性假设」，同时保留最小值和最大值。

## 6. 初始人口生存劳动力校验

### 6.1 需求与产出（正式输入）

| 难度 | 人口 | 每日食物需求 | 每日水需求 | 单农田工人日产 | 单水井工人日产 | 最少农田工人 | 最少水井工人 | 剩余劳动力 | 初始食物安全天数 | 初始水安全天数 |
| -- | -: | -: | -: | -: | -: | -: | -: | -: | -: | -: |
| 简单 | 6 | 2.25 | 2.25 | 6.00 | 6.00 | 1 | 1 | 4 | 22.22 | 13.33 |
| 普通 | 4 | 2.00 | 2.00 | 4.00 | 4.00 | 1 | 1 | 2 | 15.00 | 10.00 |
| 困难 | 3 | 1.88 | 1.88 | 3.00 | 3.00 | 1 | 1 | 1 | 10.67 | 8.00 |
| 极难 | 2 | 1.50 | 1.50 | 2.00 | 2.00 | 1 | 1 | 0 | 10.00 | 6.67 |

**计算说明**：

- 计算公式：食物需求 = 人口 × 0.5 × 难度消耗倍率；水需求 = 人口 × 0.5 × 难度消耗倍率
- 单工人日产 = 4（农田/水井基准）× 难度产出倍率
- 最少工人数 = ceil(每日需求 / 单工人日产)
- 剩余劳动力 = 人口 - 最少农田工人 - 最少水井工人
- 安全天数 = 初始库存 / 每日需求

### 6.2 关键结论（正式输入）

**普通难度4人开局**：1农田 + 1水井 = 2名生存必需工人，剩余2名劳动力可用于树场、建造或采矿。**不存在开局生存死局**。

**极难难度2人开局**：1农田 + 1水井 = 2名生存必需工人，剩余劳动力为0。极难开局所有人口被食物和水岗位完全占用，无剩余劳动力从事木材、采矿或建设。**极难开局存在严重劳动力瓶颈，但并非死局**——玩家需依赖初始30木材和15石头完成第一批建筑。

**简单难度6人开局**：1农田 + 1水井 = 2名生存必需工人，剩余4名劳动力，初始资源最为充裕。

*结论：四档难度开局都能维持最低食物和水。极难开局后没有空闲劳动力，所有玩家操作必须利用初始材料窗口期。*

### 9.1 燃料来源（正式输入）

- 6垃圾/日 → 产出 6×0.3 = 1.8燃料/日

### 9.3 是否存在燃料冷启动死锁

**结论**（正式输入 + 计算结果）：

- **垃圾处理场主链**：需要垃圾持续产生才能产出燃料。垃圾不足时熔炉房会停产。
- **树场应急炭化**：不需要燃料、不需要垃圾、不需要铁矿石即可运行。只要有树场和木材，即可产出燃料。
- **无燃料死锁**：树场（初始解锁、仅需10木材建造）可以在燃料为0且垃圾不足时独立产出燃料。应急炭化不依赖燃料链产物。
- 熔炉房恢复运行后，垃圾处理场模式D可作为正常燃料供应，减少木材消耗。
- **备用链路不需要燃料启动，可在无燃料时运行**。

### 9.4 工坊不是燃料来源（废止声明）

旧版可能将工坊列为燃料来源，**已废止**。工坊只负责相邻建筑耐久损耗-50%和工具制造，不产出任何燃料。

## 10. 连续生产、锁定与仓储阻塞

### 10.1 正式规则（来自4.2和4.3）

本模拟严格遵循以下生产状态机规则：

1. **输入锁定**：批次开始时锁定输入资源，锁定资源继续占用原存储容量，不得被其他系统使用。批次完成时锁定输入转为已消耗。
2. **逐批启动**：每批完成后逐批重新检查条件并重新锁定输入，不得预锁多批。
3. **产物缓冲**：每个生产槽最多容纳一个完整批次产物。成功转入存储后允许连续模式启动下一批。
4. **满仓处理**：合法存储空间不足时产物保留在缓冲中，状态进入产物待转移，生产停止。
5. **缺工暂停**：批次中工人离开则进入批次暂停，保留进度和锁定输入。

### 10.2 仓储容量边界（计算结果 + 模拟假设）

| 项目 | 数值 | 来源 |
| --- | ---: | --- |
| 全局仓库基础上限 | 2000 | 4.3正式 |
| 单类上限（食物/水） | 1000 | 4.3正式 |
| 单类上限（木材/石头） | 800 | 4.3正式 |
| 单类上限（铁锭） | 400 | 4.3正式 |
| 单类上限（燃料） | 300 | 4.3正式 |
| 单类上限（垃圾） | 500 | 4.3正式 |
| 每座仓库建筑扩容 | +200 | 4.3正式 |

**边界分析**（模拟假设）：

- 普通难度最小生存配置（1农田+1水井+1树场）日产食物+2、水+2、木材+3，在D10前不易触达全局或单类上限。初始食物30→D10约50，远低于1000上限。
- 当建造多个同种建筑（如3座农田日产+6食物、5座树场日产+15木材）后，木材可能率先触达800上限。
- **满仓风险最可能出现在木材和铁锭**，因这两者是主要建材且单类上限分别为800和400。
- 最低基础生存配置在D10前不会因仓储阻塞影响生产。

### 10.3 连续生产阻塞场景（场景D：存储阻塞）

**条件**：生产输入充足，全局仓储接近满，一个生产批次完成。

**验证过程**（模拟假设）：

1. 批次完成时产物写入产物缓冲（每槽最多1批）。
2. 系统尝试将产物转入合法存储。
3. 如果存储空间不足（全局仓库满或单类上限满），产物保留在缓冲。
4. 状态进入「产物待转移」，连续模式保持开启但**不得启动下一批**。
5. 玩家清出空间（消耗、出售、扩容）后，产物转入存储，下一批自动启动。

**结论**：不再假设无限连续生产或无限建筑缓存。仓储可能成为生产瓶颈，尤其是在铁锭和木材大规模生产时。

### 11.2 适用建筑（6.1明确）

### 11.3 扩产策略比较（计算结果 + 模拟假设）

## 14. 自适应支援敏感性

### 14.1 正式参数（来自6.3和7.2）

| 参数 | 值 |
| --- | --- |
| 默认 | 开启 |
| 作用对象 | 普通敌袭最终怪物数量 |

| 是否作用于BOSS | 否 |
| 是否作用于大型危机 | 否 |

### 14.2 机制说明

自适应支援（原动态难度调节）是玩家遭受连续或严重防守损失后，临时降低未来普通敌袭怪物数量的可选托底机制。它**不是**新手保护、固定难度、怪物成长或BOSS威胁值。

- 严重损失（2+NPC死亡或3+建筑摧毁）：倍率立即降低20个百分点。
- 连续2次普通损失：倍率降低10个百分点。
- 连续2次稳定防守：倍率提高10个百分点（最高100%）。
- 关闭后重新开启从100%和零计数开始。

**结论**：自适应支援已消除旧版「惩罚强者」问题（原150%上限已废止）。当前机制只为防守失败的玩家提供向下托底，不会因玩家表现良好而提高难度。

## 20. 当前结论

### 20.1 已验证结论（由正式数值和明确公式直接得出）

1. **四档难度开局都能维持最低食物和水**。即使在极难难度，1名农田工人+1名水井工人即可满足基本生存。
2. **各难度剩余劳动力**：简单4人、普通2人、困难1人、极难0人。极难开局无空闲工人。
3. **普通难度不存在开局资源死局**。初始30食物+20水+50木材+25石头+100星辉足够完成初始建造（房屋+农田+水井+树场）并维持生产。
4. **燃料冷启动无死锁**。树场应急炭化不依赖燃料链产物，可在零燃料时独立运行。恢复熔炉1日运行需1游戏日和2人日。
5. **同种建筑第3座起出现规模衰减**，但90%/80%倍率仍为正收益。暴力扩建仍然有效，只是边际递减。
6. **固定人口4人模型在普通难度可维持至少60日**不发生生存危机。

### 20.2 条件性结论（依赖玩家策略或随机事件）

1. **铁锭和燃料链解锁取决于星辉筹资速度**。需要出售木材或完成任务获得星辉才能解锁采掘场（500星辉）和熔炉房（600星辉）。
2. **防御解锁受星辉和弹药双重限制**。哨塔（400星辉）解锁后还需守卫（占用劳动力）和箭矢供应（正式源未定义箭矢生产，存疑）。
3. **连续生产可能因仓储阻塞**，尤其是木材（上限800）和铁锭（上限400）大规模生产时。
4. **普通敌袭是否触发是随机的**（30%基础概率/夜），自适应支援只在触发后调整数量。

### 20.3 尚不能得出的结论

- 保证在D15解锁哨塔（**删**）
- 保证每天获得固定星辉（**删**）
- 保证D30人口达到某数字（**删**）
- 精确平均维修费（**删**）
- 精确平均怪物战损（**删**）
- 精确满仓日期（**删**）
- D8自动生育（**删**）

## 21. 限制、假设与后续验证

### 21.1 模拟假设清单

- 固定4人模型用于最低可持续性检验。
- 默认NPC为无职业基准（4食物/4水/3木材/工人/日），不考虑职业加成、属性波动、性格影响。
- 建设工人不同时产生完整生产岗位日产量。
- 不采用双班加产（夜班不额外产生日产量）。
- 不假设随机事件、随机商人、流浪NPC等随机收入。
- 怪物掉落不作为固定收入来源。
- 出售木材是最直接的星辉来源，但有明确机会成本。

### 21.2 明确限制

- 正式源中6.1、6.3、9.2等文件的状态为「补充中」，相关数值可能在后续版本中调整。
- 箭矢生产和消耗的正式数值尚未定义，影响哨塔持续性评估。
- 人口增长缺少确定时间线，无法准确预测D30和D60的劳动力规模。
- 难度倍率对固定配方（如应急炭化、熔炉铁锭产出）的适用方式存在解释风险。主结论采用与4.2工作量规则一致的解释。
