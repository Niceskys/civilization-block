# 《文明块》任务契约Skill

## 适用范围

所有准备修改代码、文档、数据、脚本或配置的正式任务都必须加载本Skill。

本Skill不提供修改授权。实际授权只来自用户当前任务计划书。

---

## 一、任务书最低字段

任务书至少必须包含：

- 指定模型；
- 任务编号；
- 任务标题；
- 单一目标；
- 风险等级；
- 必读文件；
- 允许修改文件；
- 禁止修改范围；
- 具体步骤；
- 不要做；
- 验收标准；
- 验证命令；
- 失败停止条件；
- 回执格式；
- 是否允许提交；
- 是否允许推送。

缺少以下任一项时不得实施：

- 指定模型；
- 任务编号；
- 单一目标；
- 允许修改文件；
- 禁止修改范围；
- 验收标准。

---

## 二、模型处理

任务书声明的模型是目标模型。

AI不能通过自我描述独立验证底层实际模型，因此必须输出：

```text
TASK_DECLARED_MODEL=<任务声明>
MODEL_RUNTIME_VERIFICATION=USER_UI_REQUIRED
```

不得输出“我已确认当前就是指定模型”，除非工具提供了可验证的模型元数据。

用户负责在OpenCode界面确认模型。

---

## 三、预检

修改前执行：

```powershell
git status --short
git branch --show-current
git rev-parse HEAD
```

记录：

- 当前分支；
- 当前提交；
- 工作区状态；
- 现有未提交文件；
- 当前任务允许文件；
- 当前任务禁止文件。

工作区存在任务书未说明的修改时：

- 不修改；
- 不stash；
- 不restore；
- 不reset；
- 不clean；
- 输出`BLOCKED`；
- 保留现场。

---

## 四、范围解释

### 允许修改文件

这是修改上限，不是修改义务。

### 禁止修改文件

任何情况下都不能修改，除非用户重新发布正式任务书明确覆盖。

### 必读文件

只允许读取，不代表允许修改。

### 验收标准

决定任务是否完成，不得自行降低。

### “优化”

只允许优化任务目标范围内的实现质量，不允许：

- 扩大系统；
- 新建架构；
- 迁移技术栈；
- 重命名无关内容；
- 统一全库格式；
- 修复其他问题。

---

## 五、单任务原则

默认：

- 一个任务；
- 一个目标；
- 一个系统；
- 1至5个修改文件；
- 一组明确测试；
- 一次完整回执。

禁止把任务拆成多个未经批准的实施阶段并自动继续。

发现后续需求时记录为PLAN，不执行。

---

## 六、实施前汇报

正式修改前必须输出：

```text
TASK_UNDERSTANDING
- ...

EXPECTED_FILES
- ...

FORBIDDEN_SCOPE_CHECK
- ...

CONSEQUENCES
- ...

IMPLEMENTATION_BOUNDARY
- ...
```

如果预计需要任务外文件，直接`BLOCKED`。

---

## 七、执行限制

不得：

- 修改任务外文件；
- 新增未授权文件；
- 新增依赖；
- 联网；
- 安装；
- 提交；
- 推送；
- 切分支；
- 自动清理；
- 自动回滚；
- 放宽测试；
- 修改规则绕过实现困难；
- 修改实现掩盖规则冲突；
- 将建议当授权；
- 将旧文档当现行规则。

---

## 八、停止条件

以下情况必须停止：

- 任务书矛盾；
- 正式规则冲突；
- 权威源无法确认；
- 需要扩大范围；
- 需要新增字段到核心状态；
- 需要改变存档；
- 需要改变协议；
- 需要新增依赖；
- 必要测试无法运行；
- 任务外文件发生变化；
- 必需命令被用户拒绝；
- 发生相同原因的连续失败。

停止时不得自行选择替代架构。

---

## 九、完成定义

只有同时满足以下条件才可写“完成”：

- 目标实现；
- 修改范围合规；
- 必需验证执行；
- 验证退出码符合要求；
- Git差异没有任务外文件；
- 没有隐藏失败；
- 回执证据完整。

否则使用：

```text
PARTIAL
```

或：

```text
BLOCKED
```
